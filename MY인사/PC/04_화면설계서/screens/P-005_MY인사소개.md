## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어떤 형식으로 표시하는지**를 영역 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)
> **데이터 출처**: 본 화면은 **정적 마케팅 콘텐츠**(하드코딩 카피·목업 이미지)로만 구성된다. 외부 연동·실시간 집계·사용자별 분기가 없으며, 모든 사용자에게 동일하게 노출된다. 모바일 목업은 **실제 MY인사 기능 범위와 일치**해야 한다(미구현·범위 외 기능을 광고하지 않는다).

### 1. 히어로(lux-hero)

- **eyebrow**: 고정 텍스트 `FOR EXECUTIVES`.
- **타이틀**: `하나의 앱으로, 인사의 모든 순간`(2줄).
- **서브 카피**: `연차, 증명서, 경비, FAQ — 직원의 손끝에서 시작되어 0.3초 만에 결재됩니다`.

### 2. 기능 소개 섹션 (기능 블록 4종)

좌우 교차(2·4번 블록은 좌우 반전) 배치. 각 블록은 **번호 / eyebrow / 타이틀 / 설명 / 3개 리스트 / 모바일 목업**으로 구성된다. 모두 정적 카피이며, 목업은 해당 기능의 실제 UI를 축약 표현한다.

- **01 연차 (CALENDAR)**: 타이틀 `연차, 더 이상 결재서가 아닙니다`. 리스트 = `직관적 캘린더 인터페이스` / `실시간 결재 진행 알림` / `팀 일정과 자동 연동`. 목업 = 잔여 연차 카드(12일) + 최근 신청 1건(승인).
- **02 증명서 (DOCUMENTS)**: 타이틀 `증명서를 위해 인사팀에 가지 않습니다`. 설명·리스트는 **실제 수령 방식과 일치**해야 한다 — 발급 완료 후 **MY인사 앱·PC에서 PDF 직접 다운로드**(이메일 발송 아님). 리스트 = `모바일에서 한 번의 탭으로 신청` / `인사담당자 즉시 발급` / `앱·PC에서 바로 다운로드`. 목업 = 증명서 신청 목록(발급완료·대기중 뱃지) + '발급 완료 시 앱·PC에서 다운로드' 안내 배너.
- **03 경비·AI (EXPENSE · AI)**: 타이틀 `영수증을 풀로 붙이는 시대, 이미 지났습니다`. 리스트 = `AI 자동 OCR 인식` / `영수증 사진 클라우드 보관` / `경리나라와 자동 동기화`. 목업 = 영수증 촬영 카드 + 경비 1건(처리중).
- **04 FAQ (FAQ)**: 타이틀 `궁금한 건, 앱에서 바로 확인`. 설명 = `인사담당자가 직접 등록·관리하는 자주 묻는 질문을 앱에서 언제든 확인합니다. 연차·증명서·급여 등 인사 전반의 궁금증을 빠르게 해소합니다.` 리스트 = `인사담당자가 직접 등록·관리` / `인사 전반 자주 묻는 질문 정리` / `앱에서 언제든 열람`. 목업 = **읽기 전용 FAQ 목록**(질문 카드 3건, 각 카드 = 질문 + 답변 요약 말줄임). **'내 질문'/'자주 묻는 질문' 탭, 카테고리 뱃지, 답변 대기/완료 상태, 질문 등록 FAB 없음**(MY인사 FAQ는 카테고리·1:1 질문/답변 기능이 없는 단일 읽기 목록이므로 목업도 이를 따른다).
