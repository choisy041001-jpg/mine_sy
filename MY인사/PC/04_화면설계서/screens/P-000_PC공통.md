## 1. 레이아웃 공통

### 1.1 전체 구조

| 영역 | 설명 |
|------|------|
| 사이드바 | 좌측 내비게이션. 브랜드 영역 + 메뉴 그룹 + 하단 사용자 카드 |
| 사이드바 폭 조절 | 사이드바와 본문 경계를 드래그해 사이드바 폭을 조절할 수 있다(사이드바 전용이며 테이블 컬럼 리사이징과 별개) |
| 메인 | 우측 영역. 상단바(페이지 타이틀·브레드크럼) + 본문(화면별 콘텐츠) |

### 1.2 사이드바 구성

| 구성 요소 | 설명 |
|---------|------|
| 브랜드 영역 | 앱 로고 + "MY인사" 명칭 |
| 관리자 메뉴 그룹 | 대시보드 / 연차 관리 / 업무신청관리 / FAQ 관리 — 관리자 계정 로그인 시에만 표시 |
| 직원 메뉴 그룹 | 대시보드 / 업무신청 / 프로필 관리 — 직원 계정 로그인 시에만 표시 |
| 공통 메뉴 | MY인사 소개 / 모바일 안내 |
| 사용자 카드 | 하단에 로그인 사용자 이름·직책(부서·직급) 표시 |
| 숫자 뱃지 | 업무신청관리 메뉴에만 노출. 처리 상태 '대기' 건수와 동일하며, 0건이어도 숨기지 않고 `0`을 표시 |

### 1.3 권한별 표시 전환

- 로그인 계정 권한에 따라 사이드바 메뉴 그룹·사용자 카드·진입 기본 화면이 자동 결정된다.
- 각 화면의 상단바 타이틀·브레드크럼은 화면별로 매핑된다.

### 1.4 지원 해상도 (공통 요건) ★

> **요건**: PC 웹앱의 **최소 지원 브라우저 가로 해상도는 1280 px** 이다(세로 높이 요건 없음). 모든 PC 화면(P-001~P-012)은 가로 1280 px에서 핵심 콘텐츠가 잘림 없이 표시되도록 설계한다.

| 항목 | 요건 |
|------|------|
| 최소 해상도 | **가로 1280 px** (세로 높이 요건 없음) |
| 기준 폭 | 1280 px 너비를 기준으로 레이아웃을 구성한다. 사이드바(좌측) + 본문(우측)이 1280 px에서 잘림 없이 동시 표시된다 |
| 기준 미만 | 1280 px 미만 폭에서는 레이아웃을 축소하지 않고 **가로 스크롤**을 허용한다(콘텐츠가 깨지거나 겹치지 않도록 함) |
| **가로 스크롤 범위 — 사이드바 고정 + 우측 패널만 스크롤** ★ | 기준 폭 미만으로 좁아져 가로 스크롤이 생길 때, **좌측 사이드바는 화면에 고정**되어 좌우로 밀려나지 않고 항상 보인다. 가로 스크롤은 **우측 콘텐츠 패널(본문) 안에서만** 발생한다(페이지 전체/사이드바가 함께 밀리지 않음). 즉 우측 패널이 1280 − 사이드바 폭(기준 본문 폭) 아래로 좁아지면 그 패널 내부에 가로 스크롤바가 생기고, 사이드바와 상단 영역은 제자리를 유지한다 |

## 2. 툴바·필터 공통

| 컴포넌트 | 요건 |
|---------|------|
| 검색 입력 | 이름 검색 등. 최대 50자 / 1자 이상 입력 시 실시간 필터링(300 ms 디바운스) / 초기화 가능 |
| 기간 필터 | 날짜 범위(시작·종료). 기본 최근 1개월 등 화면별 기준 |

| 액션 버튼 | [엑셀 다운로드] / [+ … 불러오기] 등 |

## 3. 테이블 그리드 공통

| 항목 | 요건 |
|------|------|
| 고정 헤더 | 본문 스크롤 시 헤더 행은 상단에 고정되어 보인다 |
| 고정 컬럼 | 좌/우 프리즈가 필요한 화면은 해당 열을 좌측/우측에 고정 표시(P-002 직원·잔여 열 등) |
| 셀 인라인 편집 | 편집 가능한 셀은 더블클릭으로 직접 편집(P-002 총 연차 등) |
| 텍스트 오버플로우 | **긴 자유 텍스트 열**(FAQ 질문·답변 요약, 제출처/용도, 사유 등)의 셀 내용은 한 줄 말줄임 처리하며, 전체 값은 툴팁·상세 패널에서 확인한다. **구조적(정형) 텍스트 열은 말줄임 없이 자연 폭을 보장**한다(3.1 '구조적 텍스트 열 자연 폭 보장') |
| **셀 내 버튼·태그 컬럼 폭** ★ | 셀에 [수정]·[보기] 등 **버튼이나 상태 태그(비텍스트 인라인 요소)** 가 들어가는 컬럼은 **`요소 폭 + 좌우 셀 패딩` 이상**으로 컬럼 너비를 확보한다. 고정 폭(`table-layout:fixed`) 그리드에서 컬럼 폭이 부족하면 버튼/태그가 셀을 넘쳐 말줄임 처리(한 줄 말줄임 규칙)에 걸려 **요소 우측에 말줄임표(`…`)가 표기되므로** 주의한다(P-004 FAQ 관리 액션 컬럼 등) |
| **테이블 카드 높이 — 가용 세로 공간 전체 채움(패딩 제외) + 내부 스크롤** ★ | 관리자 목록 테이블 카드 높이는 **내역(행 수)과 무관하게 콘텐츠 영역의 가용 세로 공간(브라우저 높이 − 상·하단 여백/패딩)을 전부 채운다**. 건수가 적어도 카드는 줄지 않고 하단까지 채우며(행 아래 빈 영역이 카드 안에 남음), 하단에는 콘텐츠 패딩만 남긴다. 내역이 채운 높이를 초과하면 카드가 더 늘어나지 않고 **테이블 본문 영역의 내부 스크롤**로 처리한다(이때 헤더 행은 상단 고정 — 위 '고정 헤더'). 적용 화면: P-002 연차 관리·P-003 업무신청관리·P-004 FAQ 관리. ※ 직원 업무신청(M/emp-cert) 목록은 종전의 '내역 따라 유동 + 최대 높이 상한' 방식을 유지한다(관리자 목록과 별개) |
| **행 구분선 — 행마다 하단선(스크롤 시 마지막 행 제외)** ★ | 테이블 행은 **행마다 하단에 1px 구분선**을 그린다. 단 **내역이 많아 세로 스크롤이 생기는 경우에는 마지막 행 하단선을 그리지 않는다**(스크롤 영역 경계와 중복되지 않도록). 즉 스크롤이 없을 때만 마지막 행도 하단선을 가져 행 묶음이 닫히고, 스크롤이 있으면 마지막 행 하단선은 생략한다. 적용 범위: 모든 `.tbl` 목록 테이블(연차 관리·업무신청관리·FAQ 관리·각 대시보드 패널·직원 업무신청·모바일 안내 설치 현황 등) **및 테이블이 아닌 행 목록형 패널**(예: 관리자 대시보드 **미처리 업무 패널**의 `.alert-row` 목록 등)에도 동일하게 적용한다(행마다 하단선 + 세로 스크롤 시 마지막 행 하단선 생략) |
| 빈 상태 | 데이터가 없으면 목록 영역에 "조회된 데이터가 없습니다." 안내를 표시 |

### 3.1 컬럼 리사이징 (공통 요건) ★

> **요건**: 모든 테이블 그리드 화면에서 사용자가 **컬럼(항목) 간 경계를 드래그하여 열 너비를 조절**할 수 있어야 한다.

| 항목 | 요건 |
|------|------|
| 적용 범위 | PC 전 화면의 데이터 테이블 그리드 공통 |
| 동작 | 헤더 셀의 우측 경계를 드래그하면 해당 컬럼 너비가 드래그한 만큼 늘어나거나 줄어든다 |
| 마지막 컬럼 | 마지막 컬럼은 별도 조절 없이 우측 잔여 폭을 차지한다 |
| 최소 너비 | 컬럼이 내용 식별이 불가능할 정도로 좁아지지 않도록 최소 너비 이하로는 줄어들지 않는다(기본 최소 폭 적용) |
| **비텍스트 요소 셀 최소 너비** ★ | 셀에 **상태/종류 뱃지 등 알약형 라벨**(예: 상태 `대기`/`발급완료`/`반려`, 증명서 종류 `재직증명서`/`경력증명서`/`기타증명서`)이나 **액션 버튼 그룹**(예: 마지막 액션 열의 `발급처리`/`이력보기`/`사유보기`, FAQ 관리 `수정`/`삭제` 등 `row-actions`)처럼 **줄바꿈·축소되지 않는 강성(rigid) 요소**가 들어가는 열은, 리사이징·재적합 시에도 **요소가 잘려 `…`(말줄임)이 생기지 않도록** 최소 너비를 **(그 열에서 가장 넓은 강성 요소 너비 + 셀 좌우 패딩)** 이상으로 보장한다. 즉 해당 열은 요소가 온전히 보이는 폭 아래로는 줄어들지 않으며(우측 마지막 액션 열도 동일하게 보호되어 버튼이 잘리지 않음), 이 최소 폭은 기본 최소 너비보다 우선한다. |
| **구조적 텍스트 열 자연 폭 보장 — 헤더·데이터 말줄임 방지** ★ | 이름·부서·직급·일자·연락처·상태·증명서 종류·월별 일수·발송결과 등 **구조적(정형) 텍스트 열**은 브라우저 해상도에 따라 폭이 재적합될 때 **헤더 텍스트와 모든 데이터 셀 텍스트가 잘려 `…`(말줄임)이 발생하지 않도록**, 최소 너비를 **(헤더·전 데이터 셀 중 가장 넓은 텍스트 자연 폭 + 셀 좌우 패딩)** 이상으로 보장한다. 가용 폭이 넓어지면 비례 확장하고, 좁아져도 이 자연 폭 아래로는 줄지 않는다. **보장 최소 너비의 합이 표시 영역을 넘으면** 자연 폭을 유지한 채 **영역 내 가로 스크롤**로 처리한다(좌측 사이드바·상단 고정, 우측 콘텐츠 패널 내부 스크롤 — 1.4). **단, 'FAQ 질문·답변 요약', '제출처/용도', '사유' 등 긴 자유 텍스트 열은 제외**하여 종전대로 한 줄 말줄임 + 툴팁/상세로 처리한다(자연 폭 보장 시 테이블이 과도하게 넓어지므로). 위 '비텍스트 요소 셀 최소 너비'(뱃지·버튼) 보호와 병행 적용. 구현: `MY인사_PC.html` `colMinWidth`가 강성 요소 폭에 더해 **헤더·데이터 텍스트 자연 폭**(`textNaturalWidth`)을 최소 너비에 반영하고(긴 자유 텍스트 열은 헤더 `data-flex="1"`로 제외), 초기 노출·리사이즈·화면 전환 시 `refitColWidths`로 재적합. `textNaturalWidth`는 오프스크린 측정과 실제 셀 렌더링 간 **자간 누적·서브픽셀 반올림 오차**(`text-transform`·`letter-spacing` 포함)를 흡수하도록 **안전 여유(약 4px)** 를 더해, 1~3px 미세 초과로 `…`가 뜨던 현상(예: 연차 월별 그리드 `10월`·`11월`·`12월` 헤더)을 제거한다 |
| **전체 폭 한계 (오버플로 방지)** ★ | 컬럼 너비 조절 시 **모든 컬럼 너비의 합이 테이블(그리드) 표시 영역 폭을 넘지 않는다.** 특정 컬럼을 넓히거나 좁히면, 그 변화량을 **드래그한 컬럼의 우측 나머지 컬럼 전체가 현재 폭 비율대로 비례 분배하여 흡수**한다(인접 한 컬럼만 보상하지 않음). 즉 한 컬럼을 넓히면 우측 컬럼들이 비율을 유지하며 함께 줄고, 좁히면 함께 늘어 **전체 테이블 폭은 항상 일정하게 유지**되며 컬럼이 테이블 영역 밖으로 넘쳐(가로 오버플로) 잘리는 현상이 발생하지 않는다. 비례 분배 중 어떤 컬럼이 최소 너비 미만으로 줄어야 하는 경우 해당 컬럼은 최소 너비로 고정하고 **남은 변화량을 그 외 가변 컬럼끼리 다시 비례 분배**하여, 일부 컬럼이 최소 너비에 도달해도 전체 폭은 그대로 유지된다. (단, 테이블 자체가 기본적으로 가로 스크롤되는 화면은 그 스크롤 폭 내에서 동일 규칙을 적용한다) |
| **컨테이너 폭 변경 시 재적합** ★ | 브라우저 창 크기 변경·사이드바 폭 조절·화면 전환 등으로 **테이블 표시 영역(컨테이너) 폭이 바뀌면, 고정된 컬럼 폭을 새 영역 폭에 다시 맞춘다(현재 컬럼 비율 유지).** 영역이 넓어지면 컬럼이 비례 확장해 영역을 채우고, 좁아지면 비례 축소해 가로 오버플로가 생기지 않는다. 단, **기본적으로 가로 스크롤되는 테이블**(예: 연차 월별 그리드·직원 현황 등 자연 폭/최소 폭이 정의된 표)은 그 자연 폭 아래로는 줄이지 않고 가로 스크롤로 처리하며, 영역이 자연 폭보다 넓을 때만 채우도록 확장한다 |
| 다른 동작과의 관계 | 컬럼 리사이징 중에는 헤더 텍스트가 선택되지 않으며, 셀 인라인 편집·정렬 등 기존 동작과 충돌하지 않는다 |
| 고정 컬럼 | 좌/우 고정 컬럼도 너비 조절 대상에 포함되며, 고정 위치는 유지된다 |
| 값 유지 범위 | 조절한 너비는 프로토타입에서 영구 저장하지 않는다(새로고침 시 기본 너비 복원 수준의 목업) |

## 4. 데이터 표시 공통 규칙

### 4.1 부서(department) 표시 — 선택 항목 (공통 요건) ★

> **요건**: 부서는 경리나라 **직원관리에서 필수 등록 항목이 아니므로 부서명이 없는 직원이 존재할 수 있다.** 따라서 부서 정보가 없는 경우 부서 표시는 **공란 처리하거나 생략**되어야 하며, 부서 미입력으로 인해 빈칸·`undefined`·`null` 등이 노출되지 않도록 한다.

| 표시 위치 | 부서 없음 처리 |
|---------|----------------|
| 테이블 부서 컬럼 (P-001 직원현황 / P-002 연차 / P-006 설치현황 등) | 셀을 **공란**으로 표시(빈 셀). 컬럼 자체는 유지 |
| 상세 패널 — 부서 단독 행 | 부서 값이 없으면 **공란** 또는 해당 행 생략 |
| 연결형 메타 문자열 `{이름} ({부서} · {직급})` | 부서가 없으면 **부서 토큰과 구분자(· )를 함께 생략** → `{이름} ({직급})`. 카카오톡 발송 대상 정보 `{이름} ({부서} · {직급}) · {연락처}`도 동일하게 부서·구분자 생략 |
| 사용자 카드(사이드바) 직책 | `{부서} · {직급}`에서 부서가 없으면 `{직급}`만 표시 |

**조건부 규칙**
- 부서는 표시용 정보일 뿐 필터·정렬·집계의 기준으로 사용하지 않는다(부서 필터 미제공과 정합).
- 부서 미입력 직원도 모든 목록·테이블·집계에 정상 포함된다(부서 없음이 누락 사유가 되지 않음).

### 4.2 기타 표시 규칙

| 항목 | 규칙 |
|------|------|
| 일시 형식 | 신청·요청 일시 `YYYY-MM-DD HH:MM`, 날짜 `YYYY-MM-DD` |
| 텍스트 오버플로우 | **긴 자유 텍스트 열·라벨**은 한 줄 말줄임, 상세 내용은 툴팁 또는 상세 패널에서 전체 표시. **구조적 텍스트 열은 말줄임 없이 자연 폭 보장**(3.1) |
| 상태 표기 | 처리 상태(대기/발급완료/반려 등)는 상태 뱃지로 표기 |

> **구조적 열 자연 폭 보장과의 관계**: 텍스트 오버플로우(말줄임 + 툴팁/상세) 규칙은 **긴 자유 텍스트 열**에 적용되는 공통 기본값이며, **구조적(정형) 텍스트 열은 3.1 '구조적 텍스트 열 자연 폭 보장'에 따라 헤더·데이터가 잘리지 않도록 자연 폭을 확보**한다(말줄임 미발생, 폭 초과 시 영역 내 가로 스크롤). 한편 일부 화면은 영역(패널) 폭 자체를 확보해 정형 열 묶음이 잘리지 않도록 별도 최소 폭을 정의할 수 있다. 예) **P-001 관리자 대시보드 — 직원 현황 패널 최소 폭 662 px**(직원 테이블 6개 열 자연 폭 합계 보장).
