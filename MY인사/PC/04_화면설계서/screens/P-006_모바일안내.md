## 화면 상세 설계

### P-006 모바일 안내

| 항목 | 내용 |
|------|------|
| **화면 목적** | 사용자들이 모바일 앱을 설치하도록 QR·링크 안내 및 카카오 알림톡 발송을 지원한다. |
| **진입 조건** | 사이드바 하단 [모바일 안내] |
| **다음 화면** | 없음 (현재 페이지 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 앱 설치 안내 | QR 코드 + 설치 URL | `my-insa.app/install` *(예시 URL — 실제 서비스 도메인 확정 후 교체)* |
| 앱 설치 안내 | 설치 단계 안내 | QR 스캔 → 앱 설치 → 사번 로그인 |
| 앱 설치 안내 | [링크 복사] 버튼 | 클릭 시 설치 URL 클립보드 복사 |
| 사용자 설치 현황 카드 | 카드 제목 | "사용자 설치 현황" |
| 사용자 설치 현황 카드 | [설치알림이력] 버튼 | 카드 우측 상단 배치. 클릭 시 설치알림이력 팝업 오픈 |
| 사용자 설치 현황 카드 | 필터 | 전체 / 미설치만 / 설치 완료만 |
| 사용자 설치 현황 카드 | 테이블 열 구성 | 이름 / 부서 / 직급 / 연락처 / 설치 상태 / 최근 활동 / [카카오톡 발송] 버튼 |
| 사용자 설치 현황 카드 | [일괄 발송] 버튼 | 미설치 사용자 전원 대상 알림톡 일괄 발송 |
| 카카오톡 발송 팝업 | 수신자 표시 | 발송 대상 사용자 이름·소속·연락처 표시 |
| 카카오톡 발송 팝업 | 발송 메시지 | **고정 템플릿(사용자 수정 불가)**. 읽기 전용으로 표시. `{회사명}` 자리는 **로그인한 회사명**으로, `{이름}` 자리는 수신자명으로 시스템이 자동 치환 |
| 카카오톡 발송 팝업 | 고정 메시지 템플릿 | "안녕하세요, {이름}님! {회사명}에서 MY인사 사용자로 초대합니다. MY인사를 통해 어디서든 편하게 연차와 증명서를 관리해보세요!" (표시 시 {회사명}은 로그인 회사명으로 치환, {이름}은 개별 발송 시 수신자명으로 치환·일괄 발송 시 자리표시자 유지) |
| 카카오톡 발송 팝업 | 미리보기 + [발송] 버튼 | 미리보기는 치환 결과를 카카오톡 말풍선 형태로 표시 |
| 설치알림이력 팝업 | 테이블 열 구성 | 발송일시 / 사용자 / 휴대폰번호 / 발송결과 (정상 / 오류) |

**사용자 흐름**
- [링크 복사] 클릭 → 설치 URL 클립보드 복사 후 토스트 "링크가 복사되었습니다." 표시
- [카카오톡 발송] 클릭 → 발송 팝업 오픈 → 메시지 확인/편집 후 [발송] 클릭 → 알림톡 발송
- [일괄 발송] 클릭 → 미설치 사용자 전원 대상 발송 팝업 오픈 → [발송] → 알림톡 일괄 발송
- [설치알림이력] 클릭 → 설치알림이력 팝업 오픈 → 발송 이력(발송일시 / 사용자 / 휴대폰번호 / 발송결과) 확인

**조건부 규칙**

**사용자 설치 현황 기준**
- 표시 대상: 그룹웨어(경리나라) 사용자 목록
- **설치 여부 판단**: 앱 로그인 기록이 한 번이라도 존재하면 **설치 완료**, 단 한 번도 없으면 **미설치**로 간주
- **최근 활동** 컬럼: 해당 사용자의 가장 최근 로그인 일시 표시. 로그인 이력이 없으면 `—` 표시

**카카오 알림톡 발송**
- 발송 방식: **카카오 알림톡** (채널 동의 불필요)
- **메시지는 고정 템플릿**으로 발송된다. 사용자가 본문을 직접 작성·편집할 수 없으며, 발송 팝업에서는 읽기 전용으로만 표시한다. `{회사명}`·`{이름}` 치환자는 시스템이 자동으로 채운다(글자수 제한·카운터 없음).
- 동일 사용자에게 재발송 허용 (횟수 제한 없음)
- 발송 성공/실패 여부는 [설치알림이력] 팝업에서 확인

**빈 상태**
- 사용자 설치 현황 테이블에 표시할 사용자가 없을 경우: **"등록된 사용자가 없습니다."** 표시

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)
> **데이터 출처**: 설치 현황은 **그룹웨어(경리나라) 사용자 목록 + MY인사 앱 로그인 기록**으로 산출된다. 카카오 알림톡 발송 메시지는 외부 입력이 아닌 **시스템 고정 템플릿**이며, 발송 이력은 알림톡 전송 결과로 기록된다.

### 1. 앱 설치 안내 영역

- **QR 코드**: 모바일 앱 설치 URL을 인코딩한 QR 이미지. URL = `my-insa.app/install` *(예시 URL — 실제 서비스 도메인 확정 후 교체)*.
- **설치 URL 칩**: QR 하단에 설치 URL 텍스트 표시.
- **설치 단계 안내**: 3단계 고정 텍스트 — ① QR 스캔 또는 카카오톡 링크 클릭 → ② 앱 설치(또는 웹 모바일 진입) → ③ **사번으로 로그인**.
- **[링크 복사]**: 클릭 시 설치 URL을 클립보드 복사 + "링크가 복사되었습니다." 토스트.

### 2. 사용자 설치 현황 패널

각 행은 그룹웨어(경리나라) 사용자 1명이다. 열(필드)은 좌→우로 다음과 같다. **미설치 사용자 우선** 정렬(설치 완료 사용자는 뒤로).

- **이름**: 사용자명(굵게). 경리나라 직원관리 등록명.
- **부서**: 소속 부서명.
- **직급**: 직급명.
- **연락처**: 휴대폰번호(`010-0000-0000`). 보조 색상.
- **설치 상태**: 뱃지. `설치 완료`(녹색, `badge s`) / `미설치`(주황, `badge w`). **앱 로그인 기록이 1회라도 있으면 설치 완료, 없으면 미설치.**
- **최근 활동**: 가장 최근 로그인 시점(상대 시간 — 예: `5분 전` / `2시간 전` / `어제` / `3일 전`). 로그인 이력이 없으면 `-` 표시.
- **카카오톡 발송**(상태별):
  - 미설치 → **[카카오톡 발송]** 버튼(카카오 옐로) → 개별 발송 팝업(4) 오픈.
  - 설치 완료 → **`불필요`** 회색 텍스트(버튼 미노출).
- **미설치 카운트**: 패널 헤더에 미설치 인원 수(`미설치 N명`, danger 색상) 표시. 테이블 데이터 기준 자동 집계.
- **필터 칩**: `전체` / `미설치만` / `설치 완료만`. 선택 칩 강조, 즉시 목록 갱신.
- **목록 높이·스크롤**: 테이블에는 **최대 7행까지만 노출**하고, 표시 대상이 7명을 초과하면 나머지는 **테이블 내부 세로 스크롤**로 본다(헤더 행은 상단 sticky 고정). 7명 이하(필터 적용 결과 포함)면 스크롤 없이 내용만큼만 표시한다.
- **빈 상태**: 표시할 사용자가 없으면 **"등록된 사용자가 없습니다."**

### 3. 상단 액션 버튼

- **[일괄 발송]**: **미설치 사용자 전원**을 수신자로 발송 팝업(4) 오픈. 미설치 0명이면 "✅ 전 직원이 이미 설치 완료했습니다" 토스트 후 팝업 미오픈.
- **[설치알림이력]**: 설치알림이력 팝업(5) 오픈.

### 4. 카카오톡 발송 팝업

- **제목**: 개별 = `💬 카카오톡 발송 — {이름}`, 일괄 = `💬 카카오톡 일괄 발송 ({N}명)`.
- **받는 사람**: 개별 = `{이름} ({부서} · {직급}) · {연락처}`. 일괄 = 수신자 이름 목록 + `총 {N}명 (미설치)`.
- **발송 메시지**(라벨: `발송 메시지`): **고정 템플릿 · 읽기 전용**(사용자 편집 불가). 입력란이 아닌 읽기 전용 박스로 표시. 글자수 제한·카운터 없음. 표시 시 `{회사명}`은 **로그인한 회사명**으로 치환되어 노출된다(자리표시자 `{회사명}` 텍스트를 그대로 보여주지 않는다). `{이름}`은 **개별 발송 시 수신자명으로 치환**해 표시하고, **일괄 발송 시 `{이름}` 자리표시자를 그대로** 둔다(발송 시 각자 이름으로 자동 치환). → 발송 메시지·미리보기 모두 동일 기준으로 치환.
- **미리보기**: 카카오톡 말풍선 형태. `{회사명}`은 로그인 회사명으로 치환. 개별은 `{이름}`을 실제 수신자명으로 치환해 표시, 일괄은 `{이름}` 자리표시자를 그대로 노출(발송 시 각자 이름으로 자동 치환).
- **버튼**: **[취소]** / **[카카오톡 발송]**(카카오 옐로) → 발송 처리 + "💬 {대상}께 카카오톡 발송 완료" 토스트.

### 5. 설치알림이력 팝업

발송 이력을 조회한다(읽기 전용).

- **제목**: `설치알림이력`.
- **테이블 열**: `발송일시` / `사용자` / `휴대폰번호` / `발송결과`.
  - **발송일시**: `YYYY.MM.DD HH:MM` 형식.
  - **사용자**: 수신자명.
  - **휴대폰번호**: `010-0000-0000`.
  - **발송결과**: `정상`(녹색, `badge s`) / `오류`(빨강, `badge d`). 오류는 hover 시 실패 사유 툴팁("알림톡 발송 실패 — 수신 거부 또는 번호 오류") 표시.
- **버튼**: **[닫기]**.
