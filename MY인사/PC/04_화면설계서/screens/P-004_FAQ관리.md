## 화면 상세 설계

### P-004 FAQ 관리

| 항목 | 내용 |
|------|------|
| **화면 목적** | 인사 FAQ를 등록·수정·삭제 관리한다. |
| **진입 조건** | 사이드바 [FAQ 관리] |
| **다음 화면** | 없음 (현재 페이지 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 카테고리 필터·검색 없음 | FAQ는 **카테고리 구분 없는 단일 목록**으로 관리 |
| 상단 영역 | [FAQ 등록] 버튼 | 우측 정렬 |
| FAQ 목록 테이블 | 질문 / 답변 요약 / 노출 상태 / [수정] 버튼 | |
| FAQ 목록 테이블 | 노출 상태 | **읽기 전용 뱃지**(`노출 중` 녹색 / `비노출` 회색). 노출↔비노출 전환은 **수정 팝업의 노출 상태 select**에서만 변경(목록에 직접 토글 없음) |
| FAQ 등록 팝업 | 노출 상태 / 질문 입력 / 답변 입력 | 질문: **필수** · 최대 200자 · 단일 라인 / 답변: **필수** · 최대 2,000자 · 다중 라인 · 카운터 표시 |
| FAQ 수정 팝업 | 기존 내용 불러와 편집 / [저장] · [삭제] 버튼 | 등록 팝업과 동일한 입력 제약 적용 |
| 삭제 확인 팝업 | [삭제] 클릭 시 표시 | |

**사용자 흐름**
- [+ FAQ 등록] → 등록 팝업 오픈 → 입력 후 [저장] → 목록 반영
- [수정] → 수정 팝업 오픈 → [저장] 또는 [삭제]
- [삭제] → 확인 팝업 → 확인 시 삭제 처리

**조건부 규칙**
- **노출 상태 집계**: 대시보드 '노출중인 질문' KPI 카드는 본 화면의 노출 상태가 **'노출 중'** 인 FAQ 건수를 집계한다. '비노출' 상태로 전환된 FAQ는 집계에서 제외된다.
- **좌측 메뉴 숫자 뱃지 없음**: 좌측 사이드바 [FAQ 관리] 메뉴에는 숫자 뱃지를 표시하지 않는다.

**빈 상태**
- FAQ 목록에 등록된 항목이 없을 경우: **"내역이 없습니다."** 표시

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 등록/수정 — 질문 미입력 | [등록]/[저장] 차단, 질문 필수 |
| 등록/수정 — 답변 미입력 | [등록]/[저장] 차단, 답변 필수 |
| 질문 200자 초과 / 답변 2,000자 초과 | 초과 입력 차단(공통 입력 UI 규칙) |

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)
> **데이터 출처**: FAQ는 외부 연동 없이 **관리자가 P-004에서 직접 등록·수정·삭제**하는 데이터다. **'노출 중'** 상태의 FAQ만 직원 모바일 앱(M-006/M-008) FAQ에 그대로 노출되며, **'비노출'** FAQ는 직원에게 표시되지 않는다. 카테고리 구분은 없다.

### 1. 상단 영역

- **[FAQ 등록] 버튼**: 콘텐츠 영역 우측 상단 정렬. 클릭 시 **FAQ 등록 모달**(3) 오픈. 카테고리 필터·검색 입력은 없다(단일 목록).

### 2. FAQ 목록 테이블

각 행은 등록된 FAQ 1건이다. 열(필드)은 좌→우로 다음과 같다. **등록 순서 기준 정렬**(정렬 변경 UI 없음).

- **질문**: 관리자가 등록한 질문 텍스트. 단일 라인 표시, 길면 말줄임(`…`). (예: `연차는 언제부터 사용할 수 있나요?`)
- **답변 요약**: 답변 본문의 앞부분을 요약해 한 줄로 표시하고 말줄임(`…`) 처리. 보조(secondary) 색상. 전체 답변은 [수정] 모달에서 확인.
- **노출 상태**: 처리 상태 뱃지. `노출 중`(녹색, `badge s`) / `비노출`(회색, `badge muted`). **읽기 전용** — 목록에서 직접 전환하지 않으며, 전환은 [수정] 모달의 노출 상태 select로만 처리.
- **액션**: **[수정]** 버튼 → **FAQ 수정 모달**(4) 오픈. 이 컬럼은 셀 내 버튼이 잘리지 않도록 **`버튼 폭 + 좌우 셀 패딩` 이상의 너비를 확보**한다(고정 폭 그리드에서 폭이 부족하면 버튼 우측에 말줄임표 `…`가 표기됨 — P-000 3. 테이블 그리드 공통 '셀 내 버튼·태그 컬럼 폭' 요건 참조).
- **빈 상태**: 등록된 FAQ가 없으면 **"내역이 없습니다."** 표시.

### 3. FAQ 등록 모달 ([FAQ 등록])

새 FAQ를 등록하는 모달.

- **제목**: `FAQ 등록`.
- **노출 상태**(select): `노출 중` / `비노출`. 기본값 `노출 중`.
- **질문**(필수, ＊): 단일 라인 입력. **최대 200자**(카운터 미표시). placeholder `예) 연차는 언제부터 사용할 수 있나요?`.
- **답변**(필수, ＊): 다중 라인 입력. **최대 2,000자**, 카운터 표시. placeholder `답변 내용을 입력하세요. 직원 앱 FAQ에 그대로 표시됩니다.` 입력 내용은 **직원 앱(M-006/M-008) FAQ에 그대로 노출**된다.
- **버튼**: **[취소]** → 모달 닫기(처리 없음) / **[등록]**(primary) → 질문·답변 입력 시 등록 처리 + 목록 반영 + "FAQ가 등록되었습니다." 안내.

### 4. FAQ 수정 모달 ([수정])

기존 FAQ의 노출 상태·질문·답변을 편집하거나 삭제한다.

- **제목**: `FAQ 수정`.
- **로드 값**: 대상 FAQ의 노출 상태 / 질문 / 답변을 불러와 채운다.
- **입력 제약**: 등록 모달과 동일(질문 200자·답변 2,000자, 답변 카운터 표시).
- **버튼 구성**:
  - **[삭제]**(좌측, danger) → 현재 모달을 닫고 **삭제 확인 모달**(5)을 연다.
  - **[취소]** → 모달 닫기(처리 없음).
  - **[저장]**(primary) → 수정 처리 + "✅ FAQ가 수정되었습니다." 토스트.

### 5. 삭제 확인 모달 (수정 모달 [삭제])

- **제목**: `FAQ 삭제`.
- **본문**: 경고 아이콘 + "이 FAQ를 삭제하시겠습니까?" + 삭제 대상 **질문 텍스트**(`"{질문}"`) + "삭제 후 복구할 수 없습니다."(danger 톤).
- **버튼**: **[취소]** → 삭제 취소 후 **수정 모달로 복귀** / **[삭제 확정]**(danger) → 목록에서 해당 FAQ 제거 + "🗑️ FAQ가 삭제되었습니다." 토스트.
