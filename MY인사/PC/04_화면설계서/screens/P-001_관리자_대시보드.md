**조건부 규칙**

**KPI 카드 — 데이터 기준**

각 카드 수치는 아래 기준으로 집계한다. 모든 KPI 수치는 **페이지 로드 시 1회 갱신**하며 실시간 폴링은 없다.

| 카드 | 집계 데이터 기준 | 클릭 이동 |
|------|----------------|----------|
| **승인할 연차** | **로그인한 관리자 본인이 결재해야 하는** 연차 결재 대기 건수. 대상 결재 유형: ① 연차 사용 신청(휴가신청서) ② 연차 취소 신청. 경리나라 결재함에서 본인이 현재 결재선상 결재 차례인(결재 대기) 건만 집계한다. 본인이 결재자가 아니거나 이미 처리한 건은 제외. | P-002 연차 관리 |
| **미처리 증명서** | 업무신청관리(P-003)의 처리 상태가 **'대기'** 인 증명서 신청 건수. '발급완료'·'반려' 건은 제외. | P-003 업무신청관리 |
| **노출중인 질문** | FAQ 관리(P-004)의 노출 상태가 **'노출 중'** 인 FAQ 건수. '비노출' 건은 제외. | P-004 FAQ 관리 |
| **총 직원수** | 경리나라 직원관리에 등록된 일반근로자 전체 인원수(재직·휴직·퇴직 포함, 직원 현황 패널 '전체' 탭과 동일 기준). | 클릭 불가 |

**미처리 업무 패널**
- 증명서·연차 각각 최대 **10건**까지 표시. 10건 초과분은 표시하지 않으며, 전체 목록은 해당 관리 페이지에서 조회.
- 미처리 업무 패널과 직원 현황 패널의 높이는 **494 px로 고정**(세부 설계 요건 4 참조). 건수가 적어도 패널 하단이 줄어들지 않으며, 내역이 494 px를 초과하면 패널 내부 스크롤로 처리.
- 미처리 업무가 0건일 때: 패널 내 **"미처리 업무가 없습니다."** 빈 상태 표시.
- [승인] 버튼 클릭 시 경리나라 결재함을 **새 창(새 탭)으로 열어줌**. MY인사에서 직접 연차를 승인하지 않음.
- [발급] 버튼 클릭 시 **P-003 업무신청관리의 발급처리 팝업을 호출**. 팝업에서 PDF 첨부 후 발급 완료 처리.

**직원 현황 패널**
- 기본 탭: **전체**. 페이지 로드 시 전체 탭이 선택된 상태로 진입.
- 표시 대상: 경리나라 직원관리에 등록된 **일반근로자** 목록.

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)

### 1. KPI 카드 (상단 4개)

각 카드는 `라벨`(상단 텍스트) + `수치`(하단 대형 숫자) 2개 필드로 구성된다. 집계 기준은 위 **KPI 카드 — 데이터 기준** 표를 따른다. 필드별 표시 규칙은 다음과 같다.

- **라벨**: 고정 문구. 좌→우 순서로 `승인할 연차` / `미처리 증명서` / `노출중인 질문` / `총 직원수`.
- **수치**: 정수만 표시(소수점·단위 없음). 집계 결과가 0이면 `0`을 그대로 표시한다(카드를 숨기지 않음). 데이터 조회 실패 시에도 카드 레이아웃은 유지하며 수치 자리에 `-`를 표시한다.
- **갱신 시점**: 페이지 최초 로드 시 1회 조회. 이후 실시간 폴링·자동 갱신 없음. 다른 화면에서 돌아오거나 새로고침 시 재조회.
- **클릭 동작**: `승인할 연차`→P-002, `미처리 증명서`→P-003, `노출중인 질문`→P-004로 이동. `총 직원수`는 클릭 불가(커서 변화 없음).

### 2. 미처리 업무 패널 (좌측)

패널 헤더에 `미처리 업무 (N)`을 표시하며, **N = 표시 중인 증명서 행 수 + 연차 행 수**이다. 패널 본문은 증명서 신청 행과 연차 신청 행이 시간순으로 혼합되어 나열된다. 각 행은 아래 필드로 구성된다.

- **유형 아이콘**: 행 종류(증명서/연차)에 따라 서로 다른 아이콘 표시.
- **제목(`alert-title`)**:
  - 증명서 행 — `{증명서 종류} 발급 요청` (예: `재직증명서 발급 요청`). 종류는 업무신청관리 신청 데이터의 증명서 종류를 따른다.
  - 연차 행 — **직원이 경리나라 전자결재에 상신한 결재문서 제목**을 그대로 표시한다. 연차 사용 신청은 `휴가신청서` 문서로 상신되므로 제목은 **`휴가신청서`**로 시작하고, 뒤에 신청 날짜·일수 요약을 붙인다(예: `휴가신청서 — 2026-05-30 종일 (1일)`). 날짜·일수는 결재함 연차 신청 데이터를 따르며, 반차/반반차는 `오후 반차` 등 결재 데이터 표기를 따른다.
  - 연차 취소 행 — 연차 취소 신청도 동일하게 **`휴가신청서`** 문서로 상신되므로 제목은 `휴가신청서`로 시작하고, 취소 대상 날짜·일수 요약 끝에 `취소`를 명시한다(예: `휴가신청서 — 2026-06-10 종일 (1일) 취소`). 연차 사용 신청과 동일한 행 형식·아이콘·문서 제목으로 표시하되 요약에 `취소`로 사용/취소를 구분한다.
  - **결재문서 제목 표시 원칙**: 연차 사용·취소 모두 직원이 올린 결재문서 제목(`휴가신청서`)을 제목으로 노출하며, 신청/취소 구분은 제목 뒤 요약 텍스트로 처리한다. 연차 관리(P-002) 직원별 사이드 패널의 결재 이력 제목과 동일 기준이다.
- **메타(`alert-meta`)**: `{신청자명} · {부서} · {요청 일시}` 형식. 요청 일시는 신청 시각을 **`YYYY-MM-DD HH:MM`** 형식의 절대 일시로 표기한다(예: `2026-05-29 15:10`). 상대 시간(예: `2시간 전`)은 사용하지 않는다.
- **액션 버튼**:
  - 증명서 행 — `[발급]` 버튼. 클릭 시(행 전체 클릭 포함) **현재 대시보드 화면을 유지한 채** 발급처리 팝업(모달)을 그 자리에서 띄운다. P-003 업무신청관리로 화면을 전환하지 않으며, 팝업은 P-003의 발급처리 팝업과 동일한 모달을 공유한다.
  - 연차 행·연차 취소 행 — `[승인]` 버튼. 클릭 시 **경리나라 결재함을 새 창(새 탭)으로** 오픈. MY인사에서 직접 승인하지 않음.
- **표시 데이터 기준**:
  - 증명서 행 — 업무신청관리 처리 상태 `대기` 건(미처리 증명서 KPI와 동일 모집단).
  - 연차 행 — 로그인 관리자 본인이 결재 차례인 연차 사용/취소 결재 대기 건(승인할 연차 KPI와 동일 모집단).
  - 정렬: 증명서·연차 통합하여 신청 시각 최신순.
  - 각 유형 최대 **10건**까지 표시. 초과분은 표시하지 않으며 전체 목록은 해당 관리 페이지에서 조회.
- **행 구분선**: 각 행은 하단 1px 구분선을 가지며, 내역이 패널 높이를 초과해 **세로 스크롤이 생기면 마지막 행 하단선은 생략**한다(스크롤 없을 때만 마지막 행도 하단선으로 묶음을 닫음). 공통 규칙 P-000 3 '행 구분선' 참조.
- **빈 상태**: 미처리 업무가 0건이면 본문에 **"미처리 업무가 없습니다."** 표시. 이때도 패널 높이는 494 px로 고정.

### 3. 직원 현황 패널 (우측)

상단 **상태 탭** + 하단 **직원 테이블**로 구성된다.

- **상태 탭**: `전체` / `재직` / `휴직` / `퇴직` 4종. 각 탭 우측에 해당 상태 인원수를 표시(예: `전체 9`, `재직 8`). 기본 선택 탭은 **전체**이며 페이지 로드 시 전체가 선택된 상태로 진입. 탭 클릭 시 테이블이 해당 상태 직원으로 필터링된다.
- **직원 테이블 열(필드)**: `이름` / `부서` / `직급` / `입사일` / `연락처` / `상태`.
  - 모든 값은 경리나라 직원관리에 등록된 **일반근로자** 데이터에서 가져온다.
  - `입사일`은 `YYYY.MM.DD` 형식.
  - `상태`는 색상 뱃지로 표시: 재직(녹색) / 휴직(주황) / 퇴직(적색).
- **빈 상태**: 선택한 탭에 해당하는 직원이 없으면 테이블 영역에 빈 상태 안내를 표시한다.

### 4. 대시보드 2단 레이아웃 — 패널 폭·높이 요건 ★

대시보드 본문은 **좌측 미처리 업무 패널 + 우측 직원 현황 패널**의 2단 그리드로 구성된다. 우측 직원 현황 패널 폭은 다음 요건을 따른다.

- **직원 현황 패널 최소 폭 662 px**: 직원 테이블(`이름`·`부서`·`직급`·`입사일`·`연락처`·`상태` 6개 열)이 각 열의 내용이 잘리지 않는 자연 폭(합계 662 px)으로 모두 표시되도록, 우측 직원 현황 패널 폭은 **최소 662 px** 이상을 보장한다. 최소 해상도(가로 1280 px)에서도 이 최소 폭이 유지되어 `상태` 열 등에서 말줄임표가 발생하지 않는다.
- **넓은 화면에서의 확장**: 화면 폭에 여유가 있으면 우측 패널은 최소 662 px 아래로 줄지 않으면서 좌·우 패널이 기존 비율로 함께 넓어진다(우측 패널 고정 폭으로 좌측만 비대칭으로 커지지 않는다).
- **공통 텍스트 오버플로우 규칙과의 관계**: 위 최소 폭 보장은 P-000 공통의 텍스트 오버플로우(한 줄 말줄임 + 툴팁/상세) 규칙에 대한 대시보드 직원 현황 패널의 우선 적용 사항이다.
- **패널 높이 494 px 고정 + 내부 스크롤**: 좌측 미처리 업무 패널과 우측 직원 현황 패널은 각각 **높이 494 px로 고정**한다. 표시할 내역이 적어도 패널 높이는 494 px를 유지하며(하단이 줄어들지 않음), 내역이 많아 494 px를 초과하면 **패널 내부 스크롤**로 처리한다. 이때 패널 헤더(`미처리 업무 (N)` / `직원 현황`)와 직원 현황의 상태 탭 바는 스크롤되지 않고 고정되며, 본문(미처리 업무 행 목록 / 직원 테이블) 영역만 스크롤된다. 직원 테이블은 스크롤 시에도 헤더 행이 상단 고정(P-000 3. 고정 헤더)된다.
  - **494 px 산정 근거(가로 스크롤바 세로폭 보정)**: 직원 현황 테이블은 패널 폭이 좁아지면(예: 최소 해상도) 6개 열의 자연 폭 합계가 패널 폭을 넘어 **가로 스크롤바**가 생긴다. 가로 스크롤바는 본문 하단에서 세로 약 6 px를 차지하므로, 이 6 px가 본문 높이를 잠식해 **세로 스크롤바**까지 동시에 나타나는 것을 막기 위해 기준 높이(488 px)에 가로 스크롤바 세로폭 6 px를 더한 **494 px**를 패널 고정 높이로 한다. 좌·우 패널은 동일 높이(494 px)로 맞춰 하단 정렬을 유지한다.
