## 화면 상세 설계

### P-002 연차 관리

| 항목 | 내용 |
|------|------|
| **화면 목적** | 전 직원의 연차 현황을 월별로 조회하고, 결재 이력을 확인한다. 실제 승인·반려 처리는 경리나라 결재함에서 이루어진다. |
| **진입 조건** | 사이드바 [연차 관리] / 대시보드 KPI 카드 클릭 |
| **다음 화면** | 없음 (현재 페이지 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 툴바 | 연도 선택 | **기본값: 오늘 날짜 기준 현재 연도(yyyy).** 드롭다운 변경 시 해당 연도의 연차 현황으로 즉시 필터링. **과거·미래 연도 제한 없이** 자유롭게 연도를 선택하여 연차 현황 조회 및 관리 가능 |
| 상단 툴바 | 직원명 검색 | 최대 50자 / 1자 이상 입력 시 실시간 필터링 (300 ms 디바운스) / × 버튼으로 초기화 |
| 상단 툴바 | [일반근로자 불러오기] 버튼 | 경리나라 직원관리에 등록된 일반근로자 중 **조회 시점 재직 상태인 직원**만 불러옴. 일괄 등록 팝업 오픈. **현재 선택된 연도에만 저장되며, 다른 연도에는 영향을 주지 않음** |
| 상단 툴바 | [엑셀 다운로드] 버튼 | 현재 선택 연도 기준 화면에 표시된 목록 전체 다운로드. 포함 항목: 직원명 / 부서 / 총 연차 / 1월~12월 월별 사용 현황 / 잔여 연차 |
| 안내 문구 | 테이블 상단 | "총연차를 클릭하시면 연차를 등록할 수 있습니다" |
| 연차 현황 테이블 | 열 구성 | 직원명 / 부서 / 총 연차 / 사용 / 1월~12월 / 잔여 |
| 연차 현황 테이블 | 직원명 | 클릭 가능 → 우측 상세 패널 열림 |
| 연차 현황 테이블 | 총 연차 셀 | 더블클릭 시 직접 편집 가능 (**소수점 입력 허용**, 엔터/포커스 이탈 시 저장) |
| 연차 현황 테이블 | 월별 셀 | 마우스 오버 시 해당 월 사용·취소 이력 툴팁 표시 (취소 건 포함) |
| 우측 상세 패널 | 직원 기본 정보 + 연차 통계 | 직원명 클릭 시 슬라이드 오픈 |
| 우측 상세 패널 | 결재 이력 목록 | **현재 선택된 연도의 이력만 표시**. 연도 전환 시 해당 연도의 결재 이력으로 갱신 |

**사용자 흐름**
- 직원명 클릭 → 우측 상세 패널 오픈
- [일반근로자 불러오기] 클릭 → 경리나라 직원관리 일반근로자 중 조회 시점 재직 상태인 직원 목록 조회 → 직원 일괄 등록 팝업 오픈

**조건부 규칙**
- **좌측 메뉴 숫자 뱃지 없음**: 좌측 사이드바 [연차 관리] 메뉴에는 숫자 뱃지를 표시하지 않는다. (승인할 연차 현황은 대시보드 '승인할 연차' KPI 카드로 확인)
- 결재 이력 없는 직원 패널: "이력 없음" 안내 표시
- 직원 불러오기 모달 — 미등록 직원이 없을 경우: **"등록할 직원이 없습니다."** 표시
- **테이블 로딩 방식**: 직원 수가 많을 경우 무한 스크롤 방식으로 추가 로드. 페이지네이션 없음.
- **연도 기본값**: 화면 최초 진입 시 오늘 날짜 기준 현재 연도(yyyy)로 자동 설정
- **일반근로자 불러오기 대상**: 경리나라 직원관리 등록 일반근로자 중 **조회 시점 재직 상태**인 직원만 목록에 표시됨
- **일반근로자 불러오기 연도 범위**: 등록된 직원 데이터는 현재 선택된 연도에만 귀속됨. 연도를 전환해도 해당 연도 데이터에 영향 없음
- **엑셀 다운로드 범위**: 현재 선택 연도·적용 중인 직원명 필터 기준으로 화면에 표시된 목록과 동일한 데이터(직원명 / 부서 / 총 연차 / 1월~12월 월별 사용 현황 / 잔여 연차) 다운로드 (부서는 표시 컬럼으로 유지하되 필터 조건에서는 제외)

**데이터 동기화 규칙**
- 경리나라 **인사급여** 이용 고객사의 경우, 직원별 연차 데이터(총 연차·사용·잔여)는 인사급여의 연차와 자동 동기화된다.
- 인사급여 연동 고객사는 총 연차 셀 더블클릭 수동 편집이 비활성화된다 (인사급여 데이터가 원천).
- 인사급여 미사용 고객사는 총 연차 셀 더블클릭 수동 편집 방식을 그대로 유지한다.

**연차 집계·계산 규칙**
- 연차 관리 진입 시, 각 직원의 **사용 연차**와 **취소 연차**를 **월별로 집계**하여 테이블에 표시한다.
- 항목별 값은 다음 계산식에 따라 산출한다.
  - **월별 셀** = `해당 월 사용 일수 − 해당 월 취소 일수` (순값)
  - **사용** = `Σ(1~12월 월별 순값)` = `총 사용 − 총 취소`
  - **잔여** = `총 연차 − 사용` = `총 연차 − (사용 연차 − 취소 연차)`
- **집계 포함 대상**: 경리나라 결재함에서 **최종결재(접수)가 완료된 건만** 연차 계산에 포함한다.
  - 결재 흐름은 ①결재만으로 종료되는 경우와 ②접수까지 완료되어야 종료되는 경우가 있으며, **둘 다 "최종 완료" 시점에 도달한 건만** 집계한다.
  - **결재 대기 / 접수 대기 / 반려** 등 결재·접수가 완료되지 않은 건은 사용·취소·잔여 어디에도 반영하지 않는다.

**연차 차감 시점**
- 실제 승인·반려 처리는 경리나라 결재함 앱에서 이루어진다.
- MY인사는 경리나라 결재함에서 **최종결재(접수)가 완료된 시점**에 해당 직원의 연차를 차감(취소 건은 환원)한다.
- 결재 대기·접수 대기 상태(진행중)인 신청 건은 연차가 차감되지 않으며, 잔여 연차에서 제외되지 않는다.

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)

### 1. 상단 툴바

- **연도 선택**: 단일 선택 드롭다운. 기본값은 **오늘 날짜 기준 현재 연도(yyyy)**. 선택한 연도가 본 화면 전체(테이블·상세 패널 결재 현황·일반근로자 불러오기 귀속 연도)의 조회 기준이 된다. 과거·미래 연도 제한 없음.
- **직원명 검색**: 텍스트 입력(최대 50자). 1자 이상 입력 시 실시간 필터링(300ms 디바운스), × 버튼으로 초기화.

> **부서 필터 미제공**: 경리나라에는 부서를 공식 등록하는 메뉴가 없으므로 부서 필터(드롭다운)는 제공하지 않는다. 단, 연차 현황 테이블의 **부서 컬럼은 표시용으로 유지**한다.
- **안내 문구**: 고정 문구 `총연차를 클릭하시면 연차를 등록할 수 있습니다.`
- **[일반근로자 불러오기] 버튼**: 클릭 시 직원 일괄 등록 모달 오픈.
- **[엑셀 다운로드] 버튼**: 현재 선택 연도·직원명 필터 기준으로 화면 표시 목록과 동일한 데이터를 내려받음(포함 항목: 직원명 / 부서 / 총 연차 / 1~12월 월별 사용 / 잔여).

### 2. 연차 현황 테이블

각 행은 한 명의 등록 직원이며, 열(필드)은 좌→우로 다음과 같다. 표시 데이터는 **선택된 연도** 기준이다.

- **직원**(고정 열, sticky-left): 직원명. 텍스트 클릭 시 우측 상세 패널 오픈. 경리나라 직원관리 등록 일반근로자명.
- **부서**: 직원 소속 부서명.
- **총 연차**: 해당 연도 직원에게 부여된 총 연차 일수(소수점 허용, 예 `15`, `17`). 셀 더블클릭 시 인라인 편집(엔터/포커스 이탈 시 저장). 단, 인사급여 연동 고객사는 편집 비활성화(데이터 동기화 규칙 참조).
- **사용**: 해당 연도 누적 사용 일수(= 1~12월 월별 순값의 합 = `총 사용 − 총 취소`, 소수점 허용).
- **1월~12월**(12개 열): 각 월의 **순 사용 일수**(`해당 월 사용 − 해당 월 취소`). 미사용 월은 `0`. 반차·반반차는 `0.5`·`0.25`로 표기. 셀 마우스 오버 시 해당 월 사용·취소 이력 툴팁 표시.
  - **툴팁 사용 이력 형식**: `mm월 dd일 {연차종류}` (예: `01월 10일 연차`, `02월 20일 오전반차`). **일수 괄호(`(N일)`)는 표기하지 않는다.**
  - **연차기간(여러 날 연속) 표기**: 기간 연차는 **시작일+괄호 일수**로 줄이지 않고 **포함된 날짜를 모두 나열**한다(한 줄에 한 날짜씩). 예: 04월 10일~04월 11일 2일 연차 → `04월 10일 연차` / `04월 11일 연차`(2줄).
  - **툴팁 취소 이력 형식**: `mm월 dd일 {연차종류(연차/오전반차/오후반차)} 취소` (예: `01월 11일 연차 취소`). 일수 괄호 미표기. 취소 건은 해당 월 순값을 감소시킨다.
- **잔여**(고정 열, sticky-right): `총 연차 − 사용` 결과 일수(= `총 연차 − (사용 − 취소)`, 소수점 허용). **최종결재(접수) 완료로 차감/환원이 반영된 값만 집계**하며, 결재 대기·접수 대기(진행중)·반려 건은 잔여에 반영하지 않는다.
- **정렬·로딩**: 직원 수가 많으면 무한 스크롤로 추가 로드(페이지네이션 없음).
- **테이블 카드 높이 — 가용 세로 공간 전체 채움(패딩 제외) + 내부 스크롤**: 연차 현황 테이블 카드 높이는 **등록 직원(행) 수와 무관하게 콘텐츠 영역의 가용 세로 공간(하단 패딩 24 px만 남긴 높이)을 전부 채운다**. 행 수가 적어도 카드는 줄지 않고 하단까지 채우며(행 아래 빈 영역이 카드 안에 남음), 내역이 채운 높이를 초과하면 카드가 더 늘어나지 않고 **테이블 본문 영역의 내부 스크롤**로 처리하며, 이때 테이블 헤더 행은 상단 고정(P-000 3. 고정 헤더)된다. 동일 요건을 업무신청관리(P-003)·FAQ 관리(P-004) 목록 테이블에도 공통 적용한다. ※ 직원 업무신청(M/emp-cert) 목록은 종전 '내역 따라 유동 + 최대 높이 상한' 방식을 유지한다(관리자 목록과 별개).

### 3. 우측 상세 패널 — 연차 결재 현황 (직원명 클릭 시)

조회 전용 슬라이드 패널. 결재 처리 버튼은 없다(실제 승인·반려는 결재함 앱에서 수행).

- **헤더 제목**: `{직원명} · 연차 결재 현황`.
- **프로필**: 직원명 + 부서명.
- **연차 통계(3개)**: `총 연차` / `사용` / `잔여` — 테이블 해당 행과 동일 값. **선택된 연도** 기준.
- **결재 현황 목록**: 선택된 연도의 결재 건만 표시. 각 항목 필드:
  - **제목**: **직원이 경리나라 전자결재에 상신한 결재문서 제목**을 그대로 표기한다. 연차 사용 신청·연차 취소 신청 모두 동일하게 **`휴가신청서`** 문서로 상신되므로 제목은 항상 `휴가신청서`이며, 신청/취소 구분은 아래 보조(요약) 텍스트로 처리한다. <br>※ 결재 문서명 `휴가신청서`는 경리나라 전자결재의 실제 문서명이므로 화면에 그대로 표기한다(연차 관리 화면 내 그 외 '휴가' 표현은 '연차'로 통일).
  - **보조(요약·상신일)**: 제목 아래 한 줄로 `{신청 요약} · {상신일}`을 표기한다. 신청 요약은 신청 일자·종류·일수(예: `2026.05.04 (월) — 종일 1일`, `2026.05.06~05.08 — 3일`, 반차는 `오전 반차`/`오후 반차`)이며, **연차 취소 건은 요약 끝에 `… 취소`**(예: `2026.04.22 (수) — 1일 취소`)로 사용/취소를 식별한다. 상신일은 `M월 D일 상신` (예: 사용 `… — 종일 1일 · 4월 29일 상신`, 취소 `… — 1일 취소 · 4월 21일 상신`).
  - **상태 뱃지**: `대기`(주황) 또는 `승인`(녹색). 결재함의 결재 진행 상태를 반영.
- **빈 상태**: 선택 연도에 결재 건이 없으면 **"결재 이력이 없습니다"** 표시.
- **목록 스크롤**: 헤더·프로필·연차 통계(3개)는 패널 상단에 **고정**되고, 결재 현황 **목록 영역**만 가용 세로 공간을 채운다. 결재 건이 많아 영역을 넘으면 **목록 영역 내부에서만 세로 스크롤**이 발생하며(상단 고정 영역은 함께 밀리지 않음), 건수가 적으면 스크롤 없이 표시한다.

### 4. 일반근로자 불러오기 모달

경리나라 직원관리의 일반근로자 중 **연차 관리 미등록 + 조회 시점 재직 상태**인 직원을 일괄 등록한다. 등록 결과는 현재 선택된 연도에만 귀속된다.

- **요약 배너**: `{yyyy}년 미등록된 경리나라 일반근로자 {N}명이 확인됩니다` + `재직 중` 뱃지. **{yyyy}는 상단 툴바에서 현재 선택된 연도**를 그대로 반영한다(연도 변경 후 모달을 열면 해당 연도로 표시).
- **직원 카드 목록**: 각 카드 필드 = 아바타(이름 첫 글자) / 직원명 / `{부서} · {입사일} 입사`(입사일 `YYYY.MM.DD`) / [등록] 버튼(개별 등록).
- **하단 버튼**: `[닫기]` / `[전체 일괄 등록]`.
- **빈 상태**: 미등록 직원이 없을 경우 **"등록할 직원이 없습니다."** 표시.
