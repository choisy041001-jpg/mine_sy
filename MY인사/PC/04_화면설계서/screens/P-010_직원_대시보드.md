## 화면 상세 설계

### P-010 직원 대시보드

| 항목 | 내용 |
|------|------|
| **화면 목적** | 직원이 잔여 연차·사용 이력을 확인하고, 증명서 신청으로 빠르게 이동한다. |
| **진입 조건** | 직원 계정으로 로그인 시 기본 화면 / 사이드바 [대시보드] |
| **다음 화면** | 없음 (현재 페이지 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 잔여 연차 카드 | 잔여 일수(크게) + 총·사용 일수 + 진행 막대 | |
| 앱 설치 QR 카드 | 모바일 앱 설치 안내 | 설치 여부와 무관하게 **상시 표시** |
| 연차 사용 이력 테이블 | 날짜 / 구분 / 일수 / 사유 / 처리 상태 | **현재 연도(오늘 날짜 기준 yyyy) 이력만 표시**. 연차 사용일자 기준 **최신순** 정렬. **연차 사용 건과 연차 취소 요청 건을 함께 표시**(구분 열로 식별) |
| 증명서 관리 패널 | 최근 신청 내역 + [증명서 신청] 버튼 | 최근 신청 내역 최대 **5건** 표시. [증명서 신청] 클릭 시 직원 업무신청(P-011) 화면으로 이동 |

**조건부 규칙**
- 연차 신청은 모바일 MY인사 앱에서만 가능. PC 직원 화면에서는 연차 이력 조회만 지원.
- 연차 사용 이력이 없을 경우: **"내역이 없습니다."** 표시
- 증명서 신청 이력이 없을 경우: **"내역이 없습니다."** 표시

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)
> **데이터 출처**: 연차 데이터는 **경리나라 결재함 집계 결과**(P-002 연차 관리와 동일 기준 — 최종결재/접수 완료 건만 반영)에서 **로그인한 직원 본인 1인**의 값을 가져온다. 증명서 데이터는 **직원 본인의 증명서 신청 이력**이다. 본 화면은 **조회 전용**이며, 연차 신청·취소는 모바일 MY인사 앱에서만 처리한다.

### 1. 잔여 연차 카드

- **잔여 일수(대형 숫자)**: `총 연차 − 사용` 결과(소수점 허용). 단위 `일`을 우측에 표기. **본인 계정·현재 연도(오늘 날짜 기준 yyyy)** 기준.
- **총/사용 칩**: `총 {N}` / `사용 {N}` 칩 2종. 각 숫자는 accent 강조.
- **진행 막대**: 채움 비율 = `사용 ÷ 총 연차`(%). 예) 사용 4 / 총 15 → 26.7%.
- **막대 하단 라벨**: 좌측 `사용 {N}일`, 우측 `잔여 {N}일 / 총 {N}일`.
- **집계 기준**: P-002와 동일하게 **최종결재(접수) 완료로 차감/환원이 반영된 값만** 집계한다(결재 대기·접수 대기·반려 건 미반영).

### 2. 앱 설치 QR 카드

- **QR 코드**: 모바일 앱 설치 URL을 인코딩한 이미지. URL = `my-insa.app/install` *(예시 URL — 실제 서비스 도메인 확정 후 교체)*.
- **카피**: 제목 "모바일 앱 설치" + 안내 "QR 스캔으로 언제 어디서나 연차·증명서 신청".
- **설치 URL 칩**: `my-insa.app/install` 텍스트 칩 표시.
- **표시 조건**: 설치 여부와 무관하게 **상시 표시**(안내 전용). 본 카드에는 링크 복사·QR 다운로드 등 액션 버튼을 두지 않는다(P-006 모바일 안내 화면과 구분).

### 3. 연차 사용 이력 테이블

각 행은 본인의 연차 사용 건 1건이다. **현재 연도(yyyy)** 이력만, **사용일자 기준 최신순**으로 정렬한다. 열(필드)은 좌→우로 다음과 같다.

- **날짜**: 연차 사용일. `YYYY.MM.DD (요일)` 형식(예: `2026.05.04 (월)`).
- **구분**: `연차` / `오전 반차` / `오후 반차` / `연차 취소`. **`연차 취소`는 직원 본인이 모바일 MY인사 앱에서 제출한 연차 취소 요청 건**으로, 연차 사용 이력과 동일 테이블에 함께 노출되며 구분 열로 식별한다.
- **일수**: 사용(또는 취소) 일수. 종일 `1일`, 반차 `0.5일`. accent 굵게·중앙 정렬.
- **사유**: 신청 시 입력한 사유 텍스트(연차 취소 요청은 취소 사유).
- **상태**: 뱃지. `대기`(주황, `badge w`) / `승인`(녹색, `badge s`). 결재함의 진행 상태를 반영. 연차 취소 요청 건도 동일하게 `대기`/`승인`으로 표시한다.
- **빈 상태**: 현재 연도 이력이 없으면 **"내역이 없습니다."** 표시.

### 4. 증명서 관리 패널

- **[증명서 신청] 버튼**: 클릭 시 **직원 업무신청(P-011, `emp-cert`) 화면으로 이동**한다. *(증명서 종류 선택·용도 입력 2단계 신청 모달은 P-011에서 동작하며, 본 화면에서 모달을 열지 않는다.)*
- **최근 신청 내역**: 본인 증명서 신청 이력 중 **최신 최대 5건**. 각 행 필드:
  - **증명서 종류**: `재직증명서` / `경력증명서` / `기타증명서`.
  - **메타**: `{신청일} · {용도}` (신청일 `YYYY.MM.DD`).
  - **상태 뱃지**: `대기`(주황, `badge w`) / `발급완료`(녹색, `badge s`) / `반려`(빨강, `badge d`).
- **빈 상태**: 신청 이력이 없으면 **"내역이 없습니다."** 표시.

### 5. 대시보드 패널 높이 — 494 px 고정 + 내부 스크롤 ★

연차 사용 이력 패널과 증명서 관리 패널은 각각 **높이 494 px로 고정**한다.

- **고정 높이**: 표시할 내역이 적어도 패널 높이는 494 px를 유지하며 하단이 줄어들지 않는다.
- **내부 스크롤**: 내역이 많아 494 px를 초과하면 **패널 내부 스크롤**로 처리한다. 패널 헤더(`연차 사용 이력` / `증명서 관리` + [증명서 신청] 버튼)는 스크롤되지 않고 고정되며, 본문(연차 이력 테이블 / 증명서 신청 내역) 영역만 스크롤된다. 연차 이력 테이블은 스크롤 시에도 헤더 행이 상단 고정(P-000 3. 고정 헤더)된다.
- **관리자 대시보드와의 정합**: 본 요건은 P-001 관리자 대시보드 패널(미처리 업무·직원 현황)의 494 px 고정·내부 스크롤 요건과 동일 기준이다. 494 px는 기준 높이 488 px에 가로 스크롤바 세로폭(약 6 px)을 더한 값으로, 관리자 대시보드 직원 현황 패널과 높이를 일치시켜 동일 기준을 유지한다(산정 근거는 P-001 세부 설계 요건 4 참조).
