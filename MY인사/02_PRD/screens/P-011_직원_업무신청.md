## 데이터 표시 기준

**① 데이터 출처**
- 이력 테이블은 **직원 본인의 증명서 신청 내역**이다. 신청 입력값(종류·제출처/용도·요청 메모·기타증명서 명칭)은 관리자 업무신청관리(P-003)로 전달되어 발급 처리된다.
- **발급 매수(부수)는 신청 항목이 아니다** — 신청 모달·이력 테이블에 부수 입력·컬럼을 두지 않는다.

**② 이력 테이블**
- 열: 신청일(`YYYY.MM.DD HH:MM`) / 증명서 종류(뱃지) / 제출처·용도 / 처리 상태 / 액션. **신청일 최신순**.
- 처리 상태 뱃지: `대기`(주황) / `발급완료`(녹색) / `반려`(빨강).
- 액션: 대기 → [신청 취소](확인 후 처리), 발급완료 → [다운로드](PDF), 반려 → [사유보기](사유 팝업).
- 빈 상태: **"내역이 없습니다."**

**③ 증명서 신청 모달(2단계)**
- Step 1: 종류 선택 버튼 3종(설명 포함). Step 2: 기타증명서 명칭(기타 시 필수·최대 50자) / 제출처·용도(필수·최대 100자) / 요청 메모(선택).
- 신청하기 시 필수값 검증 후 완료 토스트(`✅ {종류} 신청이 완료되었습니다. 인사담당자 확인 후 발급됩니다.`).

**④ 반려 사유 팝업**
- 관리자가 입력한 반려 사유를 읽기 전용으로 표시. [확인] 버튼으로 닫기.
