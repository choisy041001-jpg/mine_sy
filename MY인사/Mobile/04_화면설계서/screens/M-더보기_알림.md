### M-006 인사 FAQ 목록

> 인사 FAQ는 **인사담당자가 관리자 PC(P-004 FAQ 관리)에서 등록한 '자주 묻는 질문과 답변'을 직원이 모바일에서 읽기 전용으로 열람**하는 기능이다. **별도 상세 화면으로 이동하지 않고, 목록에서 질문 행을 탭하면 같은 자리에서 답변이 인라인 아코디언으로 펼쳐지는** 단일 목록형 채널이며, '인사 FAQ' 채널에서는 직원의 글 작성(글쓰기 FAB)이 노출되지 않는다. 더보기(M-040)의 하위 메뉴로 진입한다.
> **구현 함수**: 더보기 [인사 FAQ] → `goPostList('인사 FAQ', true)`(`#faqPostList` 표시, FAB 숨김, 하단 탭은 더보기 유지) / 질문 행 탭 → `toggleFaqAcc(headerEl, id)`(해당 행 하단 패널에 답변 본문을 펼침/접힘, 한 번에 1개만 펼침). 답변 본문은 게시글 상세 데이터(`_POST_DETAIL[id].body`)를 단일 출처로 재사용한다.

| 항목 | 내용 |
|------|------|
| **화면 목적** | 인사담당자가 관리자 PC(P-004 FAQ 관리)에서 등록한 '자주 묻는 질문과 답변'을 단일 목록에서 확인하고, 질문을 탭해 답변을 목록 내 인라인으로 펼쳐 읽는다 (카테고리 구분 없음). FAQ 등록·수정·삭제는 관리자 PC(P-004)에서만 가능하며, 직원은 읽기 전용으로 조회만 한다. |
| **진입 조건** | 더보기(M-040) → [인사 FAQ] 메뉴 탭 (`goPostList('인사 FAQ', true)`) |
| **다음 화면** | 없음 (목록 내 인라인 펼침 — 별도 상세 화면 없음) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "인사 FAQ" | 게시판 목록(M-003) 상단바 재사용, 채널명 = 인사 FAQ |
| FAQ 질문 행 | 질문 제목 + 펼침 화살표(▾) | 질문 제목(공지 핀 없음) + 행 우측에 펼침 상태 화살표(접힘 ▾ / 펼침 시 180° 회전). 탭 시 답변 펼침/접힘 |
| FAQ 질문 행 | 메타 | `{작성자} · {게시일 YYYY.MM.DD}` (작성자 = 인사담당자 성명, 게시 날짜는 `YYYY.MM.DD`, **조회수·댓글 수 미표시**) |
| 답변 펼침 영역 | 인라인 아코디언 패널 | 질문 행 바로 하단에 펼쳐지는 영역. 인사담당자 답변 본문(HTML — 단락·강조)을 표시. **댓글·댓글 입력창 없음**(읽기 전용 Q&A). 자세한 정의는 **M-008** 참조 |
| 글쓰기 FAB | (숨김) | 인사 FAQ 채널에서는 직원 글 작성 불가 — FAB 미노출 |

**사용자 흐름**
- FAQ 질문 행 탭 → **같은 자리에서 답변 아코디언 펼침**(화면 이동 없음). 다른 질문 행 탭 → 이전 항목 접힘 + 새 항목 펼침(한 번에 1개)
- 펼쳐진 질문 행을 다시 탭 → 답변 접힘

**조건부 규칙**
- 인사 FAQ 글은 인사담당자(관리자)가 PC(P-004)에서 직접 등록·관리. 직원은 조회 전용(새 글 작성 불가)
- 글쓰기 FAB는 인사 FAQ 채널에서 항상 숨김
- 목록이 없을 경우: 빈 상태 안내 표시
- 목록 화면 재진입 시 모든 항목은 접힌(닫힌) 상태로 초기화

### M-008 인사 FAQ 답변 (인라인 펼침 영역)

> M-008은 **독립된 상세 화면이 아니라 M-006 목록 안에서 질문 행을 탭하면 펼쳐지는 인라인 답변 영역**이다. (이전: 게시글 상세 `post-detail` 재사용 → **현재: 목록 내 아코디언 펼침으로 변경**, 별도 화면 이동·댓글 없음, 읽기 전용)

| 항목 | 내용 |
|------|------|
| **화면 목적** | 선택한 FAQ 질문의 인사담당자 답변(본문)을 목록 내에서 인라인으로 펼쳐 확인한다. |
| **진입 조건** | 인사 FAQ 목록(M-006)에서 질문 행 탭 |
| **다음 화면** | 없음 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 답변 본문 | 인라인 패널 | 질문 행 하단에 펼쳐지는 답변 영역. 본문(HTML — 답변을 단락·강조로 구성)을 표시. 답변 본문은 게시글 상세 데이터(`_POST_DETAIL[id].body`)를 재사용 |

**조건부 규칙**
- FAQ 답변 본문은 인사담당자가 PC(P-004)에서 등록한 콘텐츠이며 직원은 수정·삭제할 수 없음(읽기 전용)
- **댓글·댓글 입력창 없음** — 추가 문의 기능은 제공하지 않는다 (별도 상세 화면을 사용하지 않으므로 제거)
- 조회수·댓글 수 미표시

| 알림 행 | 유형 아이콘 | 알림 유형별 색상 아이콘 (아래 알림 유형 정의 참조) |

| 알림 행 | 수신 시각 | 작은 텍스트(11px) / **모든 알림 동일하게 `YYYY.MM.DD HH:mm` 절대 시각으로 표시**(상대 시각 "N분 전·N시간 전" 미사용) |
| 빈 상태 | 안내 문구 | 수신된 알림이 없을 때 "받은 알림이 없습니다" 표시 |

**알림 유형 정의**

| 알림 유형 | 아이콘 색상 | 발생 조건 | 탭 시 이동 화면 |
|---------|----------|---------|--------------|
| 연차 승인 | 초록(success) | 결재자가 휴가신청서를 승인 처리 시 | **M-010 (연차 관리)** |
| 연차 반려 | 빨강(danger) | 결재자가 신청서를 반려 처리 시 | **M-010 (연차 관리)** |
| 증명서 발급 완료 | 파랑(accent) | 인사담당자가 증명서 발급 처리 완료 시 | **M-030 (증명서 관리)** |
| 증명서 반려 | 빨강(danger) | 인사담당자가 증명서 신청을 반려 시 | **M-030 (증명서 관리)** |
| 새 공지사항 | 파랑(accent, 연한 배경) | 관리자가 게시판에 공지글 등록 시 | M-002 (게시판 홈) |
| 결의서 결재 완료 | 초록(success) | 전자결재 ON 상태에서 결재선에 따른 결재가 최종 승인 완료 시 | M-020 (영수증 관리) |
| 결의서 결재 반려 | 빨강(danger) | 전자결재 ON 상태에서 결재 중 반려 처리 시 | M-020 (영수증 관리) |

> **알림 탭 시 이동 화면은 개별 상세 화면이 아니라 해당 기능의 메인 화면으로 이동한다**(프로토타입 `handleNotifClick` 정본): 연차 승인·반려 → 연차 관리(M-010, `leave-approval`), 증명서 발급 완료·반려 → 증명서 관리(M-030, `cert-types`), 결의서 결재 완료·반려 → 영수증 관리(M-020, `expense-hub`), 새 공지사항 → 게시판 홈(M-002, `board`). 직원은 이동한 메인 화면에서 해당 건을 찾아 상세를 확인한다.

**기기 알림 문구**

**기기 알림(디바이스 푸시) 탭 → 앱 진입 분기**

> 기기 알림에는 **수신 대상(사용자 + 사업장)** 정보와 **딥링크 목적지(알림 메뉴 M-050)** 가 포함된다. 알림 탭 시 앱 진입·로그인 처리는 **자동로그인 설정 여부**와 **'알림의 사용자/사업장' == '앱에 로그인되는 사용자/사업장' 일치 여부**에 따라 분기한다. **알림 메뉴(M-050)로의 자동 이동은 두 정보가 일치할 때만** 수행하며, 불일치 시에는 알림 메뉴로 이동하지 않고 로그인 첫 화면인 게시판 홈(M-002)을 유지한다. (로그인 흐름 연동 → **M-001** 참조)

| # | 앱 상태 / 조건 | 앱 진입 처리 | 이동 화면 |
|---|------|------|------|
| 1 | 자동로그인 설정됨 + **알림 사업장 = 자동로그인 사업장** | 해당 계정/사업장으로 **자동 로그인** | **M-050 알림 메뉴로 이동** |
| 2 | 자동로그인 미설정 → 로그인 화면에서 **알림과 동일한 사용자/사업장으로 로그인** | 앱 실행 → 로그인(M-001) | **M-050 알림 메뉴로 이동** |
| 3 | 자동로그인 미설정 → 로그인 화면에서 **알림과 다른 계정/사업장으로 로그인** | 앱 실행 → 로그인(M-001) | 게시판 홈(M-002) 유지 — **알림 메뉴 이동 안 함** |
| 4 | **이미 로그인된 상태**에서 **다른 계정/사업장**의 알림 탭 | **강제 로그아웃 → 로그인 화면(M-001)으로 전환** | (재로그인 결과에 따름 ↓) |
| 4-1 | └ (4에서) 알림과 **동일한** 사용자/사업장으로 재로그인 | 로그인(M-001) | **M-050 알림 메뉴로 이동** |
| 4-2 | └ (4에서) 알림과 **다른** 사용자/사업장으로 재로그인 | 로그인(M-001) | 게시판 홈(M-002) 유지 — 알림 메뉴 이동 안 함 |

- (참고) 이미 로그인된 상태에서 **동일 계정/사업장**의 알림을 탭한 경우: 로그아웃 없이 곧바로 **M-050 알림 메뉴로 이동**한다(4번은 계정/사업장이 다를 때만 강제 로그아웃).
- 딥링크 목적지는 개별 상세 화면이 아니라 **알림 메뉴(M-050)** 이며, 알림 메뉴 진입 이후 각 알림 행 탭 시의 화면 이동은 위 "알림 유형 정의"를 따른다.

**사용자 흐름**

1. 알림 행 탭 → 해당 알림의 읽음 상태로 전환 → 알림 유형에 따라 대상 화면으로 이동
2. [모두 읽음] 탭 → 목록 내 모든 알림 읽음 처리 → 뱃지 소멸 → 토스트 표시 → 화면 유지
3. [← 뒤로] 탭 → 게시판 홈(M-002)으로 복귀

**조건부 규칙**
- 알림 목록이 없을 때: 빈 상태 안내 표시
- 모든 알림이 이미 읽음 상태일 때: [모두 읽음] 버튼 비활성화 또는 탭 시 동작 없음
- "푸시 알림 받기" 토글(M-051)은 **디바이스(OS) 푸시 발송 여부만** 제어한다. OFF여도 **앱 내 알림 목록에는 알림이 계속 적재**되며, ON일 때만 디바이스 푸시가 전송된다
- 방해금지 모드 시간대에는 알림이 수신되지 않으며, 해당 시간 이후에 일괄 표시됨
- 알림 항목 탭 시 이동 대상 화면이 존재하지 않을 경우(삭제된 문서 등): 토스트 "해당 내역을 찾을 수 없습니다" 표시 후 알림 목록 유지

| 섹션 3 | 안내 문구 | "매일 반복 적용됩니다" |

**사용자 흐름**

1. **푸시 알림 받기 토글 OFF**: 항목별 설정 섹션 비활성(dimmed). **디바이스(OS) 푸시 발송만 즉시 중단되며, 앱 내 알림 목록(M-050)에는 알림이 계속 쌓인다**(방해금지 모드와 동일하게 앱 내 적재는 유지)
2. **푸시 알림 받기 토글 ON**: 항목별 설정 섹션 활성화, 각 항목 토글에 따라 수신 여부 개별 제어
3. **방해금지 모드 토글 ON**: 시간 설정 영역 펼쳐짐 → 시작/종료 시간 각각 선택 → 설정 즉시 저장
4. **방해금지 모드 토글 OFF**: 시간 설정 영역 접힘 → 방해금지 해제
5. 설정 변경은 별도 [저장] 버튼 없이 토글·선택 즉시 서버에 반영(Auto-save)

**조건부 규칙**
- 푸시 알림 받기가 OFF인 경우: 항목별 알림 토글과 방해금지 모드 토글 모두 비활성화(dimmed) 처리. 탭해도 상태 변경 불가
- 방해금지 시작 시간과 종료 시간이 동일한 경우: "시작 시간과 종료 시간이 같을 수 없습니다" 오류 안내 + 설정 미저장
- 방해금지 모드 적용 중에는 알림(M-050) 목록에 알림이 쌓이며, 해제 후 수신된 것으로 처리됨
- 기기 자체의 알림 권한이 거부된 경우: 화면 상단에 "기기 알림 권한이 꺼져 있습니다. 기기 설정에서 허용해 주세요" 배너 표시

- **FAQ(질문+답변)**: 인사담당자가 관리자 PC **FAQ 관리(P-004)** 에서 등록한 콘텐츠가 그룹웨어를 통해 모바일 `인사 FAQ` 채널로 노출된다. 직원은 등록·수정·삭제 권한이 없고 **읽기 전용**으로 열람한다.
- **작성자**: 인사 FAQ의 작성자는 등록한 **인사담당자 성명**(예: 한관리자)으로 표시된다. 부서명('인사팀')이 아닌 담당자 개인 성명으로 노출한다.
- **공지 핀**: 인사 FAQ 목록에는 **`공지` 핀 뱃지를 사용하지 않는다**(모든 FAQ 동일 형식 표시).
- **카테고리**: 인사 FAQ는 **카테고리 구분이 없는 단일 목록**으로 관리한다(카테고리 필터·뱃지 없음).
- **조회수·댓글 수**: 목록·답변 어디에도 **조회수·댓글 수를 표시하지 않는다**.
- **M-006 목록 + M-008 답변(인라인 아코디언)**: 더보기(M-040) [인사 FAQ] → `goPostList('인사 FAQ', true)` → 읽기 전용 목록(`#faqPostList`). 더보기 탭 강조 유지(하단 탭은 더보기). 질문 행 = 질문 제목(길면 말줄임) + 우측 펼침 화살표(▾) + 메타 `{작성자=인사담당자 성명} · {게시일 YYYY.MM.DD}`. 공지 핀·조회수·댓글 수 미표시. 글쓰기 FAB는 항상 숨김(`isFaqBoard=true`). 등록일 최신순. 빈 상태 안내.
  - **질문 행 탭 → `toggleFaqAcc(headerEl, id)`**: 별도 상세 화면으로 이동하지 않고 **질문 행 바로 하단 패널에 답변 본문을 인라인으로 펼침/접힘**(한 번에 1개만 펼침, 화살표 180° 회전). 답변 본문(HTML — 단락·강조)은 게시글 상세 데이터(`_POST_DETAIL[id].body`)를 단일 출처로 재사용하며 최초 펼침 시 1회 주입한다.
  - **댓글 없음**: 답변 펼침 영역에는 댓글 목록·댓글 입력창을 두지 않는다(읽기 전용 Q&A). 직원의 질문 작성·댓글 작성 기능 없음.
  - **재진입 초기화**: 목록 화면 재진입 시 모든 항목은 접힌(닫힌) 상태로 초기화한다.

### M-050 알림
- **수신 시각 형식**: **모든 알림 동일하게 `YYYY.MM.DD HH:mm`**(예: `2026.06.05 13:20`). 상대 시각("방금 전·N분 전·N시간 전")은 사용하지 않는다.
- **날짜 섹션 헤더**: "오늘" / "어제" / "M월 D일 (요일)"(예: `5월 18일 (일)`).
- **미읽음 표시**: 연파랑(#fafcff) 배경 + 우측 파란 점(●). [모두 읽음] 탭 시 전체 읽음 처리·뱃지 소멸·토스트 "모든 알림을 읽음 처리했습니다".
- **게시판 알림 아이콘 뱃지(●)**: 미읽음 알림이 1건 이상일 때만 게시판 홈 우측 상단 🔔 아이콘에 빨간 점(●)을 표시한다(**M-000 §4** 미읽음 표시 규칙). **미읽음이 0건이면 빨간 점을 표시하지 않는다** — 개별/모두 읽음 처리로 미읽음이 0이 되거나, 초기 진입 시 미읽음이 없으면 뱃지 미노출(구현: `updateNotifBadge()`가 미읽음 유무로 동기화).
- **탭 시 이동**: 개별 상세 화면 직접 딥링크가 아니라 **해당 기능 메인 화면**으로 이동(연차 관리 M-010 / 증명서 관리 M-030 / 영수증 관리 M-020 / 게시판 홈 M-002).
- **기기 알림(디바이스 푸시) 탭 → 앱 진입 딥링크**: 기기 알림에 담긴 **수신 대상(사용자+사업장)** 과 **앱에 로그인되는 사용자/사업장의 일치 여부 + 자동로그인 설정**에 따라 진입을 분기한다. 일치 시 알림 메뉴(M-050)로 이동, 불일치 시 게시판 홈(M-002) 유지. 이미 로그인된 상태에서 **다른 계정/사업장** 알림 탭 시 강제 로그아웃 후 로그인 화면(M-001)으로 전환한다. (상세 분기표 → 위 M-050 "기기 알림(디바이스 푸시) 탭 → 앱 진입 분기", 로그인 흐름 → M-001)
- **유형별 본문 예시(프로토타입 정적 데이터)**: 증명서 발급 완료 "재직증명서 발급이 완료되었습니다. 확인해 보세요." / 연차 승인 "5월 4일(월) 휴가신청서가 승인되었습니다." / 증명서 반려 "기타증명서 신청이 반려되었습니다. 사유를 확인해 주세요." / 새 공지사항 "2026년 5월 정기 회의 일정 및 안건이 게시되었습니다." / 결의서 결재 완료 "제출하신 4월 거래처 미팅 비용 결의서가 최종 승인되었습니다." / 결의서 결재 반려 "제출하신 결의서가 반려되었습니다. 사유를 확인해 주세요."

### M-051 알림 설정
- **토글 라벨 / 기본값 / 설명**: 푸시 알림 받기(ON · "그룹웨어 앱의 모든 푸시 알림을 받습니다") / 연차 / 휴가(ON · "신청·승인·반려 알림") / 증명서(ON · "발급완료·반려 알림") / **결의서**(ON · "전자결재 완료·반려 알림" — 영수증 등록 자체는 알림 미발송) / 게시판 / 공지(ON · "새 공지사항 알림") / 방해금지 모드(OFF · "설정한 시간 동안 알림을 받지 않습니다").
- **방해금지 시간 기본값**: 시작 `22:00` / 종료 `07:00`, "매일 반복 적용됩니다". 시·분 선택 옵션은 시작 시 22·23·00·21·20 / 종료 시 07·08·09·06 / 분 공통 00·30.
- **저장 방식**: 별도 [저장] 버튼 없이 토글·시간 선택 즉시 반영(Auto-save). 푸시 알림 받기 OFF 시 항목별 토글·방해금지 모드 영역 dimmed(탭 불가).
- **단말기 간 설정 공유(계정 기준)**: 알림 설정(푸시 알림 받기·항목별 토글·방해금지 모드 시간)은 **단말기가 아니라 계정에 귀속**된다. 하나의 계정으로 여러 단말기(기기)에 로그인한 경우 한 기기에서 변경한 설정이 **모든 기기에 동일하게 적용**되며(서버 동기화), 기기별로 따로 설정값을 갖지 않는다.
