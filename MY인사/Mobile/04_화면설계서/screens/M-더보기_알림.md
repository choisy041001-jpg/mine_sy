### M-050 알림

| 항목 | 내용 |
|------|------|
| **화면 목적** | 앱에서 수신한 푸시 알림 내역을 날짜 순서로 확인하고, 알림 항목을 탭하여 관련 화면으로 바로 이동한다. |
| **진입 조건** | 게시판 홈(M-002) 상단 알림 아이콘(🔔) 탭 |
| **다음 화면** | 알림 유형에 따라 해당 화면으로 분기 (아래 조건부 규칙 참조) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] 버튼 + "알림" 타이틀 | |
| 상단 내비게이션 | [모두 읽음] 버튼 | 우측 정렬. 탭 시 전체 알림 읽음 처리 + 뱃지 소멸 + 토스트 "모든 알림을 읽음 처리했습니다" |
| 날짜 섹션 헤더 | 날짜 라벨 | "오늘" / "어제" / "M월 D일 (요일)"(예: `5월 18일 (일)`) 형식으로 날짜별 그룹화 |
| 알림 행 (미읽음) | 행 배경 | 연파랑(#fafcff) 배경 + 우측 파란 점(●) 표시로 미읽음 구분 |
| 알림 행 (읽음) | 행 배경 | 기본 흰색 배경, 파란 점 없음 |
| 알림 행 | 유형 아이콘 | 알림 유형별 색상 아이콘 (아래 알림 유형 정의 참조) |
| 알림 행 | 알림 제목 | 굵은 텍스트, 13px |
| 알림 행 | 알림 내용 | 보조 텍스트, 12px, 최대 2줄 말줄임 |
| 알림 행 | 수신 시각 | 작은 텍스트(11px) / 당일은 "N분 전·N시간 전", 전날 이전은 "YYYY.MM.DD HH:mm" |
| 빈 상태 | 안내 문구 | 수신된 알림이 없을 때 "받은 알림이 없습니다" 표시 |

**알림 유형 정의**

| 알림 유형 | 아이콘 색상 | 발생 조건 | 탭 시 이동 화면 |
|---------|----------|---------|--------------|
| 연차 승인 | 초록(success) | 결재자가 휴가신청서를 승인 처리 시 | **M-010 (연차 관리)** |
| 연차 반려 | 빨강(danger) | 결재자가 신청서를 반려 처리 시 | **M-010 (연차 관리)** |
| 증명서 발급 완료 | 파랑(accent) | 인사담당자가 증명서 발급 처리 완료 시 | **M-030 (증명서 관리)** |
| 증명서 반려 | 빨강(danger) | 인사담당자가 증명서 신청을 반려 시 | **M-030 (증명서 관리)** |
| 새 공지사항 | 파랑(accent, 연한 배경) | 관리자가 게시판에 공지글 등록 시 | M-002 (게시판 홈) |
| 결의서 결재 완료 | 초록(success) | 전자결재 ON 상태에서 결재선에 따른 결재가 최종 승인 완료 시 | M-020 (영수증 관리) |
| 결의서 결재 반려 | 빨강(danger) | 전자결재 ON 상태에서 결재 중 반려 처리 시 | M-020 (영수증 관리) |

> **알림 탭 시 이동 화면은 개별 상세 화면이 아니라 해당 기능의 메인 화면으로 이동한다**(프로토타입 `handleNotifClick` 정본): 연차 승인·반려 → 연차 관리(M-010, `leave-approval`), 증명서 발급 완료·반려 → 증명서 관리(M-030, `cert-types`), 결의서 결재 완료·반려 → 영수증 관리(M-020, `expense-hub`), 새 공지사항 → 게시판 홈(M-002, `board`). 직원은 이동한 메인 화면에서 해당 건을 찾아 상세를 확인한다.

**기기 알림 문구**

OS 알림 센터·잠금화면·인앱 배너에 실제로 표시되는 제목(title)과 본문(body). 앱 이름 "MY인사"는 시스템이 자동 표시하며 개발 제어 범위 밖이다.

| 알림 유형 | 제목 (title) | 본문 (body) | 표시 예시 |
|---------|------------|-----------|---------|
| 연차 신청 승인 | 휴가신청서 결재가 승인되었습니다 | `{신청일자} {휴가종류}` 휴가 신청을 `{결재자}`님이 승인하였습니다. | 5/26~5/27 연차 휴가 신청을 홍길동님이 승인하였습니다. |
| 연차 신청 반려 | 휴가신청서 결재가 반려되었습니다 | `{신청일자} {휴가종류}` 휴가 신청이 반려되었습니다. 사유를 확인해 주세요. | 5/26 오전 반차 휴가 신청이 반려되었습니다. 사유를 확인해 주세요. |
| 증명서 발급 완료 | 증명서 발급이 완료되었습니다 | 신청하신 `{증명서종류}`가 발급되었습니다. 확인 후 저장하세요. | 신청하신 재직증명서가 발급되었습니다. 확인 후 저장하세요. |
| 증명서 발급 반려 | 증명서 발급이 반려되었습니다 | `{증명서종류}` 발급 신청이 반려되었습니다. 사유를 확인해 주세요. | 재직증명서 발급 신청이 반려되었습니다. 사유를 확인해 주세요. |
| 새 공지사항 게시 | 새 공지사항이 등록되었습니다 | `{게시판명}`에 새 게시글이 등록되었습니다. | 경영지원부서 공지에 새 게시글이 등록되었습니다. |
| 결의서 결재 완료 | 결의서 결재가 승인되었습니다 | 제출하신 `{결의서제목}` 결의서가 최종 승인되었습니다. | 경비 결의서가 최종 승인되었습니다. |
| 결의서 결재 반려 | 결의서 결재가 반려되었습니다 | 제출하신 `{결의서제목}` 결의서가 반려되었습니다. 사유를 확인해 주세요. | 경비 결의서가 반려되었습니다. 사유를 확인해 주세요. |

> **치환자 규칙**
> - `{신청일자}`: 단일 날짜는 `5/26`, 범위는 `5/26~5/27`
> - `{휴가종류}`: `연차` / `오전 반차` / `오후 반차`
> - `{결재자}`: 결재자 이름 (예: `홍길동`)
> - `{증명서종류}`: `재직증명서` / `경력증명서` / `기타증명서`
> - `{게시판명}`: 게시글이 등록된 게시판 이름 (예: `경영지원부서 공지`)
> - `{결의서제목}`: 결의서 작성 시 입력한 제목

**사용자 흐름**

1. 알림 행 탭 → 해당 알림의 읽음 상태로 전환 → 알림 유형에 따라 대상 화면으로 이동
2. [모두 읽음] 탭 → 목록 내 모든 알림 읽음 처리 → 뱃지 소멸 → 토스트 표시 → 화면 유지
3. [← 뒤로] 탭 → 게시판 홈(M-002)으로 복귀

**조건부 규칙**
- 알림 목록이 없을 때: 빈 상태 안내 표시
- 모든 알림이 이미 읽음 상태일 때: [모두 읽음] 버튼 비활성화 또는 탭 시 동작 없음
- 알림 수신은 알림 설정(M-051)의 "푸시 알림 받기" 토글이 ON인 경우에만 동작
- 방해금지 모드 시간대에는 알림이 수신되지 않으며, 해당 시간 이후에 일괄 표시됨
- 알림 항목 탭 시 이동 대상 화면이 존재하지 않을 경우(삭제된 문서 등): 토스트 "해당 내역을 찾을 수 없습니다" 표시 후 알림 목록 유지

### M-051 알림 설정

| 항목 | 내용 |
|------|------|
| **화면 목적** | 앱 푸시 알림 수신 여부를 전체 또는 항목별로 설정하고, 방해금지 모드(시간대 지정)를 구성한다. |
| **진입 조건** | 더보기(M-040) → [알림 설정] 탭 |
| **다음 화면** | 없음 (현재 화면 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "알림 설정" | |
| 섹션 1 — 앱 푸시 알림 | 섹션 레이블 | "앱 푸시 알림" |
| 섹션 1 | 푸시 알림 받기 토글 | 기본값: ON. OFF 시 하단 항목별 설정 영역 비활성화(dimmed) 처리. 모든 알림 수신 중단 |
| 섹션 2 — 알림 항목 | 섹션 레이블 | "알림 항목" |
| 섹션 2 | 연차 / 휴가 토글 | 기본값: ON. 연차 신청 승인·반려 알림 수신 여부 |
| 섹션 2 | 증명서 토글 | 기본값: ON. 증명서 발급완료·반려 알림 수신 여부 |
| 섹션 2 | 결의서 토글 | 기본값: ON. 결의서 전자결재 완료·반려 알림 수신 여부 (영수증 등록은 알림 미발송) |
| 섹션 2 | 게시판 / 공지 토글 | 기본값: ON. 새 공지사항 게시 알림 수신 여부 |
| 섹션 3 — 방해금지 모드 | 섹션 레이블 | "방해금지 모드" |
| 섹션 3 | 방해금지 모드 토글 | 기본값: OFF. ON으로 전환 시 시작·종료 시간 설정 영역 펼쳐짐 |
| 섹션 3 | 시작 시간 설정 | 시·분 선택 (기본값: **22:00**, 시 옵션 22·23·00·21·20 / 분 옵션 00·30). 방해금지 모드 ON 시에만 노출 |
| 섹션 3 | 종료 시간 설정 | 시·분 선택 (기본값: **07:00**, 시 옵션 07·08·09·06 / 분 옵션 00·30). 방해금지 모드 ON 시에만 노출 |
| 섹션 3 | 안내 문구 | "매일 반복 적용됩니다" |

**사용자 흐름**

1. **푸시 알림 받기 토글 OFF**: 항목별 설정 섹션 비활성(dimmed), 모든 알림 수신 즉시 중단
2. **푸시 알림 받기 토글 ON**: 항목별 설정 섹션 활성화, 각 항목 토글에 따라 수신 여부 개별 제어
3. **방해금지 모드 토글 ON**: 시간 설정 영역 펼쳐짐 → 시작/종료 시간 각각 선택 → 설정 즉시 저장
4. **방해금지 모드 토글 OFF**: 시간 설정 영역 접힘 → 방해금지 해제
5. 설정 변경은 별도 [저장] 버튼 없이 토글·선택 즉시 서버에 반영(Auto-save)

**조건부 규칙**
- 푸시 알림 받기가 OFF인 경우: 항목별 알림 토글과 방해금지 모드 토글 모두 비활성화(dimmed) 처리. 탭해도 상태 변경 불가
- 방해금지 시작 시간과 종료 시간이 동일한 경우: "시작 시간과 종료 시간이 같을 수 없습니다" 오류 안내 + 설정 미저장
- 방해금지 모드 적용 중에는 알림(M-050) 목록에 알림이 쌓이며, 해제 후 수신된 것으로 처리됨
- 기기 자체의 알림 권한이 거부된 경우: 화면 상단에 "기기 알림 권한이 꺼져 있습니다. 기기 설정에서 허용해 주세요" 배너 표시
