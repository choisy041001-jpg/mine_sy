### M-006 인사 FAQ 목록

> 인사 FAQ는 **인사담당자가 관리자 PC(P-004 FAQ 관리)에서 등록한 '자주 묻는 질문과 답변'을 직원이 모바일에서 읽기 전용으로 열람**하는 기능이다. 전용 화면이 아니라 **게시판(M-게시판)의 게시글 목록·게시글 상세 화면을 재사용**하는 게시판형 채널이며, '인사 FAQ' 채널에서는 직원의 글 작성(글쓰기 FAB)이 노출되지 않는다. 더보기(M-040)의 하위 메뉴로 진입한다.
> **구현 함수**: 더보기 [인사 FAQ] → `goFaqList()`(→ `faq` 목록) / 목록 행 탭 → `openFaq(id)`(→ `post-detail` 재사용, `_postDetailContext='faq'`로 하단 탭은 더보기 유지).

| 항목 | 내용 |
|------|------|
| **화면 목적** | 인사담당자가 관리자 PC(P-004 FAQ 관리)에서 등록한 '자주 묻는 질문과 답변' 글을 단일 목록으로 확인한다 (카테고리 구분 없음). FAQ 등록·수정·삭제는 관리자 PC(P-004)에서만 가능하며, 직원은 읽기 전용으로 조회만 한다. |
| **진입 조건** | 더보기(M-040) → [인사 FAQ] 메뉴 탭 (`goFaqList()` → `faq` 목록 화면) |
| **다음 화면** | 인사 FAQ 상세 M-008 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "인사 FAQ" | 게시판 목록(M-003) 상단바 재사용, 채널명 = 인사 FAQ |
| FAQ 글 목록 | 게시글 행 | 질문 제목 (공지 핀 없음) |
| FAQ 글 목록 | 메타 | `{작성자} · {게시일 YYYY.MM.DD} · 댓글 {N}` (작성자 = 인사담당자 성명, 게시 날짜는 `YYYY.MM.DD`, 조회수 미표시) |
| 글쓰기 FAB | (숨김) | 인사 FAQ 채널에서는 직원 글 작성 불가 — FAB 미노출 |

**사용자 흐름**
- FAQ 글 행 탭 → 인사 FAQ 상세(M-008)

**조건부 규칙**
- 인사 FAQ 글은 인사담당자(관리자)가 PC(P-004)에서 직접 등록·관리. 직원은 조회 전용(새 글 작성 불가)
- 글쓰기 FAB는 인사 FAQ 채널에서 항상 숨김
- 목록이 없을 경우: 빈 상태 안내 표시

### M-008 인사 FAQ 상세

> M-008은 게시판의 **게시글 상세(`post-detail`, M-004)** 화면을 `인사 FAQ` 채널로 그대로 재사용한다(FAQ 항목을 게시글 데이터에 `board:'인사 FAQ'`·`mine:false`·`notice:false`로 추가 → ⋯ 수정/삭제 메뉴·공지핀 미노출, 읽기 전용).

| 항목 | 내용 |
|------|------|
| **화면 목적** | 선택한 FAQ 글의 질문 내용과 인사담당자 답변(본문)을 확인하고, 댓글로 추가 문의·보충 답변을 주고받는다. |
| **진입 조건** | 인사 FAQ 목록(M-006)에서 글 행 탭 |
| **다음 화면** | 없음 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 채널명("인사 FAQ") + 더보기(⋯) | 게시글 상세(M-004) 화면 재사용 |
| 게시글 본문 | 제목 / 메타 / 본문 | 제목(공지 핀 없음) / `{작성자} · {게시일 YYYY.MM.DD}`(작성자 = 인사담당자 성명, 게시 날짜는 시각 없이 `YYYY.MM.DD`, 조회수 미표시) / 본문(HTML — 질문 안내·답변을 단락·강조로 구성) |
| 댓글 영역 | 댓글 목록 | `댓글 {N}` + 각 댓글: 작성자 **이름만**(부서·아바타 미표시 — M-000 §5.2) / 내용 / 작성 시각. 인사담당자도 동일 성명으로 댓글 보충 답변 |
| 댓글 입력 | 입력 바 + [등록] | 직원이 추가 질문을 댓글로 작성(목업). 게시글 상세 공용 입력 바 |

**조건부 규칙**
- FAQ 글 본문(질문+답변)은 인사담당자가 PC(P-004)에서 등록한 콘텐츠이며 직원은 본문을 수정·삭제할 수 없음(본인 글이 아니므로 ⋯ 메뉴에 수정/삭제 미노출)
- 조회수는 본문 메타에 표시하지 않음
- 댓글 [등록] 동작은 프로토타입 목업 수준

### M-050 알림

| 항목 | 내용 |
|------|------|
| **화면 목적** | 앱에서 수신한 푸시 알림 내역을 날짜 순서로 확인하고, 알림 항목을 탭하여 관련 화면으로 바로 이동한다. |
| **진입 조건** | 게시판 홈(M-002) 상단 알림 아이콘(🔔) 탭 |
| **다음 화면** | 알림 유형에 따라 해당 화면으로 분기 (아래 조건부 규칙 참조) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] 버튼 + "알림" 타이틀 | |
| 상단 내비게이션 | [모두 읽음] 버튼 | 우측 정렬. 탭 시 전체 알림 읽음 처리 + 뱃지 소멸 + 토스트 "모든 알림을 읽음 처리했습니다" |
| 날짜 섹션 헤더 | 날짜 라벨 | "오늘" / "어제" / "M월 D일 (요일)"(예: `5월 18일 (일)`) 형식으로 날짜별 그룹화 |
| 알림 행 (미읽음) | 행 배경 | 연파랑(#fafcff) 배경 + 우측 파란 점(●) 표시로 미읽음 구분 |
| 알림 행 (읽음) | 행 배경 | 기본 흰색 배경, 파란 점 없음 |
| 알림 행 | 유형 아이콘 | 알림 유형별 색상 아이콘 (아래 알림 유형 정의 참조) |
| 알림 행 | 알림 제목 | 굵은 텍스트, 13px |
| 알림 행 | 알림 내용 | 보조 텍스트, 12px, 최대 2줄 말줄임 |
| 알림 행 | 수신 시각 | 작은 텍스트(11px) / **모든 알림 동일하게 `YYYY.MM.DD HH:mm` 절대 시각으로 표시**(상대 시각 "N분 전·N시간 전" 미사용) |
| 빈 상태 | 안내 문구 | 수신된 알림이 없을 때 "받은 알림이 없습니다" 표시 |

**알림 유형 정의**

| 알림 유형 | 아이콘 색상 | 발생 조건 | 탭 시 이동 화면 |
|---------|----------|---------|--------------|
| 연차 승인 | 초록(success) | 결재자가 휴가신청서를 승인 처리 시 | **M-010 (연차 관리)** |
| 연차 반려 | 빨강(danger) | 결재자가 신청서를 반려 처리 시 | **M-010 (연차 관리)** |
| 증명서 발급 완료 | 파랑(accent) | 인사담당자가 증명서 발급 처리 완료 시 | **M-030 (증명서 관리)** |
| 증명서 반려 | 빨강(danger) | 인사담당자가 증명서 신청을 반려 시 | **M-030 (증명서 관리)** |
| 새 공지사항 | 파랑(accent, 연한 배경) | 관리자가 게시판에 공지글 등록 시 | M-002 (게시판 홈) |
| 결의서 결재 완료 | 초록(success) | 전자결재 ON 상태에서 결재선에 따른 결재가 최종 승인 완료 시 | M-020 (영수증 관리) |
| 결의서 결재 반려 | 빨강(danger) | 전자결재 ON 상태에서 결재 중 반려 처리 시 | M-020 (영수증 관리) |

> **알림 탭 시 이동 화면은 개별 상세 화면이 아니라 해당 기능의 메인 화면으로 이동한다**(프로토타입 `handleNotifClick` 정본): 연차 승인·반려 → 연차 관리(M-010, `leave-approval`), 증명서 발급 완료·반려 → 증명서 관리(M-030, `cert-types`), 결의서 결재 완료·반려 → 영수증 관리(M-020, `expense-hub`), 새 공지사항 → 게시판 홈(M-002, `board`). 직원은 이동한 메인 화면에서 해당 건을 찾아 상세를 확인한다.

**기기 알림 문구**

OS 알림 센터·잠금화면·인앱 배너에 실제로 표시되는 제목(title)과 본문(body). 앱 이름 "MY인사"는 시스템이 자동 표시하며 개발 제어 범위 밖이다.

| 알림 유형 | 제목 (title) | 본문 (body) | 표시 예시 |
|---------|------------|-----------|---------|

### M-050 알림
- **수신 시각 형식**: **모든 알림 동일하게 `YYYY.MM.DD HH:mm`**(예: `2026.06.05 13:20`). 상대 시각("방금 전·N분 전·N시간 전")은 사용하지 않는다.
- **날짜 섹션 헤더**: "오늘" / "어제" / "M월 D일 (요일)"(예: `5월 18일 (일)`).
- **미읽음 표시**: 연파랑(#fafcff) 배경 + 우측 파란 점(●). [모두 읽음] 탭 시 전체 읽음 처리·뱃지 소멸·토스트 "모든 알림을 읽음 처리했습니다".
- **게시판 알림 아이콘 뱃지(●)**: 미읽음 알림이 1건 이상일 때만 게시판 홈 우측 상단 🔔 아이콘에 빨간 점(●)을 표시한다(**M-000 §4** 미읽음 표시 규칙). **미읽음이 0건이면 빨간 점을 표시하지 않는다** — 개별/모두 읽음 처리로 미읽음이 0이 되거나, 초기 진입 시 미읽음이 없으면 뱃지 미노출(구현: `updateNotifBadge()`가 미읽음 유무로 동기화).
- **탭 시 이동**: 개별 상세 화면 직접 딥링크가 아니라 **해당 기능 메인 화면**으로 이동(연차 관리 M-010 / 증명서 관리 M-030 / 영수증 관리 M-020 / 게시판 홈 M-002).
- **유형별 본문 예시(프로토타입 정적 데이터)**: 증명서 발급 완료 "재직증명서 발급이 완료되었습니다. 확인해 보세요." / 연차 승인 "5월 4일(월) 휴가신청서가 승인되었습니다." / 증명서 반려 "기타증명서 신청이 반려되었습니다. 사유를 확인해 주세요." / 새 공지사항 "2026년 5월 정기 회의 일정 및 안건이 게시되었습니다." / 결의서 결재 완료 "제출하신 4월 거래처 미팅 비용 결의서가 최종 승인되었습니다." / 결의서 결재 반려 "제출하신 결의서가 반려되었습니다. 사유를 확인해 주세요."

### M-051 알림 설정
- **토글 라벨 / 기본값 / 설명**: 푸시 알림 받기(ON · "MY인사 앱의 모든 푸시 알림을 받습니다") / 연차 / 휴가(ON · "신청·승인·반려 알림") / 증명서(ON · "발급완료·반려 알림") / **결의서**(ON · "전자결재 완료·반려 알림" — 영수증 등록 자체는 알림 미발송) / 게시판 / 공지(ON · "새 공지사항 알림") / 방해금지 모드(OFF · "설정한 시간 동안 알림을 받지 않습니다").
- **방해금지 시간 기본값**: 시작 `22:00` / 종료 `07:00`, "매일 반복 적용됩니다". 시·분 선택 옵션은 시작 시 22·23·00·21·20 / 종료 시 07·08·09·06 / 분 공통 00·30.
- **저장 방식**: 별도 [저장] 버튼 없이 토글·시간 선택 즉시 반영(Auto-save). 푸시 알림 받기 OFF 시 항목별 토글·방해금지 모드 영역 dimmed(탭 불가).
