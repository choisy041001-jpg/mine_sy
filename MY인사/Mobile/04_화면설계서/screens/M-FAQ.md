## 화면 상세 설계

### M-006 인사 FAQ 목록

| 항목 | 내용 |
|------|------|
| **화면 목적** | 인사담당자가 관리자 PC(P-004 FAQ 관리)에서 등록한 '자주 묻는 질문과 답변' 글을 단일 목록으로 확인한다 (카테고리 구분 없음). FAQ 등록·수정·삭제는 관리자 PC(P-004)에서만 가능하며, 직원은 읽기 전용으로 조회만 한다. |
| **진입 조건** | 더보기(M-040) → [인사 FAQ] 메뉴 탭 (`goPostList('인사 FAQ', true)`) |
| **다음 화면** | 인사 FAQ 상세 M-008 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "인사 FAQ" | 게시판 목록(M-003) 상단바 재사용, 채널명 = 인사 FAQ |
| FAQ 글 목록 | 게시글 행 | 질문 제목 (공지 핀 없음) |
| FAQ 글 목록 | 메타 | `{작성자} · {등록일} · 댓글 {N}` (작성자 = 인사담당자 성명, 조회수 미표시) |
| 글쓰기 FAB | (숨김) | 인사 FAQ 채널에서는 직원 글 작성 불가 — FAB 미노출 |

**사용자 흐름**
- FAQ 글 행 탭 → 인사 FAQ 상세(M-008)

**조건부 규칙**
- 인사 FAQ 글은 인사담당자(관리자)가 PC(P-004)에서 직접 등록·관리. 직원은 조회 전용(새 글 작성 불가)
- 글쓰기 FAB는 인사 FAQ 채널에서 항상 숨김
- 목록이 없을 경우: 빈 상태 안내 표시

### M-008 인사 FAQ 상세

| 항목 | 내용 |
|------|------|
| **화면 목적** | 선택한 FAQ 글의 질문 내용과 인사담당자 답변(본문)을 확인하고, 댓글로 추가 문의·보충 답변을 주고받는다. |
| **진입 조건** | 인사 FAQ 목록(M-006)에서 글 행 탭 |
| **다음 화면** | 없음 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 채널명("인사 FAQ") + 더보기(⋯) | 게시글 상세(M-004) 화면 재사용 |
| 게시글 본문 | 제목 / 메타 / 본문 | 제목(공지 핀 없음) / `{작성자} · {등록일시 YYYY.MM.DD HH:MM}`(작성자 = 인사담당자 성명, 조회수 미표시) / 본문(HTML — 질문 안내·답변을 단락·강조로 구성) |
| 댓글 영역 | 댓글 목록 | `댓글 {N}` + 각 댓글: 작성자 이름 + 부서(아바타 없음) / 내용 / 작성 시각. 인사담당자도 동일 성명으로 댓글 보충 답변 |
| 댓글 입력 | 입력 바 + [등록] | 직원이 추가 질문을 댓글로 작성(목업). 게시글 상세 공용 입력 바 |

**조건부 규칙**
- FAQ 글 본문(질문+답변)은 인사담당자가 PC(P-004)에서 등록한 콘텐츠이며 직원은 본문을 수정·삭제할 수 없음(본인 글이 아니므로 ⋯ 메뉴에 수정/삭제 미노출)
- 조회수는 본문 메타에 표시하지 않음
- 댓글 [등록] 동작은 프로토타입 목업 수준

## 세부 설계 요건 — 데이터 표시 기준

> 인사 FAQ는 게시판(M-게시판)의 게시글 목록·상세 화면을 `인사 FAQ` 채널로 재사용한다. 표시 데이터의 출처·형식·읽기 전용 규칙을 필드 단위로 정의한다.

### 공통 — 데이터 출처
- **FAQ 글(질문+답변)**: 인사담당자가 관리자 PC **FAQ 관리(P-004)** 에서 등록한 글이 경리나라 게시판 앱을 통해 모바일 `인사 FAQ` 채널로 노출된다. 직원은 등록·수정·삭제 권한이 없고 **읽기 전용**으로 열람한다.
- **작성자**: 인사 FAQ 글의 작성자는 글을 등록한 **인사담당자 성명**(예: 한관리자)으로 표시된다. 부서명('인사팀')이 아닌 담당자 개인 성명으로 노출하며, 댓글로 보충 답변하는 인사담당자도 동일 성명으로 표기된다.
- **공지 핀**: 인사 FAQ 목록·상세에는 **`공지` 핀 뱃지를 사용하지 않는다**(모든 FAQ 글 동일 형식 표시).
- **카테고리**: 인사 FAQ는 **카테고리 구분이 없는 단일 목록**으로 관리한다(목록·상세에 카테고리 필터·뱃지 없음).
- **조회수**: 게시글 목록·상세 어디에도 **조회수를 표시하지 않는다**(`_renderPostDetail` 메타 미출력, 데이터 필드 제거 — M-게시판 정합).

### M-006 인사 FAQ 목록
- **진입**: 더보기(M-040) [인사 FAQ] → `goPostList('인사 FAQ', true)`. 게시글 목록 화면(M-003)을 `인사 FAQ` 채널로 표시하며, 더보기 탭 강조 상태를 유지한다.
- **글 행**: 질문 제목(길면 말줄임). 메타는 `{작성자=인사담당자 성명} · {등록일 M월 D일} · 댓글 {N}` 형식이며 공지 핀·조회수는 표시하지 않는다.
- **정렬**: 등록일 최신순(프로토타입 정적 데이터 기준).
- **글쓰기 FAB**: 인사 FAQ 채널에서는 항상 숨김(`isFaqBoard=true` → FAB `display:none`). 직원은 글을 작성할 수 없다.
- **빈 상태**: 등록된 FAQ 글이 없으면 빈 상태 안내 문구 표시.

### M-008 인사 FAQ 상세
- **상단바**: 채널명 = `인사 FAQ`(목록에서 전달). 우측 더보기(⋯)는 본인 글에 한해 수정/삭제 메뉴를 노출하나, FAQ 글은 인사담당자 작성 글이므로 직원에게는 수정/삭제가 노출되지 않는다.
- **본문 메타**: `{작성자=인사담당자 성명} · {등록일시 YYYY.MM.DD HH:MM}`. 공지 핀·조회수 미표시.
- **본문**: HTML 본문으로 질문 안내와 답변을 단락(`<p>`)·강조(`<strong>`)로 구성한다(예: 연차 부여 기준·증명서 발급 소요 시간·급여 공제 항목 등 인사 안내).
- **댓글**: `댓글 {N}` 헤더 + 댓글 행(작성자 이름 + 부서(없으면 생략, 아바타 없음) / 내용 / 작성 시각 `M월 D일 HH:mm`). 직원의 추가 문의와 인사담당자의 보충 답변이 댓글로 누적된다(인사담당자 댓글도 성명으로 표기).
- **댓글 입력**: 하단 고정 입력 바(`댓글을 입력하세요` + [등록]). 댓글 작성은 가능(목업)하되 새 게시글(질문) 작성은 불가하다.

## 화면 전환 흐름

```
더보기(M-040)
  └─ [인사 FAQ] → 인사 FAQ 목록(M-006 = 게시글 목록 M-003, isFaqBoard)
                    └─ FAQ 글 탭 → 인사 FAQ 상세(M-008 = 게시글 상세 M-004)
                                     └─ 댓글 작성(목업) / [← 뒤로] → 목록 복귀
```

> 인사 FAQ 글의 등록·수정·삭제는 관리자 PC **FAQ 관리(P-004)** 에서 수행된다(모바일은 조회·댓글만).
