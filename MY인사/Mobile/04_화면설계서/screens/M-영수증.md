## 화면 상세 설계

### M-020 영수증 관리

| 항목 | 내용 |
|------|------|
| **화면 목적** | 사업장에 등록된 전체 영수증 목록을 날짜별로 조회하고, 결의서 작성을 위해 영수증을 선택한다. 등록자와 무관하게 모든 영수증을 수정·삭제할 수 있다. |
| **진입 조건** | 하단 탭바 [영수증] 탭 |
| **다음 화면** | 영수증 상세 M-022 / 영수증 등록 M-021 / 결의서 작성 M-023 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 | "영수증 관리" |
| 영수증 목록 | 날짜 헤더 + 영수증 카드 | 좌측: **사용처(강조)** + 바로 하단 **용도** / 우측: **금액**(행 세로 중앙 정렬). 지급처·이동 화살표(›)는 표시하지 않는다 |
| 영수증 목록 | "결의서 있음" 뱃지 | 결재요청(진행중) 또는 결재완료(승인) 상태인 결의서에 포함된 영수증에 표시 |
| 하단 | 영수증 FAB(+) | 영수증 등록 / 결의서 작성 (선택 모드 진입) |

**선택 모드 진입 시 추가 UI**

| 컴포넌트 | 상세 |
|---------|------|
| 상단 [취소] 버튼 | 탭 시 선택 모드 해제, 일반 모드로 복귀 |
| 선택 체크박스 | 각 카드 좌측에 표시 |
| "결의서 있음" 영수증 | 선택 불가 (회색 처리) |
| 하단 고정 버튼 | **[결의서 작성 (N건)]** — 전자결재 ON/OFF와 무관하게 항상 동일 명칭. 선택 건수 실시간 반영. 전자결재 설정에 따른 분기는 결의서 작성(M-023) 화면에서만 처리 |

**사용자 흐름**
- 카드 탭 (일반 모드) → 영수증 상세(M-022)
- FAB → [영수증 등록] → M-021
- FAB → [결의서 작성] → 선택 모드 진입 → 선택 후 [결의서 작성 (N건)] 탭 → M-023
- 선택 모드 중 상단 [취소] 탭 → 선택 모드 해제, 일반 모드 복귀

**조건부 규칙**
- 선택 모드에서 "결의서 있음" 영수증 탭 시: 선택 불가, 시각적 비활성만 표시 (별도 안내 없음)
- 연결 결의서가 반려 또는 보류 상태가 된 경우: 해당 영수증의 "결의서 있음" 뱃지 자동 제거 → 일반 영수증과 동일하게 처리 (선택·수정·삭제 가능). 반려된 결의서와의 연결 관계는 끊김
- 영수증이 하나도 없을 경우: 빈 상태 안내
- [결의서 작성] 버튼: 선택 건수가 0이면 비활성화

### M-021 영수증 등록

| 항목 | 내용 |
|------|------|
| **화면 목적** | 경비 영수증 이미지와 사용 정보를 입력하여 등록(또는 수정)한다. 등록 즉시 경리나라 **매입(간이영수증)**에 자동 저장된다. |
| **진입 조건** | **등록 모드**: 영수증 FAB → [영수증 등록] → 이미지 선택 팝업(카메라/갤러리) → **이미지 선택 완료 후 본 화면 진입** / **수정 모드**: 영수증 상세(M-022) → [수정] |
| **다음 화면** | 영수증 관리 M-020 |

> **등록/수정 모드 구분**: 등록 모드는 상단 타이틀 `영수증 등록`·제출 버튼 `등록하기`, 수정 모드는 `영수증 수정`·`수정하기`로 표시한다.

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 타이틀 | 등록 모드 "영수증 등록" / 수정 모드 "영수증 수정" |
| 상단 배너 | 경리나라 자동 저장 안내 | "등록된 영수증은 경리나라 매입(간이영수증)에 자동 저장됩니다." |
| 영수증 이미지 | 빈 상태(등록) / 미리보기 상태 | **등록 모드**: 이미지 미선택 시 첨부 유도 영역(탭 시 카메라/갤러리 팝업). **이미지 선택 후·수정 모드**: 영수증 이미지 **미리보기 썸네일 + [변경] 버튼** 표시. [변경] 탭 시 카메라/갤러리 팝업으로 재선택(현재 화면 유지). 썸네일 탭 시 이미지 확대 |
| 이미지 선택 팝업 | 카메라 촬영 / 갤러리 선택 | **카메라 촬영** → 단말 **카메라 앱**(촬영, `capture`) / **갤러리 선택** → 단말 **사진 보관함(갤러리)**. 이미지가 실제 선택된 뒤에만 다음 단계(등록 모드는 본 화면 진입 / 미리보기 갱신)로 진행하며, 선택 취소 시 아무 동작 없음 |
| 입력 폼 | 사용일시 | **필수** / 단일 탭 필드(`YYYY.MM.DD HH:mm` 형식 표시) / **등록 모드 진입 시 현재 시각으로 기본 세팅**(분은 5분 단위 반올림, 시계 피커 정합) — 탭 시 바텀 시트 팝업 호출. **팝업 구성**: 상단 달력 영역(월 이동·날짜 그리드 선택) + 하단 아날로그 시계 영역(24시간제; 시 탭과 분 탭으로 모드 구분 — 시 선택 시 자동으로 분 선택 모드 전환 / 각 탭 터치로 수동 전환 가능 / 분은 5분 단위 선택). [확인] 탭 시 필드에 반영. 과거·현재·미래 날짜 모두 선택 가능. |
| 입력 폼 | 금액 (원 단위) | **필수** / 숫자만 허용 / 최대 12자리 / 3자리 콤마 자동 삽입 / 0원 등록 불가 |
| 입력 폼 | 사용처 방식 선택 칩 | **거래처 / 직접입력** — 기본값: 직접입력. **필수** |
| 입력 폼 | 사용처 — 직접입력 | 직접입력 칩 선택 시 표시. 최대 100자 / 단일 라인. 하단에 **사업자번호** 입력 필드 추가 표시 (선택 / 숫자 10자리 / 단일 라인) |
| 입력 폼 | 사용처 — 거래처 | 거래처 칩 선택 시 표시. **① 거래처명 검색 필드** + **② 사업자번호 필드(비활성·읽기 전용)** 2개로 구성. 거래처명 입력 시 실시간 레이어로 거래처 목록(명칭+사업자번호) 표시 → 거래처 선택 시 거래처명 필드에 명칭, **사업자번호 필드에 해당 거래처의 사업자번호 자동 반영**. 사업자번호 필드는 직접 입력 불가(비활성). 동일 거래처명의 다른 사업장을 사업자번호로 식별하기 위함. 거래처명을 다시 입력·수정하면 자동 반영된 사업자번호는 초기화 (경리나라 거래처 기준) |
| 입력 폼 | 용도 | 드롭다운 단일 선택 / **선택** (*예시 선택지: 거래처 식대 · 회의 다과 · 야근 교통 · 출장비 · 업무 소모품 · 기타 — 실제 항목은 시스템 설정에 따라 상이할 수 있음*) |
| 입력 폼 | 내용 | 선택 / 다중 라인 / 사용 내역 설명 입력 (예: "A사 미팅 식대 (4명)") |
| 입력 폼 | 지급처 선택 칩 | **직원 / 거래처 / 기타** — 기본값: 직원. **선택**. 사용처 방식이 직접입력인 경우 거래처 칩 미표시 (직원·기타만 선택 가능) |
| 입력 폼 | 지급처 — 직원 검색 | 지급처 = 직원 칩 선택 시 표시. 직원명 입력 시 실시간 레이어로 명칭 포함 직원 목록 표시 → 선택 (경리나라 직원관리 기준) |
| 입력 폼 | 지급처 — 거래처 검색 | 지급처 = 거래처 칩 선택 시 표시. 거래처명 입력 시 실시간 레이어로 명칭 포함 거래처 목록 표시 → 선택 (경리나라 거래처 기준) |
| 입력 폼 | 지급처 — 기타 직접입력 | 지급처 = 기타 칩 선택 시 표시. 최대 100자 / 단일 라인 |
| 입력 폼 | 지급수단 선택 카드 | **개인 경비 / 회사 현금으로 지급 / 전도금으로 지급** — 라디오형 카드 단일 선택. 기본값: 개인 경비. **필수**. 각 카드에 지급수단별 경리나라 자동 연동 결과 태그 표시 (아래 "세부 설계 요건 — 데이터 표시 기준" 참조) |
| 하단 | [등록하기] 버튼 | 전체 너비 |

**유효성 규칙**

> **필수 입력 항목 = 사용일시 · 사용처 · 금액**. [등록하기] 탭 시 위에서부터 첫 번째 미입력 항목을 찾아 **`{항목}이/가 입력되지 않았습니다.`** 알림 팝업을 띄우고 등록을 중단한다(받침 유무에 따라 이/가 자동 선택 — 예: "사용일시가 입력되지 않았습니다.", "금액이 입력되지 않았습니다.", "사용처가 입력되지 않았습니다."). 영수증 등록·수정 모두 동일하게 적용.

| 오류 조건 | 처리 결과 |
|---------|---------|
| 사용일시 미선택 | `사용일시가 입력되지 않았습니다.` 알림 팝업, 등록 불가 |
| 금액 미입력 | `금액이 입력되지 않았습니다.` 알림 팝업, 등록 불가 |
| 사용처 미입력 (직접입력 명칭 공란 또는 거래처 미선택) | `사용처가 입력되지 않았습니다.` 알림 팝업, 등록 불가 |
| 이미지 파일 용량 초과 (10 MB) | "파일은 최대 10 MB까지 첨부 가능합니다" 안내 후 파일 첨부 취소 |
| 0원 금액 입력 | 등록 불가 |

**조건부 규칙**
- 사용처 방식 칩 변경 (거래처 → 직접입력): 선택된 거래처 정보 초기화. 지급처가 거래처로 설정된 경우 자동 초기화 (직원·기타 중 재선택 필요)
- 사용처 방식 칩 변경 (직접입력 → 거래처): 입력된 직접입력 내용 초기화. 지급처 거래처 칩 활성화
- 지급처 칩 변경 시 이전에 입력·선택된 정보 초기화

### M-022 영수증 상세

| 항목 | 내용 |
|------|------|
| **화면 목적** | 개별 영수증의 상세 정보를 확인하고, 수정·삭제·결의서 작성을 진행한다. |
| **진입 조건** | 영수증 관리(M-020) → 영수증 카드 탭 (일반 모드) |
| **다음 화면** | 영수증 관리 M-020 / 영수증 등록 M-021 / 결의서 작성 M-023 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "영수증 상세" | |
| 상태 정보 | 결의/결재 상태 뱃지 + 금액(크게) + 사용처 | 위에서부터 **뱃지 → 금액 → 사용처** 순. 뱃지는 **미결의**(결의/결재 미진행) / **결의완료**(결의서·전자결재 진행/완료) |
| 영수증 이미지 | 탭 시 **전체 화면 미리보기** | 이미지를 탭하면 화면 전체(상단바·탭바 위)를 덮는 어두운 오버레이로 영수증 이미지를 크게 표시한다. 오버레이가 탭바·[뒤로] 위에 위치하여 미리보기 중에는 화면 전환이 차단된다. **우측 하단에 [저장](이미지 다운로드) 버튼, 우측 상단에 [×] 닫기 버튼**을 제공하며, 닫기는 [×] 버튼으로만 수행한다(배경 탭으로는 닫히지 않음) |
| 상세 정보 목록 | 사용일시 / 사용처(방식·명칭·사업자번호) / 용도 / 지급처 / 지급수단 / 사용금액 | 사용일시 `YYYY.MM.DD HH:mm` |
| 내용 | 등록 시 입력했을 경우 표시 | |
| 하단 버튼 3종 | [삭제] · [수정] · [결의서 작성] | 마지막 버튼은 항상 [결의서 작성](전자결재 ON/OFF 무관). 전자결재 분기는 결의서 작성(M-023) 화면에서만 처리. **결의완료 영수증은 [수정]·[삭제]·[결의서 작성] 3개 버튼 모두 비활성화** |

> 처리 이력(타임라인) 영역은 표시하지 않는다.

**조건부 규칙**
- **수정·삭제 권한**: 등록자와 무관하게 누구든 수정·삭제할 수 있다(미결의 건 한정).
- **하단 버튼 활성/비활성**: 해당 영수증이 **지출결의 또는 간편지급(지급확인) 처리 완료(결의완료)** 상태이면 **[수정]·[삭제]·[결의서 작성] 3개 버튼을 모두 비활성화**(흐림·클릭 불가)한다. 미결의 건은 3개 버튼 모두 활성. (별도 알림 팝업 없이 버튼 자체를 비활성화 처리 — 수정·삭제 불가 + 중복 결의 방지)
- [삭제]: (미결의 건) 탭 시 "영수증을 삭제하시겠습니까?" 확인 팝업 → 확인 시 삭제 후 영수증 관리(M-020) 복귀
- [수정]: (미결의 건) 영수증 등록(M-021)으로 이동, 기존 정보 자동 입력 상태로 표시
- [결의서 작성]: (미결의 건) 해당 영수증 1건 선택된 상태로 M-023으로 이동. (제출 버튼명 [상신하기]/[저장하기] 분기는 M-023 화면에서 전자결재 설정에 따라 결정)

### M-023 결의서 작성 / 결재요청

| 항목 | 내용 |
|------|------|
| **화면 목적** | 선택된 영수증을 묶어 경비 결의서를 작성한다. 경리나라 지출결의서 전자결재 설정에 따라 결재요청(ON) 또는 지출결의서관리 전송(OFF)으로 처리한다. |
| **진입 조건** | 영수증 관리(M-020) 선택 모드 → [결의서 작성 (N건)] / 영수증 상세(M-022) → [결의서 작성] |
| **다음 화면** | 영수증 관리 M-020 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 화면 타이틀 | **전자결재 ON**: "결재요청" / **전자결재 OFF**: "결의서 작성" |
| 입력 폼 | 결의서 제목 | **필수** / 최대 100자 / 단일 라인 / **화면 진입 시 기본값 자동 입력**: `지출결의서(간이영수증) {N}건, {#,###}원`(N=엮인 영수증 건수, 금액=영수증 총 합계). 사용자가 수정 가능 |
| 입력 폼 | 내용 텍스트 입력 | **필수** / 최대 1,000자 / 다중 라인 / 카운터 표시 |
| 영수증 내역 섹션 | 영수증별 **일자·사용처·용도·금액** | 읽기 전용. 일자는 **`YYYY.MM.DD`까지만** 표시(시·분 미표시) |
| 합계 금액 | 영수증 금액 합산 표시 | 자동 계산 |
| 결재선 | 신청자 → 결재자 순서 표시 | **전자결재 ON 시만 표시** — 경리나라 자동 지정 안내 문구 포함. **결재선이 2개 이상인 경우 [결재선 변경] 선택 시** 라디오 버튼으로 결재선 선택 팝업 호출 |
| 하단 | 제출 버튼 | **전자결재 ON**: [상신하기] / **전자결재 OFF**: [저장하기] — 전체 너비 |

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 제목 미입력 | 오류 표시, 제출 불가 |
| 내용 미입력 | 오류 표시, 제출 불가 |
| 영수증 0건 | 이전 화면(M-020)에서 진입 자체 차단 |

**조건부 규칙 — 전자결재 설정**
- **전자결재 ON** (경리나라 지출결의서 전자결재 옵션 활성): 화면 타이틀 "결재요청" · 하단 버튼 **[상신하기]** 표시. 결재선 섹션 표시. [상신하기] 탭 시 결재선을 따라 결재 진행 (경리나라 결재함에서 처리). 결재 완료·반려 시 앱 푸시 알림(M-050) 발송.
- **전자결재 OFF** (경리나라 지출결의서 전자결재 옵션 비활성): 화면 타이틀 "결의서 작성" · 하단 버튼 **[저장하기]** 표시. 결재선 섹션 미표시. [저장하기] 탭 시 경리나라 지출결의서관리로 전송.
- **결재선 변경**: 결재선이 2개 이상 등록된 경우 [결재선 변경]을 선택하면 라디오 버튼 팝업이 호출되며 원하는 결재선을 선택할 수 있다. (전자결재 ON 시만 해당)
- 처리 완료 후 영수증 관리(M-020)로 이동.

## 세부 설계 요건 — 데이터 표시 기준

> 화면별로 표시되는 데이터의 **출처·형식·연동 결과**를 필드 단위로 정의한다. 모든 영수증·결의서 데이터는 경리나라와 실시간 연동된다.

### 공통 — 데이터 출처

> 데이터 출처 공통 원칙은 **M-000 §5.1**, 전자결재·결재선(자동 지정·[결재선 변경] 팝업·결재함 처리·결재 결과 푸시)은 **M-000 §6** 참조. 본 절은 영수증/결의서 고유 출처·연동만 정의.

- **영수증 데이터**: 등록 즉시 경리나라 **매입(간이영수증)**에 자동 저장된다. 지급수단에 따라 추가 장부(현금출납장·전도금원장)에 연동 기록된다(아래 매핑 참조).
- **거래처 목록**: 경리나라 거래처 마스터(명칭·사업자번호) 기준. 사용처·지급처 거래처 검색 레이어에 사용.
- **직원 목록**: 경리나라 직원관리 기준. 지급처 직원 검색 레이어에 사용.
- **결의서/전자결재 동작**: 경리나라 지출결의서 **전자결재 설정(ON/OFF)**에 따라 버튼명·결재선 표시·전송 경로가 달라진다.
- **금액 표기**: 원 단위, 3자리 콤마 자동 삽입(예: `25,000`).

### 공통 — 지급수단별 경리나라 연동 매핑
| 지급수단 | 카드 설명 문구 | 경리나라 자동 연동(카드 태그) |
|---------|--------------|---------------------------|
| **개인 경비** (기본값) | 개인카드·현금 등 본인 부담으로 지출한 경우 | 매입(간이영수증) 등록 *(파랑 태그)* |
| **회사 현금으로 지급** | 회사 현금을 수령해 지출한 경우 | 매입(간이영수증) 등록 *(파랑)* + 현금출납장 자동 등록 *(초록)* |
| **전도금으로 지급** | 미리 지급받은 전도금으로 지출한 경우 | 매입(간이영수증) 등록 *(파랑)* + 전도금원장 자동 등록 *(주황)* |

- 라디오형 카드 단일 선택, 기본 선택값은 **개인 경비**. 카드 내 태그 색상은 선택 시 발생하는 경리나라 연동 결과를 시각적으로 안내한다(파랑=간이영수증, 초록=현금출납장, 주황=전도금원장).

### M-020 영수증 관리
- **날짜 헤더**: 영수증 사용일시의 날짜(YYYY.MM.DD) 기준으로 그룹핑하여 표시.
- **영수증 카드**: 좌측에 사용처 명칭(강조)과 바로 하단 용도, 우측에 금액(원, 3자리 콤마, 행 세로 중앙 정렬). 지급처·이동 화살표(›)는 표시하지 않는다. 행 높이는 컴팩트하게(약 59px) 고정한다. 데이터는 경리나라 매입(간이영수증) 기준.
- **"결의서 있음" 뱃지**: 해당 영수증이 결재요청(진행중) 또는 결재완료(승인) 상태의 결의서에 포함된 경우 표시. 연결 결의서가 반려·보류되면 자동 제거.
- **하단 고정 버튼명**: 항상 `[결의서 작성 (N건)]`(전자결재 ON/OFF 무관). N = 선택 모드에서 선택한 영수증 건수(실시간 반영).

### M-021 영수증 등록
- **모드(등록/수정)**: 등록 모드(FAB 진입)는 타이틀 `영수증 등록`·버튼 `등록하기`, 수정 모드(상세 [수정] 진입)는 `영수증 수정`·`수정하기`. 저장 시 토스트도 모드에 따라 `영수증이 등록되었습니다` / `영수증이 수정되었습니다`.
- **영수증 이미지**:
  - **이미지 선택 팝업**: [카메라 촬영]은 단말 **카메라 앱**(`<input capture>`), [갤러리 선택]은 단말 **사진 보관함(갤러리)**(`<input>` 일반)을 연다.
  - **등록 모드**: FAB → [영수증 등록] → 팝업에서 카메라/갤러리로 **이미지를 실제 선택한 뒤** 영수증 등록 화면으로 진입한다(선택 취소 시 진입하지 않음). 진입 후 이미지는 **미리보기 썸네일 + [변경]** 으로 표시.
  - **수정 모드**: 진입 즉시 기존 등록 영수증 이미지를 **미리보기 썸네일 + [변경]** 으로 표시. [변경] 탭 → 카메라/갤러리 팝업으로 재선택(현재 화면 유지, 썸네일 갱신).
  - 썸네일 탭 시 이미지 확대 보기.
- **사용일시(필수)**: 필드는 `YYYY.MM.DD HH:mm` 형식으로 현재 선택값 표시. 탭 시 바텀시트 호출.
  - **상단 달력**: 월 이동(‹ ›), 날짜 그리드(주말 색 구분, 선택일 강조). 과거·현재·미래 날짜 모두 선택 가능.
  - **하단 아날로그 시계(24시간제)**: 상단의 `시 : 분` 숫자로 현재 값 표시. **시 모드** — 바깥 링 1~12시, 안쪽 링 13~24시(자정/24시는 `00` 표기, 12시는 `12` 표기). **분 모드** — 0~55분, 5분 단위. 시를 선택하면 자동으로 분 모드로 전환되며, 상단 `시`/`분` 숫자 탭으로 수동 전환 가능.
  - `[확인]` 탭 시 선택값을 필드에 반영.
- **금액(필수)**: 원 단위 숫자, 최대 12자리, 3자리 콤마 자동 삽입, 0원 등록 불가.
- **사용처(필수)**: 방식 칩(직접입력 기본 / 거래처).
  - **직접입력**: 명칭(최대 100자) + 사업자번호(선택·숫자 10자리, 직접 입력).
  - **거래처**: 거래처명 검색 필드 + **사업자번호 필드(비활성·읽기 전용)**. 검색 레이어에서 경리나라 거래처(명칭+사업자번호) 선택 시 거래처명·사업자번호가 각 필드에 자동 입력된다. 사업자번호 필드는 비활성으로 직접 수정 불가(동일명 거래처를 사업자번호로 식별). 거래처명을 다시 입력하면 사업자번호는 초기화.
- **용도(선택)**: 드롭다운 단일 선택(시스템 설정 항목).
- **내용(선택)**: 다중 라인 텍스트, 사용 내역 설명.
- **지급처(선택)**: 방식 칩(직원 기본 / 거래처 / 기타). 직원·거래처는 검색 레이어에서 경리나라 데이터 선택, 기타는 직접입력(최대 100자). 사용처가 직접입력이면 거래처 칩 미표시.
- **지급수단(필수)**: 위 "지급수단별 경리나라 연동 매핑" 참조. 선택값에 따라 등록 시 연동되는 경리나라 장부가 결정된다.

### M-022 영수증 상세
- **상태 정보**: (위에서부터) **결의/결재 상태 뱃지 → 금액(크게, 원·콤마) → 사용처 명칭**. 뱃지 값은 결의서/전자결재 진행 여부에 따라 **`미결의`**(앰버·wait) 또는 **`결의완료`**(그린·done)로 표시한다(결의/결재 전=미결의, 결의서 포함·전자결재 진행/완료=결의완료).
- **[결의서 작성] 버튼**: 미결의 건은 활성. **결의완료 건은 비활성화**하여 중복 결의를 방지한다.
- **상세 정보 목록**: 사용일시(`YYYY.MM.DD HH:mm`) / 사용처(방식·명칭·사업자번호) / 용도 / 지급처 / **지급수단** / 사용금액. 모두 등록 시 입력값(경리나라 매입 기준) 표시.
- **내용**: 등록 시 입력했을 경우 표시.
- **처리 이력 영역 없음**: 타임라인 형태의 처리 이력은 표시하지 않는다.
- **[수정]·[삭제] 버튼**: 미결의 건은 활성. **결의완료(지출결의·간편지급 처리 완료) 건은 비활성화**(흐림·클릭 불가)하여 수정·삭제를 막는다(알림 팝업 없이 버튼 자체 비활성).
- **영수증 이미지 전체 화면 미리보기**: 영수증 이미지를 탭하면 화면 전체(상단바·탭바 위, 어두운 배경)를 덮는 미리보기 오버레이로 이미지를 크게 표시한다. 오버레이가 탭바·[뒤로]보다 위에 위치하므로 **미리보기가 열린 동안에는 탭·뒤로가기로 화면이 전환되지 않으며**, 화면 전환이 일어날 경우에도 미리보기는 자동으로 닫힌다. **우측 하단 [저장](이미지 다운로드) 버튼, 우측 상단 [×] 닫기 버튼**을 제공하며 닫기는 [×]로만 수행한다(배경 탭으로는 닫히지 않음). (등록/수정 화면에서 선택한 실제 이미지가 있으면 그 이미지를, 없으면 placeholder를 표시)
- **하단 버튼명**: 마지막 버튼은 항상 `[결의서 작성]`(전자결재 ON/OFF 무관). 제출 버튼명 분기는 M-023에서 처리.

### M-023 결의서 작성 / 결재요청
- **전자결재 설정 기준**: 화면 진입 시 경리나라 지출결의서 전자결재 설정(ON/OFF)에 따라 타이틀·결재선 표시·제출 버튼이 분기된다.
- **우측 상단 '전자결재 ON/OFF' 토글 — ⚠️ 프로토타입 미리보기 전용 / 개발 범위 제외**: 이 토글은 검수자가 ON·OFF 두 상태를 한 화면에서 비교 확인하도록 만든 **상품·개발 편의용 장치**이며, **고객에게 제공되는 기능이 아니므로 실제 개발에서는 구현하지 않는다**. 실서비스에서는 사용자가 전환할 수 없고, 화면 상태는 경리나라 전자결재 설정값에 따라 고정된다.
- **화면 타이틀**: 전자결재 ON → "결재요청" / OFF → "결의서 작성".
- **제목(필수)**: 최대 100자 단일 라인. 진입 시 기본값 `지출결의서(간이영수증) {N}건, {#,###}원`(N=영수증 건수, 금액=총 합계) 자동 입력, 수정 가능. **내용(필수)**: 최대 1,000자 다중 라인, 카운터 표시.
- **영수증 내역 섹션**: 선택된 영수증별 **일자(`YYYY.MM.DD`만, 시·분 미표시)·사용처·용도·금액**(읽기 전용).
- **합계 금액**: 선택 영수증 금액 자동 합산(원·콤마).
- **결재선**(전자결재 ON 시만 표시): 자동 지정·[결재선 변경] 팝업 등 동작은 **M-000 §6** 따름. (OFF 시 결재선 섹션 미표시는 영수증 고유)
- **제출 버튼명**: 전자결재 ON → `[상신하기]`(경리나라 결재함 상신, 완료·반려 시 푸시 알림 M-050) / OFF → `[저장하기]`(경리나라 지출결의서관리 전송).
