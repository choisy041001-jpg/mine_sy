## 1. 화면 셸 공통

### 1.1 상단바

| 영역 | 요건 |
|------|------|
| 좌측 | 하위 화면은 [← 뒤로] 노출, 탭 진입 메인 화면은 화면 타이틀/로고 노출 |
| 가운데 | 현재 화면 타이틀(또는 채널명) |
| 우측 | 화면별 액션. 알림(🔔, 미읽음 시 뱃지) / 더보기(⋯) / 검색·필터(≡) 등 화면별로 노출 |

### 1.2 하단 탭바

| 항목 | 요건 |
|------|------|
| 구성 | 하단 고정 5개 탭: **게시판 · 연차 · 영수증 · 증명서 · 더보기** |
| 기본 진입 | 로그인 후 기본 진입 화면은 게시판 홈(M-002) |
| 선택 표시 | 현재 위치한 탭을 강조 표시. 하위 화면(상세·작성 등)에서도 소속 탭 강조 유지 |
| 노출 범위 | 로그인 후 메인 흐름에서 표시. 로그인·팝업 오버레이 등에서는 미표시 |

### 1.3 FAB(주요 액션 버튼)

| 항목 | 요건 |
|------|------|
| 위치·형태 | 화면 우하단 고정(+). 탭 시 세부 액션 목록 표시 또는 해당 작성 화면으로 직접 이동 |
| 노출 화면 | 게시판 글 작성·연차 신청/취소·영수증 등록 등 작성 액션이 있는 목록 화면 |
| 숨김 규칙 | **인사 FAQ 채널(M-006)** 등 읽기 전용 화면에서는 FAB을 노출하지 않는다 |

### 1.4 반응형 — 태블릿 분할

| 항목 | 요건 |
|------|------|
| 기준 폭 | 뷰포트 너비 **≥ 900px**(태블릿)에서 목록↔상세를 **좌우 50:50 분할** 표시. **< 900px**(모바일)는 화면 전환 + [← 뒤로]. *아이패드 미니 8.3" 세로(744px)는 900px 미만이라 모바일 동작, 가로(1133px)는 분할* |
| 적용 흐름 | 게시판 홈→게시글 상세(좌 목록·우 상세), 인사 FAQ 목록→상세 등. 상세(우) 영역은 뒤로가기 미표시, 하단 탭바는 좌측 영역에만 표시 |
| 탭 이동 | 분할 중 다른 탭 이동 시 분할 해제 후 전체 배율로 복원 |

> 분할 패턴·기준폭 등 구현 세부는 퍼블리싱 규칙 `rules_LAYOUT — 태블릿 분할 레이아웃`을 따른다.

| 빈 상태 | 목록/검색 결과가 없을 때 일러스트 + 안내 문구 표시 |

| 로딩 중 버튼 | API 호출 중 해당 버튼 비활성화 + 스피너 표시(중복 제출 방지) — 마스터 1.4 정합 |
| 처리 이력 타임라인 | 신청·결재/발급 등 처리 단계를 시간순으로 표시. 각 단계 = 단계명 + `{YYYY.MM.DD HH:mm} · {담당자}`. 연차·증명서·영수증 상세에서 공통 사용 |
| 파일 첨부 | [첨부] 탭 시 **첨부 방식 선택 바텀시트**(`카메라 촬영하기` / `앨범에서 선택하기` / `파일 선택하기`)로 디바이스 카메라·갤러리·파일 선택기를 호출. 허용 형식·개수·용량은 화면별 정의(예: 게시판 글 작성, 영수증 등록). 첨부 목록은 이미지=썸네일·문서=파일명/형식 아이콘으로 미리보기하고 개별 × 삭제 |

### 2.1 처리 상태 상세 공통 패턴

신청/처리 결과를 보여주는 **진행중 / 완료(승인·발급) / 반려** 3개 상세 화면은 **동일 레이아웃을 공유**하며, 상태에 따라 아래 요소만 달라진다. (적용: 연차 M-015~017 · 증명서 M-033~035 · 영수증 상세)

| 구성 | 공통 | 상태별 차이 |
|------|------|------------|
| 상단 | [← 뒤로] + 타이틀 + 상태 뱃지(§2 상태 뱃지) | 뱃지 색·타이틀 문구 |
| 상태 배너 | 상단 안내 배너 | 진행중/완료/반려 문구·색상 |
| 본문 | 신청·처리 정보(키-값), 처리 이력 타임라인 | 완료=결과 정보 영역 / 반려=**반려 사유 박스**(박스 내 `반려 사유` 뱃지 + 담당자 메시지 + `{처리 일시} · {담당자}` — 별도 섹션 라벨 없이 박스 안에 뱃지로 표기) |
| 하단 | — | 완료만 액션 버튼(예: 저장·전송), 진행중·반려는 **읽기 전용** |

> 모든 상세 화면은 종료 상태(완료·반려)에서 **편집 액션 없이 조회 전용**으로 동작한다(§5 읽기 전용 상세 정합).
> **배경**: 상세 본문은 흰색(surface)을 기본으로 하되, 현황/통계 등 일부 영역은 배경을 비워(앱 기본 배경) 시각적으로 구분할 수 있다.

## 5. 데이터 표시 공통 규칙

### 5.1 데이터 출처 공통 원칙 ★

> 모바일 전 화면의 표시 데이터는 **경리나라 및 그룹웨어와 연동**되며, **현재 로그인한 사업장 · 로그인 직원** 기준으로 표시한다.

| 항목 | 규칙 |
|------|------|
| 연동 기준 | 연차·증명서·영수증·인사정보·사업장 등은 **경리나라**, 게시판·게시글·댓글 등은 **그룹웨어**와 연동. 모든 데이터는 **현재 로그인 사업장·직원** 기준으로만 표시(타 사업장·미소속/미권한 데이터 미노출) |
| 역할 분담 | MY인사 모바일은 **조회·신청 상신·화면 전환 중심**. 실제 처리(결재 승인/반려, 증명서 발급/반려, 장부 기록, 게시판 운영 등)는 **경리나라·그룹웨어** 각 원천 시스템에서 수행 |
| 그룹 문서 기술 | 각 그룹 화면설계서는 **자기 출처만** 명시: 게시판=**그룹웨어 게시판**, 연차=경리나라 결재함, 증명서=경리나라 직원관리/업무신청, 영수증=경리나라 매입·지출결의서. 공통 원칙은 본 절을 참조 |

### 5.2 부서(department) 표시 — 선택 항목 (공통 요건) ★

> **요건**: 부서는 경리나라 **직원관리에서 필수 등록 항목이 아니므로 부서명이 없는 직원이 존재할 수 있다.** 따라서 부서 정보가 없는 경우 부서 표시는 **공란 처리하거나 생략**되어야 하며, 부서 미입력으로 인해 빈칸·`undefined`·`null` 등이 노출되지 않도록 한다.

| 표시 위치 | 부서 없음 처리 |
|---------|----------------|
| 댓글 작성자 (M-004 게시글 상세 · M-008 인사 FAQ 상세) | 작성자 이름 옆 부서 표기를 **생략**(아바타 없음, 이름만 표시) |
| 프로필 요약 (M-040 더보기 · M-041 내 정보) | `직급 · 부서`에서 부서가 없으면 직급만 표시. 인사 정보 목록의 부서 행은 **공란** 또는 생략 |
| 연결형 메타 문자열 `{이름} ({부서} · {직급})` | 부서가 없으면 **부서 토큰과 구분자(· )를 함께 생략** → `{이름} ({직급})` |

**조건부 규칙**
- 부서는 표시용 정보일 뿐 필터·정렬·집계의 기준으로 사용하지 않는다.
- 부서 미입력 직원도 모든 목록·집계에 정상 포함된다(부서 없음이 누락 사유가 되지 않음).

### 5.3 기타 표시 규칙

| 항목 | 규칙 |
|------|------|
| 날짜(목록·카드) | `YYYY.MM.DD`. 당일 → "오늘", 전일 → "어제". 시각 필요 시 `YYYY.MM.DD HH:mm` — 마스터 1.4 정합 |
| 날짜(상세·이력) | `YYYY년 MM월 DD일 HH:mm` — 마스터 1.4 정합 |
| 날짜 섹션 헤더(목록 그룹) | 알림 등 날짜 그룹 헤더는 **오늘 / 어제 / M월 D일 (요일)** |
| 텍스트 오버플로우 | 목록·카드·셀은 1줄(제목 등) 또는 2줄(FAQ 질문 등) 말줄임. 길게 누르기(500 ms) 시 전체 내용 툴팁 — 마스터 1.4 정합 |
| 읽기 전용 상세 | 반려·완료 등 종료 상태 상세 화면은 편집 액션 없이 조회 전용으로 동작 |

## 6. 전자결재·결재선 공통

> 연차 신청/취소(휴가신청서)·경비 결의서 등 **전자결재 상신 화면이 공유하는 결재선·상신 동작 규칙**을 정의한다. 각 화면은 상신되는 경리나라 문서 종류만 별도 명시한다.

| 항목 | 요건 |
|------|------|
| 상신 처리 | 신청·상신은 경리나라 **전자결재 문서로 등록**되며, 실제 승인·반려는 경리나라 **결재함 앱**에서 처리된다. MY인사는 결재 결과를 받아 상태를 표시(읽기 중심) |
| 결재선 표시 | `신청자 → (결재/접수 단계…)` 순서로 표시. 결재선은 **경리나라 기준으로 자동 지정**되며 안내 문구(`결재선은 경리나라를 기준으로 자동 지정됩니다`) 노출. **각 단계는 결재(결재자) 또는 접수(접수자)일 수 있고, 다단계(예: 신청 1 + 결재·접수 최대 여러 단계)일 수 있으므로** 단계 수가 많아도 깨지지 않게 표시(가로 스크롤 등). 각 단계에 역할(신청/N차 결재/접수)과 상태(상신완료·진행중·승인·반려·접수완료·대기)를 함께 표기 |
| 결재선 변경 | 등록 결재선이 **2개 이상**인 경우에만 [결재선 변경] 노출 → 탭 시 라디오 버튼 팝업에서 선택(취소/확인). 1개면 변경 없이 고정. **팝업의 각 결재선 옵션은 단계별 역할(결재/접수)을 구분 표시**(예: `김팀장(결재) → 박차장(접수)`) |
| 결재 결과 알림 | 결재 완료·반려 시 앱 푸시 발송(§4) → 알림 탭 시 해당 기능 메인 화면으로 이동 |

> **문서 매핑(각 그룹 문서 소관)**: 연차 신청/취소 → **휴가신청서**(신청 구분 신청/취소, 휴가 종류 항상 `연차`) / 경비 결의서 → **지출결의서**(경리나라 지출결의서 전자결재 ON일 때 상신). 매핑 상세는 해당 그룹 화면설계서 참조.
