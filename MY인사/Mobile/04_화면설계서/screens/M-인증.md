## 화면 상세 설계

### M-001 로그인

| 항목 | 내용 |
|------|------|
| **화면 목적** | 사용자가 아이디·비밀번호로 인증하고, 소속 사업장을 선택하여 앱에 진입한다. |
| **진입 조건** | ① 앱 최초 실행 시 ② 로그아웃 후 ③ 세션 만료(자동 로그인 미설정 시 앱 재실행) 시 → 하단 탭바 미표시 |
| **다음 화면** | 사업장 선택 팝업 M-001-P |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 로고 영역 | 앱 아이콘 + 앱 이름 | 파란 원형 아이콘 + "그룹웨어" |
| 입력 영역 | 아이디 입력 필드 | placeholder "아이디 또는 이메일" / 최대 100자 / 영문·숫자·특수문자 |
| 입력 영역 | 비밀번호 입력 필드 | 텍스트 가림 처리 + 우측 표시/숨기기 토글 아이콘 / 최대 50자 |
| 옵션 영역 | 자동 로그인 체크박스 | 기본값: 미체크 |
| 액션 영역 | [로그인] 버튼 | 화면 전체 너비 |
| 액션 영역 | 아이디 찾기 링크 | 버튼 하단. 탭 시 M-009(아이디 찾기 모드)로 이동 |
| 액션 영역 | 비밀번호 변경 링크 | 버튼 하단. 탭 시 M-009(비밀번호 변경 모드)로 이동 |

**사용자 흐름**

1. 아이디 + 비밀번호 입력
2. [로그인] 버튼 탭
   - 인증 성공 → 사업장 선택 팝업(M-001-P) 표시
   - 인증 실패 → 에러 메시지 표시 (화면 유지)
3. 사업장 선택 → 게시판 홈(M-002) 진입
- [아이디 찾기] 탭 → M-009 (아이디 찾기 모드)
- [비밀번호 변경] 탭 → M-009 (비밀번호 변경 모드)

**조건부 규칙**
- 자동 로그인 체크 시: 다음 앱 실행 시 로그인 화면을 건너뛰고 바로 진입한다. **여러 사업장에 로그인할 수 있는 계정인 경우, 가장 최근에 로그인했던 사업장으로 자동 로그인**한다(사업장 선택 팝업 미표시)
- 비밀번호 표시/숨기기: 토글 아이콘 탭마다 전환
- **기기 알림(디바이스 푸시) 탭으로 앱 진입 시**: 알림에 담긴 사용자/사업장과 자동로그인 설정 여부에 따라 진입을 분기한다.
  - 자동로그인 설정됨 + 알림 사업장 = 자동로그인 사업장 → 자동 로그인 후 **알림 메뉴(M-050)로 이동**.
  - 자동로그인 미설정 → 로그인 화면 표시. 로그인한 사용자/사업장이 **알림의 사용자/사업장과 일치하면 알림 메뉴(M-050)로 이동**, **다른 계정/사업장으로 로그인하면 게시판 홈(M-002) 유지**(알림 메뉴 이동 안 함).
  - **이미 로그인된 상태에서 다른 계정/사업장의 알림을 탭하면 강제 로그아웃 → 로그인 화면(M-001)으로 전환**한 뒤, 재로그인한 사용자/사업장이 알림과 일치하면 알림 메뉴(M-050), 다르면 게시판 홈(M-002)을 유지한다.
  - (상세 분기표는 **M-더보기_알림 M-050 "기기 알림(디바이스 푸시) 탭 → 앱 진입 분기"** 참조)

**유효성 규칙**

| 오류 조건 | 오류 표시 위치 | 오류 메시지 | 처리 결과 |
|---------|-------------|-----------|---------|
| 아이디 또는 비밀번호 불일치 | 비밀번호 입력 필드 하단 | "아이디 또는 비밀번호가 올바르지 않습니다" | 비밀번호 필드 초기화, 화면 유지. 실패 횟수 누적 |
| 로그인 5회 연속 실패 | 비밀번호 입력 필드 하단 | "로그인 시도 횟수를 초과하였습니다. 계정이 잠겼습니다. 관리자에게 문의하세요" | 계정 잠금 (경리나라 로그인 정책 연동). 잠금 해제 전까지 로그인 불가 |

### M-001-P 사업장 선택 팝업 `[팝업 오버레이]`

| 항목 | 내용 |
|------|------|
| **화면 목적** | 로그인 성공 후, 해당 계정에 연결된 사업장 중 하나를 선택하여 해당 사업장 기준으로 앱을 사용한다. |
| **진입 조건** | 로그인 버튼 탭 → 인증 성공 시 현재 화면 위에 오버레이로 표시 |
| **다음 화면** | 게시판 홈 M-002 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 헤더 영역 | 소제목 | "로그인" |
| 헤더 영역 | 제목 | "사업장을 선택하세요" |
| 헤더 영역 | 부제 | "{이름} 님이 소속된 사업장 목록입니다" |
| 사업장 목록 | 항목 | 사업장명 / 사업자번호 · 가입 제품명(we경리나라 등) + 우측 화살표(→) |
| 하단 버튼 | [취소] 버튼 | 팝업 닫기, 로그인 화면 유지 |

**사용자 흐름**
- 사업장 항목 탭 → 해당 사업장으로 로그인 → 게시판 홈(M-002) 이동
- [취소] 탭 → 팝업 닫힘, 로그인 화면 유지 (재로그인 가능)

**조건부 규칙**
- 소속 사업장이 1개뿐인 경우: 팝업 없이 바로 해당 사업장으로 진입 (UX 간소화)
- 소속 사업장이 2개 이상인 경우: 팝업 표시
- 목록이 일정 높이 초과 시: 내부 스크롤 처리

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시·검증하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 공통 입력 UI 규칙 참조)
> **데이터 출처**: 로그인 인증·계정 잠금·아이디/비밀번호 찾기는 **경리나라 로그인 정책**에 연동된다. 사업장 목록은 **경리나라**에서 해당 계정에 연결된 사업장이다. 그룹웨어는 인증 결과만 받아 화면을 전환한다.

### 1. M-001 로그인

- **로고 영역**: 파란 원형 아이콘 `MY` + 앱 이름 `그룹웨어`(굵게) + 서브 카피 `중소기업용 통합 솔루션`. 정적 표기값이며 데이터 연동 없음.
- **아이디 입력 필드**(`#loginIdInput`): placeholder `아이디 또는 이메일`. 로그인 아이디는 **경리나라 계정의 아이디**로 인증한다. 아이디는 원칙적으로 이메일 형식이나, **이메일 구조가 아닌 아이디를 사용하는 사용자도 존재**하므로 아이디·이메일을 모두 입력 가능하도록 안내한다. 최대 100자, 영문·숫자·특수문자 허용. (프로토타입은 데모용으로 `gd.hong` 프리필) ※ 모바일 인증은 **사번을 사용하지 않는다**.
- **자동 로그인 체크박스**: **기본값 미체크**. 체크 시 다음 앱 실행에서 로그인 화면을 건너뛴다. **여러 사업장에 로그인할 수 있는 계정인 경우, 다음 실행 시 사업장 선택 팝업 없이 가장 최근에 로그인했던 사업장으로 자동 로그인**한다.
- **로그인 오류 안내 박스**(`#loginLockMsg`): 평소 숨김. **5회 연속 실패** 시 비밀번호 필드 하단에 danger 박스로 `로그인 시도 횟수를 초과하였습니다. 계정이 잠겼습니다. 관리자에게 문의하세요.` 노출. 계정 잠금은 경리나라 로그인 정책에 연동된다.
- **[로그인] 버튼**: 화면 전체 너비. 탭 시 인증 → 성공이면 사업장 선택 팝업(M-001-P) 표시. 인증 실패 시 비밀번호 필드 하단/토스트로 `아이디 또는 비밀번호가 올바르지 않습니다` 표시, 실패 횟수 누적.
<!-- [개발자 선확인 필요] 위 로그인 오류 메시지 문구는 임시(예시) 기준이다. 실제로는 **비즈나라 인증 API의 오류 응답 메시지를 그대로 알림에 표시**하는 방식일 수 있으므로, 개발 전 비즈나라 인증 API 오류 응답 스펙(에러 코드·메시지 필드)을 먼저 확인할 것. -->
- **하단 링크**: `아이디 찾기`(→ M-009 아이디 찾기 모드) / `비밀번호 변경`(→ M-009 비밀번호 변경 모드).

### 2. M-001-P 사업장 선택 팝업

- **헤더**: 소제목 `로그인` / 제목 `사업장을 선택하세요` / 부제 `{이름} 님이 소속된 사업장 목록입니다`. `{이름}`은 인증된 계정의 사용자명(경리나라 사용자관리).
- **사업장 목록 항목**(경리나라 — 해당 계정에 연결된 사업장 목록): 항목 표시 형식은 **M-000 §3 팝업 항목 형식** 공통(사업장명 + `{사업자번호} · {가입 제품명}` + 화살표). 가입 제품명은 해당 사업장의 경리나라 제품 표기(예: `we경리나라`).
  - 예시 데이터: `(주)웹케시 / 123-45-67890 · we경리나라`, `웹케시테크(주) / 234-56-78901 · we경리나라`, `웹케시서비스(주) / 345-67-89012 · we경리나라`.
- **표시 규칙**: 연결 사업장이 **1개면 팝업 없이 바로 진입**, **2개 이상이면 팝업 표시**. 목록이 길면 내부 스크롤.
- **항목 탭**: 해당 사업장 기준으로 로그인 확정 → 게시판 홈(M-002) 이동. **[취소]**: 팝업 닫고 로그인 화면 유지.

### 3. M-009 아이디 찾기 / 비밀번호 변경

<!-- [개발자 선확인 필요] 아이디 찾기 / 비밀번호 변경(찾기)은 경리나라/그룹웨어가 아니라 **비즈나라(인증 플랫폼) 제공 화면을 그대로 사용(웹뷰/리다이렉트)하고 있을 확률이 높다.** 아래 입력 항목·검증·결과 팝업 설계는 추정 기준이며, 개발 전 ① 자체 구현 vs 비즈나라 화면 재사용 여부 ② 진입/반환 방식·UI 커스터마이징 범위 ③ 인증 수단 종속성을 먼저 확인할 것. 확인 결과에 따라 본 설계가 대체될 수 있다. -->

- **헤더**: ×(닫기 → M-001 복귀) + **타이틀**(`#pfMainTitle`) 진입 모드에 따라 `아이디 찾기` / `비밀번호 변경` 동적 표시.
- **인증 방식 탭**: `휴대폰` / `이메일`. 탭 전환 시 해당 인증 폼 패널로 전환(별도 모드 전환 없음, 진입 시 모드 고정).
- **입력 필드 형식·검증**
  - 성명: 직접 입력(공통).
  - 아이디: **비밀번호 변경 모드에만** 노출, 직접 입력.
  - 휴대폰번호: 국가코드 `+82` 고정(선택 불가, 드롭다운 없음) + 번호. 우측 **[인증]** 버튼 탭 시 인증번호 발송 → 토스트 `인증번호가 발송되었습니다`.
  - 이메일: 직접 입력(이메일 인증 방식).
  - CAPTCHA: 이미지 텍스트(예: 아이디찾기 `mY4n5a` / 비번변경 `fCa3d9`) + **새로고침** 버튼(탭 시 토스트 `새 CAPTCHA가 생성되었습니다`) + 텍스트 직접 입력 필드.
  - 인증번호: 발송된 번호 입력 + 우측 **[확인]** 버튼(탭 시 토스트 `인증되었습니다`).
  - **입력 누락 검증**: [찾기]/[변경] 탭 시 폼의 필수 항목을 **위에서부터** 검사하여 첫 미입력 항목에 대해 팝업 `{항목명}이(가) 입력되지 않았습니다.`(조사는 받침에 따라 이/가) 표시 후 동작 중단. 항목명 — 휴대폰 탭: `성명`·`휴대폰번호`·`자동입력 방지문자`·`인증번호`, 이메일 탭: `성명`·`이메일`, 비밀번호 변경에는 `아이디` 추가.
- **실행 버튼**
  - 아이디 찾기 **[찾기]**: 별도 토스트 없이 곧바로 **결과 팝업** 표시.
  - 비밀번호 변경 **[변경]**: 입력한 **아이디·성명·휴대폰/이메일**의 계정 일치 여부 검증. **일치** 시 새 비밀번호 설정(M-009-S)으로, **불일치** 시 비밀번호 변경 결과(M-009-R)로 이동. (임시 비밀번호 발송 방식 아님)
  - 새 비밀번호 설정 **[변경하기]**(M-009-S): `비밀번호`/`비밀번호 확인` 검증 통과 시 팝업 `변경이 완료되었습니다.` → [확인] 시 로그인(M-001) 이동. 변경 불가 시 사유 팝업 표시 → [확인] 시 화면 유지.
  - 비밀번호 변경 결과(M-009-R): `[입력정보]`(아이디/이름/휴대폰 또는 이메일)는 직전 화면에서 사용자가 입력한 값을 그대로 표시. [다시찾기] → M-009 복귀.
  - **[취소]**: M-001 복귀.
- **아이디 찾기 결과 팝업**: 제목 `아이디 찾기 결과` / 안내 `회원님의 아이디는` / **아이디 값**(예: `gd.hong`) / **`가입일: {YYYY.MM.DD}`**(예: `2024.03.15`) / 버튼 [닫기] · [로그인]. 아이디·가입일은 경리나라 계정 정보 기준.

### 4. M-009-S 새로운 비밀번호 설정

- **타이틀/안내**: `새로운 비밀번호 설정하기`(accent 밑줄) + 안내 `비밀번호를 재설정하여 주세요.` — 정적 표기값.
- **비밀번호**(`#newPw`): `type=password` 가림 처리, 최대 50자. *정식 구현 시 경리나라 비밀번호 정책(길이·문자 조합)을 적용. 프로토타입은 입력·일치 검증만 수행.*
- **비밀번호 확인**(`#newPwConfirm`): `type=password`, 위 비밀번호와 **동일해야** 통과.
- **[변경하기]**: 검증 — ① 미입력 시 팝업 `비밀번호를 입력해주세요.`, ② 불일치 시 팝업 `비밀번호가 일치하지 않습니다.`(둘 다 [확인] 시 화면 유지·재입력). 통과 시 팝업 `변경이 완료되었습니다.` → [확인] 탭 시 로그인(M-001) 이동. 실제 비밀번호 변경은 **경리나라 로그인 정책**에 반영된다.
- **[닫기]** → M-001 / **[이전 페이지]**(링크) → M-009 비밀번호 변경 화면.

### 5. M-009-R 비밀번호 변경 결과 (정보 불일치)

- **타이틀**: `비밀번호 변경 결과`(accent 밑줄). 진입 = M-009 비밀번호 변경 [변경] 시 **아이디·성명·휴대폰/이메일이 일치하는 계정이 없을 때**.
- **안내 문구**(정적): `입력하신 정보와는 일치하는 사용자가 없습니다. 본인의 사용자 정보가 맞다면 다음과 같은 이유로 오류가 표시될 수 있습니다.`
- **오류 사유 목록**(정적): `아이디 입력 시 대/소문자가 잘못 입력된 경우` / `아이디 뒤에 자동으로 공란이 들어간 경우`.
- **[입력정보]**(직전 M-009 비밀번호 변경 화면에서 사용자가 입력한 값 그대로 표시): `아이디` / `이름`(성명) / 인증 방식에 따라 **`휴대폰`(국가코드 `+82` + 번호)** 또는 **`이메일`** — 라벨·값은 진입 시 사용한 인증 방식(휴대폰/이메일)에 따라 동적.
- **[다시찾기]** → M-009 비밀번호 변경(재입력) / **[취소]·×(닫기)** → M-001.
