## 화면 상세 설계

### M-030 증명서 관리

| 항목 | 내용 |
|------|------|
| **화면 목적** | 증명서 종류를 선택하여 신청하고, 최근 발급 이력을 인라인으로 조회한다. |
| **진입 조건** | 하단 탭바 [증명서] 탭 |
| **다음 화면** | M-031 / M-032 / M-033 / M-034 / M-035 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 + 부제 | "증명서 관리" / "증명서 발급 신청 및 이력 조회" |
| 증명서 종류 카드 | 재직증명서 | 아이콘 + 설명 문구 |
| 증명서 종류 카드 | 경력증명서 | 아이콘 + 설명 문구 |
| 증명서 종류 카드 | 기타증명서 | 아이콘 + 설명 문구 |
| 발급 이력 | 섹션 헤더 | "발급 이력" + [전체 보기] (우측, → M-032) |
| 발급 이력 | 이력 행 (구분선형 · 전체 너비) | 종류 뱃지 / 용도 / 신청·발급·반려 일시 / 상태 뱃지 / 화살표 |

**사용자 흐름**
- 종류 카드 탭 → `openCertModal()` 호출 → 바텀 시트 모달(M-031) 표시
- [전체 보기] 탭 → M-032 발급 이력 전체 목록
- 이력 행 탭: 진행중 → M-033 / 발급완료 → M-034 / 반려 → M-035

**조건부 규칙**
- 발급 이력 없을 경우: "신청한 증명서 내역이 없습니다" 빈 상태 표시

### M-031 증명서 신청

| 항목 | 내용 |
|------|------|
| **화면 목적** | 선택한 종류의 증명서를 신청한다. |
| **진입 조건** | 증명서 관리(M-030) → 종류 카드 탭 → `openCertModal()` 호출. 별도 화면 전환 없이 현재 화면 위에 바텀 시트 오버레이로 표시됨 |
| **다음 화면** | 증명서 상세 — 진행중 M-033 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 핸들 바 | — | 드래그 핸들 |
| 헤더 | 종류명 소제목 + "발급 신청" 타이틀 + 닫기(×) 버튼 | |
| 입력 폼 | 증명서 명칭 | 기타증명서일 때만 표시 — **필수** / 최대 50자 / 단일 라인 |
| 입력 폼 | 용도 입력 | **필수** / 최대 100자 / 단일 라인 / placeholder "예: 은행 제출용" |
| 안내 배너 | 처리 안내 | 예상 처리 시간 + 발급 완료 시 앱 알림 안내 |
| 하단 | [발급하기] 버튼 | 전체 너비 |

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 용도 미입력 | 해당 필드 오류 표시, 신청 불가 |
| 기타증명서 명칭 미입력 | 해당 필드 오류 표시, 신청 불가 |

**조건부 규칙**
- 증명서 명칭 필드: 종류가 "기타증명서"일 때만 노출. 재직·경력증명서는 숨김
- 닫기(×) 탭 또는 배경 탭 → 모달 닫힘, M-030 유지
- 유효성 통과 후 [발급하기] 탭 → 모달 닫힘 → M-033으로 이동

### M-033 / M-034 / M-035 증명서 상세 (진행중 · 발급완료 · 반려)

> 3개 화면은 동일한 레이아웃이며, 처리 상태에 따라 표시 내용과 하단 버튼만 다르다.

**공통 화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 상태별 타이틀 | |
| 상태 헤더 | 처리 상태 뱃지(진행중 주황 / 발급완료 초록 / 반려 빨강) + 증명서 종류 + 용도 | |
| 신청 정보 | 증명서 종류 / 용도 / 신청일시 / 담당자(인사 담당자) | |

**조건부 규칙**

| 화면 | 타이틀 | 추가 영역 | 하단 버튼 |
|------|-------|---------|---------|
| M-033 진행중 | "신청 상세" | 처리 이력 타임라인(접수→검토 중) + 안내 배너(통상 1~2 영업일) | 없음 (읽기 전용) |
| M-034 발급완료 | "발급 완료" | 발급 정보(발급일시·처리 담당자·직인 적용 여부) + 처리 이력 타임라인(접수→검토→발급) + 인사담당자 메모 | [이메일로 전송] · [저장 (다운로드)] |
| M-035 반려 | "반려 상세" | 반려 사유 박스 + 신청 정보 + 인사담당자 메모 + 처리 이력 타임라인(접수→반려) | 없음 (읽기 전용) |

> **[이메일로 전송] 탭 시**: 이메일 전송 바텀 시트 표시. 수신 이메일 직접 입력 후 [발송하기] 탭. (첨부 파일명·용량 함께 표시)

## 세부 설계 요건 — 데이터 표시 기준

> 화면별 표시 데이터의 출처·형식·연동 결과를 필드 단위로 정의한다. 직원이 모바일에서 제출한 증명서 신청 건은 관리자용 MY인사 PC의 **업무신청관리(P-003)** 목록에 표시되며, **인사 담당자**가 발급 또는 반려 처리를 수행한다. (별도 '인사팀' 부서로 신청하는 구조가 아니라, 인사 담당자가 직접 처리한다.)

### 공통 — 데이터 출처
- **증명서 종류**: 재직증명서 · 경력증명서 · 기타증명서 3종 고정. 기타증명서만 [증명서 명칭] 직접 입력.
- **신청자**: 로그인 직원 정보(이름).
- **처리 담당자**: 관리자 PC 업무신청관리(P-003)에서 발급/반려 처리한 **인사 담당자**(이름). 부서 단위가 아닌 인사 담당자 개인 기준이며, 별도 '담당부서' 개념은 사용하지 않는다.
- **발급 부수**: 별도 수량(부수/매수) 입력·표시 항목 **없음**. 1건 신청 = 1건 발급 기준.
- **상태값**: 진행중 / 발급완료 / 반려 3종. 상태 변경(발급완료·반려) 시 앱 푸시 알림 발송, 알림 행 탭으로 M-034·M-035 딥링크 진입.
- **일시 형식**: 목록은 `M월 D일 HH:mm`(연도 생략) 또는 `신청/발급/반려 M월 D일`, 상세는 `YYYY.MM.DD HH:mm`.

### M-030 증명서 관리
- 종류 카드: 종류명 + 한 줄 설명(고정 문구). 기타증명서 카드 설명은 "파일을 첨부하여 신청하는 증명서".
- 발급 이력(인라인): 최근 신청 건을 종류 뱃지 + 용도 + 신청/발급 일시 + 상태 뱃지로 표시. 부수 표기 없음. [전체 보기] → M-032.
- 이력 없을 경우: "신청한 증명서 내역이 없습니다" 빈 상태.

### M-031 증명서 신청 (바텀 시트 모달)
- **진입 UI**: 종류 카드 탭 시 `openCertModal(type)` 호출 — 현재 화면 위 바텀 시트 오버레이(별도 화면 전환 아님). 헤더에 선택 종류명 표기.
- **증명서 명칭(기타증명서 한정·필수)**: 최대 50자, 단일 라인. 재직·경력증명서는 미노출.
- **용도(필수)**: 단일 라인 입력. placeholder "예) 금융기관 제출, 병원 보험 청구 등".
- **처리 안내 배너**: 종류별 예상 처리 시간 문구. 재직·경력 "당일~1 영업일", 기타 "인사 담당자 검토 후 1~2 영업일". 발급 완료 시 앱 알림 안내 포함.
- **[발급하기]**: 유효성(용도 필수, 기타는 명칭 필수) 통과 시 신청 접수 → 모달 닫힘 → M-033 진행중 상세 이동. 부수 입력 없음.

### M-032 증명서 발급 이력
- 탭 필터(전체/진행중/발급완료/반려): 선택 상태에 해당하는 행만 노출.
- 이력 행: 종류 뱃지 + 용도(반려 시 "용도 · 사유 요약") + 처리 상태 텍스트(예: "인사 담당자 검토 중") + 일시 + 상태 뱃지. 부수 표기 없음.

### M-033 증명서 상세 — 진행중
- 상태 헤더: 진행중 뱃지(주황) + 종류 + 용도. 신청 정보: 종류·용도·신청일시(`YYYY.MM.DD HH:mm`)·담당자(인사 담당자).
- 처리 이력 타임라인: 신청 접수(신청자) → 인사 담당자 검토 중. 안내 배너: "통상 1~2 영업일 소요".

### M-034 증명서 상세 — 발급완료
- 상태 헤더: 발급완료(초록) + 안내 문구("아래 버튼으로 저장하거나 이메일로 받으세요").
- 발급 정보: 종류·용도·발급일시·처리 담당자(이름·소속)·직인 적용 여부(적용됨 ✓). 부수 표기 없음.
- 처리 이력 타임라인: 신청 접수 → 인사 담당자 검토 → 발급 완료(각 일시·담당자). 인사 담당자 메모(있을 경우).
- 하단: [이메일로 전송](바텀 시트 — 수신 이메일 직접 입력·첨부 파일명/용량 표시·[발송하기]) / [저장 (다운로드)](PDF 다운로드).

### M-035 증명서 상세 — 반려
- 상태 헤더: 반려(빨강) + 종류 + 용도. 반려 사유 박스: 담당자 메시지(첨부 누락 등) + 반려 처리 일시·소속.
- 신청 정보(종류·용도·신청일시) + 인사담당자 메모 + 처리 이력 타임라인(신청 접수 → 반려 처리).
- 하단 버튼 없음(읽기 전용). 재신청은 M-030에서 종류 카드로 다시 신청.
