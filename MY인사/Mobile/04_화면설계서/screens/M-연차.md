## 화면 상세 설계

### M-010 연차 관리

| 항목 | 내용 |
|------|------|
| **화면 목적** | 나의 잔여 연차 현황을 확인하고, 연차 이력 전체를 인라인 스크롤로 조회하며, FAB를 통해 연차 신청·취소를 작성한다. |
| **진입 조건** | 하단 탭바 [연차] 탭 |
| **다음 화면** | M-011 / M-014 / M-015 / M-016 / M-017 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 | "연차 관리" |
| 잔여 연차 카드 | 잔여 일수 | 크게 강조 표시 |
| 잔여 연차 카드 | 총 연차 · 사용 일수 · 진행 막대 | |
| 연차 이력 | 이력 행 (구분선형 · 전체 너비) | 문서명·종류·일수·상신일·결재자 + 상태 뱃지 + 화살표 / 날짜 역순 / 별도 섹션 헤더 없음 |
| 하단 | 연차 FAB(+) | 연차 신청 / 취소 신청 |

**사용자 흐름**
- 이력 행 탭: 진행중 → M-015 / 승인 → M-016 / 반려 → M-017
- FAB(+) 탭 → 액션 목록: [연차 신청] → M-011 / [취소 신청] → M-014

**조건부 규칙**
- 잔여 연차 0일: 현황 카드의 진행 막대 0% 표시
- 연차 이력 없을 경우: "신청한 연차 내역이 없습니다" 빈 상태 표시

### M-011 연차 신청

| 항목 | 내용 |
|------|------|
| **화면 목적** | 캘린더에서 날짜를 선택하고 휴가 종류·사유를 입력하여 연차 결재를 상신한다. |
| **진입 조건** | 연차 관리(M-010) FAB → [연차 신청] (**전체 화면 전환** — 바텀 시트 오버레이 아님) |
| **다음 화면** | 연차 관리 M-010 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "휴가신청서" | |
| 기간 입력 | 시작일 ~ 종료일 | 날짜 선택 필드 |
| 캘린더 | 이전/다음 월 이동 버튼 | |
| 캘린더 | 날짜 탭 또는 드래그로 범위 선택 | 선택 구간 하이라이트 표시 |
| 캘린더 | 오늘 날짜 구분 표시 | |
| 휴가 종류 | 단일 선택 토글 | 연차 / 오전 반차 / 오후 반차 — **필수** / 오전·오후 반차 0.5일 차감. **반차는 하루만 선택 가능** — 캘린더에서 기간을 먼저 지정한 뒤 반차로 변경하면 **시작일만 남기고** 알림 팝업 `반차는 하루만 선택해주세요.` 표시 |
| 사유 입력 | 텍스트 입력 영역 | **필수** / 최대 200자 / 다중 라인 / 카운터 표시 |
| 비상 연락망 | 텍스트 입력 필드 | 선택 / 최대 20자 / 숫자만 허용 / 하이픈 자동 삽입 (예: 010-1234-5678) |
| 잔여 연차 미리보기 | 신청 시 차감 후 잔여 일수 표시 | |
| 결재선 | 신청자 → 결재자 순서 표시 | 경리나라 자동 지정 안내 문구. **결재선이 2개 이상인 경우 [결재선 변경] 선택 시** 라디오 버튼으로 결재선 선택 팝업 호출 |
| 하단 | [신청하기] 버튼 | 전체 너비 |

**조건부 규칙 (이탈 처리)**
- [← 뒤로] 탭 (날짜·종류·사유 등 하나라도 입력된 경우): "작성 중인 내용이 있습니다. 취소하시겠습니까?" 확인 팝업 → 확인 시 이전 화면(M-010) 복귀 / 취소 시 팝업 닫기

**유효성 규칙**

| 오류 조건 | 오류 메시지 | 처리 결과 |
|---------|-----------|---------|
| 날짜 미선택 | "날짜를 선택하세요" | 진행 불가 |
| 잔여 연차 부족 | "잔여 연차가 부족합니다" 경고 | [신청하기] 비활성화 |
| 사유 미입력 | 해당 필드 오류 표시 | 진행 불가 |

## 세부 설계 요건 — 데이터 표시 기준

> 표시 데이터의 **출처·형식·표기 규칙**을 화면별로 정의한다. UI 레이아웃은 위 화면 상세 설계를, 결재 처리 흐름은 PRD를 참조한다.

### 공통 — 데이터 출처

> 데이터 출처 공통 원칙은 **M-000 §5.1**, 전자결재·결재선(자동 지정·[결재선 변경] 팝업·결재함 처리·결재 결과 푸시)은 **M-000 §6** 참조.

- **연차 데이터**(잔여 연차·사용 일수·연차 이력): 경리나라(결재함 등) 연동, **현재 로그인 사업장·직원** 기준. 실제 결재(승인·반려)는 경리나라 **결재함** 앱에서 처리하고 MY인사는 결과만 표시한다.

### 공통 — 경리나라 전자결재 상신 매핑 (연차 문서)

> 결재선·상신 일반 규칙은 M-000 §6. 본 절은 **연차의 문서 매핑**만 정의한다.

- **연차 신청(M-011)** → **휴가신청서**(`신청 구분`=**신청**, `휴가 종류`=항상 **연차**), 기간·일수=신청 화면 지정값.
- **연차 취소 신청(M-014)** → **휴가신청서**(`신청 구분`=**취소**, `휴가 종류`=항상 **연차**), 기간·일수=취소 화면 지정값.
- 취소 신청 날짜는 **승인된 연차로 제한하지 않으며**, 사용자가 지정한 기간·일수를 그대로 상신한다.
 
### M-010 연차 관리
- **타이틀/부제**: `연차 관리` / `연차 신청 및 사용 내역`.
- **잔여 연차 카드**: **`잔여 연차` 라벨을 카드 좌측 상단(대형 숫자 위)에 두고**, 그 아래 잔여 일수 대형 숫자(예 `12일`), 우측에 `총 연차 {N}일`·`사용 {N}일` 칩, 하단 진행 막대는 `사용/총` 비율. 잔여 0일이면 막대 0%.
- **연차 이력 행**(헤더·섹션 구분 없는 구분선형 — **별도 섹션 헤더 없음**, 날짜 역순): 1행 `{문서명}`(예 `휴가신청서`), 2행 **`{사용일자} {휴가 종류} · {사용 일수}`**(예 `6월 10일 연차 · 1일`, `5월 21일 오전 반차 · 0.5일`, 기간은 `5월 6일~7일 연차 · 2일`), 3행 `상신 {M월 D일} · 결재자 {결재자명}`, 우측 상태 뱃지(`진행중`·`승인`·`반려`) + 화살표. 이력 없으면 `신청한 연차 내역이 없습니다` 빈 상태.
- **FAB(+)**: `연차 신청`(→ M-011) / `취소 신청`(→ M-014).

### M-011 연차 신청
- **타이틀**: `휴가신청서`. 전체 화면 전환.
- **기간/캘린더**: 시작일~종료일 date 입력 + 캘린더(이전/다음 월 이동, 오늘 날짜 구분, **드래그로 기간 선택**(또는 탭)·구간 하이라이트). **별도 점(●) 표시는 사용하지 않는다.**
- **휴가 종류 토글**(필수): `연차`(1일 차감) / `오전 반차`(0.5일) / `오후 반차`(0.5일). **반차 선택 시 캘린더는 하루만 선택**(드래그도 단일). 범위가 선택된 상태에서 반차로 전환하면 시작일만 유지 + 알림 팝업 `반차는 하루만 선택해주세요.`
- **사유**(필수): 텍스트에어리어, 최대 200자·카운터.
- **비상 연락망**(선택): placeholder `예) 010-1234-5678`. *프로토타입은 자릿수 제한·하이픈 자동 삽입 미구현(목업). 정식 구현 시 최대 20자·숫자만·하이픈 자동.*
- **잔여 연차 미리보기**: `현재 잔여 {N}일` → `신청 시 {차감}일 차감 → 잔여 {N}일`.
- **결재선**: `홍길동(신청자) → 김팀장(1차 결재)` + 안내 `결재선은 경리나라를 기준으로 자동 지정됩니다` + [결재선 변경].
- **[신청하기]**: 검증 통과 시 토스트 `✅ 결재 상신이 완료되었습니다` 후 M-010 복귀. 휴가신청서(신청 구분 `신청`, 종류 `연차`)로 전자결재에 등록.

### M-014 연차 취소 신청
- **타이틀**: `연차 취소 신청`. 안내 배너(warning): `취소할 연차 날짜를 선택하세요. 선택한 기간·일수만큼 취소 신청이 상신됩니다.`
- **취소할 날짜**(필수): 시작일~종료일 date 입력 + 캘린더(탭/드래그 범위 선택). 하단에 `선택: {YYYY년 M월 D일 (요일)}` 표시. **승인된 연차 날짜로 제한하지 않으며, 점(●) 표시·범례 없음(어느 날짜든 선택 가능).**
- **휴가 종류 토글**(필수, **기본값 `연차`**): `연차`(1일) / `오전 반차`(0.5일) / `오후 반차`(0.5일). 취소 대상 휴가의 종류를 선택. 반차 선택 시 캘린더는 하루만 선택(M-011과 동일 — 범위 상태에서 반차 전환 시 시작일만 유지 + 알림). 취소 후 복구 일수도 종류에 따라 1일/0.5일.
- **취소 사유**(필수): 토글 `일정 변경` / `업무 일정` / `개인 사정` / `기타` + 텍스트에어리어(추가 입력, 선택).
- **취소 후 연차 현황**: `현재 잔여 연차 {N}일` / `취소 후 복구 +{N}일` / `취소 후 잔여 {N}일`.
- **결재선**: M-011과 동일(자동 지정 + [결재선 변경]).
- **[취소 신청하기]**: 토스트 `✅ 취소 신청이 상신되었습니다` 후 M-010 복귀. 휴가신청서(신청 구분 `취소`, 종류 `연차`)로 전자결재에 등록.

### M-015 / M-016 / M-017 휴가신청서 상세
- **타이틀/상태 뱃지**: `휴가신청서` + 상단 상태 뱃지(진행중 주황 / 승인 초록 / 반려 빨강). M-016·M-017은 앱 푸시 알림 탭으로 직접 진입 가능.
- **상태 배너 안내**: 진행중 `결재 진행중 / {결재자명} 확인 전입니다` / 승인 `{결재자명}이 승인하였습니다` / 반려 `{결재자명}이 반려하였습니다`.
- **배경**: 상세 본문은 **흰색**으로 채우되, **연차 현황 영역만 배경을 비워**(앱 기본 배경) 시각적으로 구분(M-000 §2.1 정합). 진행중·승인·반려 3개 화면 공통.
- **결재선**(상단): 신청자→결재/접수 단계 표시. **다단계 가능**하며 각 단계는 **결재(결재자) 또는 접수(접수자)** — 역할(신청/N차 결재/접수)과 단계별 뱃지(`상신완료`·`진행중`·`승인`·`반려`·`접수완료`·`대기`) 표기. 단계가 많아도 깨지지 않게 가로 스크롤. (공통 규칙 M-000 §6)
- **신청 내용**(rd-row): `신청 유형`(휴가신청서) / `휴가 종류`(예 `종일 (1일)` · `오전 반차 (0.5일)`) / `날짜` / `사용 일수` / `사유` / `메모`(사유와 별도 필드).
- **연차 현황 패널**: **진행중·승인 화면만** 표시(반려 화면 제외).
- **처리 이력 타임라인**: 상신·결재 처리 단계와 일시(`YYYY.MM.DD HH:MM · {결재자명}`).
- **M-017 반려**: 추가로 **반려 사유 박스** 표시(연차 현황 패널 없음). 박스는 별도 섹션 라벨 없이 **붉은 박스 안에 `반려 사유` 뱃지** + 반려 메시지 + `{처리 일시} · {반려 처리자}`로 구성(M-000 §2.1).
