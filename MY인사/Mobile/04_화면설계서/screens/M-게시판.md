## 화면 상세 설계

### M-002 게시판 홈

| 항목 | 내용 |
|------|------|
| **화면 목적** | 전 게시판의 최근 게시글을 피드 형태로 보여주는 앱의 기본 진입 화면. |
| **진입 조건** | 로그인 → 사업장 선택 후 / 하단 탭바 [게시판] 탭 탭 |
| **다음 화면** | 게시글 목록 M-003 / 글 작성 M-005 / 알림 M-050 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 | "게시판" |
| 상단 영역 | 부제 | "우리 회사 소식과 자유로운 대화" |
| 상단 영역 | 필터 아이콘(≡) | 우측 상단 — 게시판 선택 시트 열기 |
| 상단 영역 | **알림 아이콘(🔔)** | **필터 아이콘 우측 — 알림 목록(M-050) 진입. 읽지 않은 알림이 1건 이상 존재하면 아이콘 우상단에 빨간 점(●) 뱃지 표시** |
| 게시글 목록 | 게시글 행 | 제목 / NEW 뱃지 / 게시판명·작성자 / 날짜 / 화살표 |
| 게시판 선택 시트 | 바텀 시트 | 전체 + 사업장에서 운영 중인 게시판 목록 (API 동적 로드). *예시: 자유게시판 / 경조사게시판 등* |
| 하단 | 글쓰기 FAB(+) | 화면 우하단 고정 |

**사용자 흐름**
- 게시글 행 탭 → **게시글 상세(M-004)로 직접 이동**(뒤로가기 시 홈 복귀)
- 필터(≡) 아이콘 탭 → 게시판 선택 시트 표시 → 게시판 선택 → **홈 피드를 해당 게시판으로 인플레이스 필터링**(부제가 게시판명으로 대체, 필터 아이콘 강조). `전체` 선택 시 필터 해제
- **알림(🔔) 아이콘 탭 → 알림 목록(M-050)으로 이동**
- FAB(+) 탭 → 글 작성(M-005)으로 이동

**조건부 규칙**
- 선택한 게시판에 게시글이 없을 때: "등록된 게시글이 없습니다. 첫 번째 글을 작성해 보세요!" 빈 상태 표시
- 게시판 필터 선택 후: 필터 아이콘 테두리 색 강조 표시 (필터 적용 중임을 알림)
- NEW 뱃지(빨간 원형 N): **신규 게시글이면서 사용자가 아직 확인(열람)하지 않은 글**에만 게시글 **제목 옆**에 표시. 해당 게시글 상세를 열람하면 그 글의 N 뱃지는 소멸한다. **댓글에는 N 뱃지를 표시하지 않는다.**
- **알림 뱃지(●): 읽지 않은 알림이 1건 이상일 때 알림 아이콘 우상단에 빨간 점 표시. 모든 알림을 읽으면 뱃지 자동 소멸**

**조건부 규칙**

### M-004 게시글 상세

| 항목 | 내용 |
|------|------|
| **화면 목적** | 게시글 본문을 조회하고, 댓글을 읽거나 작성한다. |
| **진입 조건** | 게시글 목록(M-003)에서 게시글 행 탭 |
| **다음 화면** | 없음 (현재 화면 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 게시판명 | |
| 게시글 영역 | 게시판 구분 뱃지 / 제목 / 작성자·날짜 / 본문 내용 / 첨부 이미지 | |
| 게시글 영역 | 더보기(···) 버튼 | 작성자 본인인 경우에만 우측 상단에 표시. 탭 시 [수정하기] · [삭제하기] 액션 시트 표시 |
| 댓글 영역 | 댓글 수 표시 + 댓글 목록 | 작성자 **이름만** + 내용 + 작성 시간(**부서·아바타 미표시** — M-000 §5.2 댓글 작성자 규칙). 내 댓글에는 우측 상단 더보기(···) 버튼 표시 |
| 하단 고정 | 텍스트 입력창 + [등록] 버튼 | 최대 300자 / 다중 라인 / 카운터 표시 / 빈 입력 시 [등록] 비활성화 |

**사용자 흐름**
- 댓글 입력 → [등록] 탭 → 댓글 목록에 추가, 화면 유지
- [← 뒤로] 탭 → 게시글 목록(M-003)으로 복귀

**조건부 규칙**
- 본문에 이미지가 없으면 이미지 영역 표시 안 함
- **게시글 더보기(···) — 수정하기**: M-005 글 작성 폼과 동일한 UI로 전환 (화면명: "글 수정" / [등록] → [저장] 버튼). 기존 제목·게시판·본문이 자동 입력된 상태로 진입. 저장 완료 후 게시글 상세 화면으로 복귀. [취소] 탭 → "작업을 취소하시겠습니까?" 확인 팝업 → 확인 시 수정 내용 버리고 게시글 상세 복귀 / 취소 시 팝업 닫기.
- **게시글 더보기(···) — 삭제하기**: "게시글을 삭제하시겠습니까?" 확인 팝업 → 확인 시 삭제 후 게시글 목록(M-003)으로 이동
- **댓글 더보기(···) — 수정하기**: 해당 댓글 본문이 인라인 textarea로 전환 — 기존 내용 자동 입력 상태. 최대 3줄 표시 / 내용 3줄 초과 시 박스 내 스크롤 처리. [저장] · [취소] 버튼 하단 표시. [저장] 탭 시 **내용이 비어 있으면 토스트 `내용을 입력해주세요.` 표시 후 편집 유지**, 입력값이 있으면 수정 내용 반영 후 화면 유지. [취소] 탭 시 원래 내용 복원.
- **댓글 더보기(···) — 삭제하기**: "댓글을 삭제하시겠습니까?" 확인 팝업 → 확인 시 삭제 후 화면 유지

### M-005 글 작성

| 항목 | 내용 |
|------|------|
| **화면 목적** | 직원이 원하는 게시판에 새 게시글을 작성하고 등록한다. |
| **진입 조건** | 게시판 홈(M-002) 또는 게시글 목록(M-003)의 FAB(+) 탭 |
| **다음 화면** | 게시판 홈 M-002 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [취소] 버튼(좌) + "글 작성" 타이틀(중앙) + [등록] 버튼(우, 강조) | |
| 입력 폼 | 게시판 선택 드롭다운 | 사업장에서 운영 중인 게시판 목록 (API 동적 로드). *예시: 자유게시판 / 경조사게시판 등* — **필수** |
| 입력 폼 | 제목 입력 필드 | **필수** / 최대 100자 / 단일 라인 |
| 입력 폼 | 본문 텍스트 입력 영역 | **필수** / 최대 3,000자 / 다중 라인 |
| 입력 폼 | 파일 첨부 버튼 | 탭 시 **첨부 방식 선택 바텀시트**(+ 취소) 표시. 항목별로 디바이스 기능을 호출한다 — **카메라 촬영하기**: 디바이스 카메라 실행 / **앨범에서 선택하기**: 디바이스 갤러리(사진)에서 선택 / **파일 선택하기**: 디바이스 파일(폴더)에서 선택. 이미지(JPG·PNG) 및 문서(PDF·DOC·DOCX·XLS·XLSX·PPT·PPTX) — **다건 첨부 가능**, 최대 5개 / 파일당 최대 10 MB / **전체 합계 최대 50 MB** / 첨부 목록 가로 스크롤 미리보기(이미지는 썸네일, 문서는 파일명·형식 아이콘), 각 항목 우측 상단 × 버튼으로 개별 삭제 가능 |

**사용자 흐름**
- [취소] 탭 (내용 입력이 하나라도 있는 경우) → "작업을 취소하시겠습니까?" 확인 팝업 → 확인 시 이전 화면 복귀 (작성 내용 초기화) / 취소 시 팝업 닫기
- [취소] 탭 (아무것도 입력하지 않은 경우) → 확인 팝업 없이 바로 이전 화면 복귀
- [등록] 탭 (필수 항목 모두 입력 완료 시) → 등록 후 게시판 홈(M-002)으로 이동

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 게시판 미선택 / 제목 미입력 / 본문 미입력 | [등록] 비활성화 또는 해당 필드 오류 표시, 진행 불가 |
| 파일 형식 오류 | 이미지(JPG·PNG)·문서(PDF·DOC·DOCX·XLS·XLSX·PPT·PPTX)만 지원. 기타 형식 선택 시 "JPG·PNG·PDF·DOC·XLS·PPT 파일만 첨부 가능합니다" 안내 후 선택 취소 |
| 파일 5개 초과 선택 | "파일은 최대 5개까지 첨부 가능합니다" 안내 후 초과분 선택 차단 |
| 파일당 10 MB 초과 | "파일은 최대 10 MB까지 첨부 가능합니다" 안내 후 해당 파일 첨부 취소 |
| 첨부 총 용량 50 MB 초과 | 다건 첨부 시 **전체 파일 용량 합계가 50 MB를 초과**하면 "첨부 파일 총 용량은 50 MB를 초과할 수 없습니다" 안내 후 추가 첨부 차단 |

## 반응형 — 태블릿 분할 보기 (게시판 ↔ 상세)

| 항목 | 내용 |
|------|------|
| 적용 폭 | 뷰포트 너비 **≥ 900px**(태블릿). *예: 아이패드 미니 8.3" **가로** 1133px 등. 아이패드 미니 **세로**(744px)는 900px 미만이라 모바일 동작* |
| 분할 레이아웃 | 게시판 홈(M-002)에서 게시글 선택 시 화면이 **좌우 50:50**으로 분할 — **좌: 게시판 목록(피드), 우: 게시글 상세(M-004)** |
| 상세 뒤로가기 | 분할 시 우측 상세 화면에는 **[← 뒤로] 버튼을 표시하지 않는다**(좌측 목록이 항상 보임) |
| 하단 탭바 | 분할 시 하단 탭바는 **좌측(목록) 영역에만** 표시(너비 50%) |
| 목록 ↔ 상세 연동 | 좌측 목록에서 다른 게시글 선택 시 우측 상세가 해당 글로 갱신. **게시글 수정 저장·작성 취소 후 상세 복귀 시에도 분할(좌 목록·우 상세) 유지** |
| 탭 이동 | 다른 탭(연차·영수증·증명서·더보기)으로 이동 시 분할 해제 → **전체 배율로 복원**하며 해당 탭으로 이동 |
| 폭 경계 전환 | 분할(≥900px) 중 900px 미만으로 좁히면 상세가 풀스크린으로 전환되고 [← 뒤로] 재노출. 반대로 모바일 상세에서 900px 이상으로 넓히면 분할로 전환 |

> **모바일(< 900px)**: 게시판 목록 → 게시글 상세로 **화면 전환**되며, 상세 상단 **[← 뒤로]** 버튼으로 목록 복귀.
> 분할 기준폭·레이아웃 구현 기준은 퍼블리싱 규칙 `rules_LAYOUT — 태블릿 분할 레이아웃(≥900px)`을 따른다.

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 공통 입력 UI 규칙 참조)
> **데이터 출처**(공통 원칙 → M-000 §5.1): 게시판·게시글·댓글은 **그룹웨어 게시판**에서 관리되며, MY인사 모바일 앱은 API로 연동해 **현재 로그인한 사업장(그룹웨어)에서 본인이 속한(접근 권한이 있는) 게시판의 게시글만** 표시한다(타 사업장·미소속 게시판은 노출 안 함). 게시판 생성·삭제 등 운영 관리는 그룹웨어 게시판에서 수행한다. 게시판 목록은 API 동적 로드한다.

> **공통 표시 규칙 정합**: 날짜·텍스트 오버플로우·부서 없음 처리 등은 모바일 공통(M-000) **5. 데이터 표시 공통 규칙**을 따른다. 날짜는 목록·카드 `YYYY.MM.DD`(당일=오늘/전일=어제, 본 화면은 `M월 D일` 약식 사용), 상세·이력은 `YYYY.MM.DD HH:MM`. 부서는 경리나라 필수 항목이 아니므로 **없으면 토큰·구분자까지 생략**(5.1).

### 1. M-002 게시판 홈 (`board`)

| 영역/필드 | 데이터 출처 | 표시 형식 | 동작·규칙 |
|---|---|---|---|
| 타이틀 | 정적 | `게시판` | 고정 |
| 부제(`#boardFilterLabel`) | 정적 / 필터 상태 | `우리 회사 소식과 자유로운 대화` | 필터 적용 시 선택 **게시판명**으로 대체, `전체`면 기본 문구 |
| 필터 아이콘(≡) | — | 아이콘 | 탭 → 게시판 선택 바텀 시트. 필터 적용 중이면 테두리·아이콘색 accent 강조 |
| 알림 아이콘(🔔)·뱃지(`#notifBadge`) | 알림(M-050) 미읽음 여부 | 빨간 점(●) | 미읽음 ≥1건이면 표시, 전체 읽음 시 소멸. 탭 → `notifications` |
| 게시판 선택 시트 항목 | 그룹웨어 게시판(현재 사업장·**본인 소속 게시판**) API 동적 로드 | `전체` + 게시판명 목록 | 현재 선택 항목 강조. 선택 시 홈 피드 인플레이스 필터 |

**게시글 피드 행** (출처: 그룹웨어 게시판 — 현재 사업장·본인 소속 게시판의 최근 게시글)

- **제목**: 게시글 제목. **1줄 말줄임**(초과분 `…`, 공통 5.2). 길게 누르기(500ms) 시 전체 툴팁.
- **N 뱃지**: 제목 끝. **신규 게시글 + 본인이 아직 열람하지 않은 글**일 때만 빨간 원형 `N` 표시. 해당 글 상세 열람 시 소멸. (댓글에는 표시하지 않음)
- **메타**: `{게시판명} · {작성자}`. 게시판명=그룹웨어 게시판 채널명, 작성자=경리나라 사용자관리 사용자명. (피드 행에는 부서·시각 미표기)
- **날짜**: `{M월 D일}`(목록 약식). 당일=`오늘`, 전일=`어제`로 대체 가능.
- **우측 화살표**: 탭 → 해당 게시글 **상세(M-004) 직접 이동**(뒤로가기 시 홈 복귀).
- **정렬**: 최신 등록순(상단이 최신).
- **빈 상태**: 필터 결과 0건이면 `등록된 게시글이 없습니다.` + `아직 작성된 게시글이 없어요. 첫 번째 글을 작성해 보세요!`.
- **글쓰기 FAB(+)**: 우하단 고정. 탭 → `compose`.

### 2. M-003 게시글 목록 (`post-list`)

> 본 화면은 특정 게시판(채널) 목록 화면이다. 게시판 홈(M-002)은 인플레이스 필터로 상세에 직접 이동하므로, 본 화면은 채널형 진입(예: 인사 FAQ)에서 사용된다. *(인사 FAQ 채널 상세는 더보기_알림 M-006/M-008 참조)*

- **상단**: [← 뒤로] + **게시판명(채널명)**(`#postListTitle`, 그룹웨어 게시판 출처).
- **게시글 행 필드**:
  - **제목**: 1줄 말줄임. **공지 글**은 제목 앞에 `공지` 핀 뱃지가 붙고 목록 **상단 고정**.
  - **메타**: `{작성자} · {M월 D일} · 댓글 {N}`. 작성자=사용자관리 사용자명, 날짜=`M월 D일`, `댓글 {N}`=댓글 수(0 이상 정수).
  - **NEW 뱃지**: 신규·미확인 상위 글에 `N` 뱃지.
- **글쓰기 FAB**: 일반 게시판은 노출 → `compose`. **읽기 전용 채널(예: 인사 FAQ)은 숨김.**

### 3. M-004 게시글 상세 (`post-detail`)

- **상단**: [← 뒤로] + **게시판명**(`#postDetailBoardTitle`) + 더보기(···). 더보기는 **작성자 본인일 때만** 노출(본인 판정: 현재 로그인 사용자 == 게시글 작성자). 탭 → 액션 시트 [수정하기]/[삭제하기].
- **게시글 본문 필드** (출처: 그룹웨어 게시판 게시글):
  - **제목**: 전체 표시(말줄임 없음). **공지 글**은 앞에 `공지` 핀 뱃지.
  - **메타**: `{작성자} · {등록일시}`. 작성자=사용자관리 사용자명, 등록일시=**`YYYY.MM.DD HH:MM`**(상세 형식). **조회수는 표시하지 않는다.**
  - **본문**: 그룹웨어 게시판 본문(HTML 단락·강조 허용), 흰색 배경 영역에 표시.
  - **첨부**: 있을 때만 표시. 이미지=썸네일(탭 시 확대), 문서=파일명·형식 아이콘. 첨부 없으면 영역 자체 미표시.
- **댓글 영역**:
  - **헤더**: `댓글 {N}`(N=댓글 수).
  - **각 댓글 필드**: 작성자 `{이름}` — **이름만 표시**(부서·아바타 미표시, M-000 §5.2 댓글 작성자 규칙) / 내용 / 작성 시간. 시간 형식은 기존 이력 `M월 D일 HH:MM`, **신규 작성 댓글은 등록 시각 `YYYY.MM.DD HH:MM`**.
  - **내 댓글**(작성자 본인): 우측 더보기(···) → [수정하기](인라인 편집)/[삭제하기].
- **댓글 입력창**(하단 고정): placeholder `댓글을 입력하세요`, 최대 300자. **[등록]·댓글 [저장] 시 내용 필수** — 비어 있으면 토스트 `내용을 입력해주세요.` 후 중단(편집 유지). 입력값 있으면 등록(작성자=현재 사용자, 시각=등록 시점 `YYYY.MM.DD HH:MM`)·입력창 초기화·토스트 `댓글이 등록되었습니다`.
- **수정/삭제 흐름**: 게시글 [수정하기] → 글 수정 폼(`compose` 편집 모드, 기존 게시판·제목·본문 pre-fill, [저장]) → 저장 시 상세 복귀·토스트 `게시글이 수정되었습니다`. 게시글 [삭제하기] → `게시글을 삭제하시겠습니까?`(우측 빨강) 확인 → 삭제 후 목록(홈) 복귀. 댓글 [수정하기] → 인라인 textarea(기존 내용 pre-fill, 최대 3줄·초과 스크롤)·[저장]/[취소]. 댓글 [삭제하기] → `댓글을 삭제하시겠습니까?` 확인 후 화면 유지.

### 4. M-005 글 작성 / 글 수정 (`compose`)

- **상단 내비**: [취소](좌) / 타이틀(작성=`글 작성`, 수정=`글 수정`)(중앙) / 액션(작성=`등록`, 수정=`저장`)(우). 액션 버튼은 **게시판·제목·본문이 모두 채워지면** accent 활성, 아니면 비활성색(`checkComposeReady()`).
- **게시판 선택**(`#composeBoardSelect`, **필수**): 현재 사업장에서 본인이 작성 가능한 운영 게시판 목록(그룹웨어 게시판 API 동적 로드). **인사 FAQ는 선택지에서 제외**(직원 작성 불가).
- **제목**(**필수**): 단일 라인, 최대 100자.
- **본문**(**필수**): 다중 라인, 최대 3,000자. (글자수 카운터는 표시하지 않는다)
- **파일 첨부**: 첨부 방식 선택 바텀시트(카메라/앨범/파일)·디바이스 호출·미리보기·삭제 등 **공통 동작은 M-000 §2 파일 첨부** 참조. 게시판 글 작성 **허용 범위**(고유) — 이미지(JPG·PNG)+문서(PDF·DOC·DOCX·XLS·XLSX·PPT·PPTX), **다건 첨부**·최대 5개·파일당 10 MB·**총 합계 50 MB 이내**. *(프로토타입은 디바이스 선택기 호출까지 동작 — 선택 후 목록·미리보기 렌더링 미구현)*
- **등록/저장/취소**:
  - 작성 [등록] → 필수값 검증 통과 시 토스트 `✅ 게시글이 등록되었습니다` + 홈(`board`) 복귀, 글은 피드 상단에 추가(작성자=현재 사용자, 작성 시각=등록 시점 `YYYY.MM.DD HH:MM`).
  - 수정 [저장] → 게시판·제목·본문 갱신 후 상세 복귀·토스트 `게시글이 수정되었습니다`.
  - [취소] → 입력 내용이 하나라도 있으면 `작업을 취소하시겠습니까?` 확인 팝업, 없으면 즉시 이전 화면 복귀.
