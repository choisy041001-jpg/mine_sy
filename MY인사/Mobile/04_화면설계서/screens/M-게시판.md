## 화면 상세 설계

### M-002 게시판 홈

| 항목 | 내용 |
|------|------|
| **화면 목적** | 전 게시판의 최근 게시글을 피드 형태로 보여주는 앱의 기본 진입 화면. |
| **진입 조건** | 로그인 → 사업장 선택 후 / 하단 탭바 [게시판] 탭 탭 |
| **다음 화면** | 게시글 목록 M-003 / 글 작성 M-005 / 알림 M-050 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 | "게시판" |
| 상단 영역 | 부제 | "우리 회사 소식과 자유로운 대화" |
| 상단 영역 | 필터 아이콘(≡) | 우측 상단 — 게시판 선택 시트 열기 |
| 상단 영역 | **알림 아이콘(🔔)** | **필터 아이콘 우측 — 알림 목록(M-050) 진입. 읽지 않은 알림이 1건 이상 존재하면 아이콘 우상단에 빨간 점(●) 뱃지 표시** |
| 게시글 목록 | 게시글 행 | 제목 / NEW 뱃지 / 게시판명·작성자 / 날짜 / 화살표 |
| 게시판 선택 시트 | 바텀 시트 | 전체 + 사업장에서 운영 중인 게시판 목록 (API 동적 로드). *예시: 자유게시판 / 경조사게시판 등* |
| 하단 | 글쓰기 FAB(+) | 화면 우하단 고정 |

**사용자 흐름**
- 게시글 행 탭 → **게시글 상세(M-004)로 직접 이동**(뒤로가기 시 홈 복귀)
- 필터(≡) 아이콘 탭 → 게시판 선택 시트 표시 → 게시판 선택 → **홈 피드를 해당 게시판으로 인플레이스 필터링**(부제가 게시판명으로 대체, 필터 아이콘 강조). `전체` 선택 시 필터 해제
- **알림(🔔) 아이콘 탭 → 알림 목록(M-050)으로 이동**
- FAB(+) 탭 → 글 작성(M-005)으로 이동

**조건부 규칙**
- 선택한 게시판에 게시글이 없을 때: "등록된 게시글이 없습니다. 첫 번째 글을 작성해 보세요!" 빈 상태 표시
- 게시판 필터 선택 후: 필터 아이콘 테두리 색 강조 표시 (필터 적용 중임을 알림)
- NEW 뱃지: 읽지 않은 새 게시글에 표시 (빨간 원형 N)
- **알림 뱃지(●): 읽지 않은 알림이 1건 이상일 때 알림 아이콘 우상단에 빨간 점 표시. 모든 알림을 읽으면 뱃지 자동 소멸**

### M-004 게시글 상세

| 항목 | 내용 |
|------|------|
| **화면 목적** | 게시글 본문을 조회하고, 댓글을 읽거나 작성한다. |
| **진입 조건** | 게시글 목록(M-003)에서 게시글 행 탭 |
| **다음 화면** | 없음 (현재 화면 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + 게시판명 | |
| 게시글 영역 | 게시판 구분 뱃지 / 제목 / 작성자·날짜 / 본문 내용 / 첨부 이미지 | |
| 게시글 영역 | 더보기(···) 버튼 | 작성자 본인인 경우에만 우측 상단에 표시. 탭 시 [수정하기] · [삭제하기] 액션 시트 표시 |
| 댓글 영역 | 댓글 수 표시 + 댓글 목록 | 작성자 이름 + 부서 + 내용 + 작성 시간(아바타 없음). 내 댓글에는 우측 상단 더보기(···) 버튼 표시 |
| 하단 고정 | 텍스트 입력창 + [등록] 버튼 | 최대 300자 / 다중 라인 / 카운터 표시 / 빈 입력 시 [등록] 비활성화 |

**사용자 흐름**
- 댓글 입력 → [등록] 탭 → 댓글 목록에 추가, 화면 유지
- [← 뒤로] 탭 → 게시글 목록(M-003)으로 복귀

**조건부 규칙**
- 본문에 이미지가 없으면 이미지 영역 표시 안 함
- **게시글 더보기(···) — 수정하기**: M-005 글 작성 폼과 동일한 UI로 전환 (화면명: "글 수정" / [등록] → [저장] 버튼). 기존 제목·게시판·본문이 자동 입력된 상태로 진입. 저장 완료 후 게시글 상세 화면으로 복귀. [취소] 탭 → "작업을 취소하시겠습니까?" 확인 팝업 → 확인 시 수정 내용 버리고 게시글 상세 복귀 / 취소 시 팝업 닫기.
- **게시글 더보기(···) — 삭제하기**: "게시글을 삭제하시겠습니까?" 확인 팝업 → 확인 시 삭제 후 게시글 목록(M-003)으로 이동
- **댓글 더보기(···) — 수정하기**: 해당 댓글 본문이 인라인 textarea로 전환 — 기존 내용 자동 입력 상태. 최대 3줄 표시 / 내용 3줄 초과 시 박스 내 스크롤 처리. [저장] · [취소] 버튼 하단 표시. [저장] 탭 시 수정 내용 반영 후 화면 유지. [취소] 탭 시 원래 내용 복원.
- **댓글 더보기(···) — 삭제하기**: "댓글을 삭제하시겠습니까?" 확인 팝업 → 확인 시 삭제 후 화면 유지

| 입력 폼 | 이미지 첨부 버튼 | 카메라 촬영 또는 갤러리 선택 — 선택 / 최대 5장 / JPG·PNG · 파일당 최대 10 MB / 첨부된 이미지 썸네일 가로 스크롤 미리보기, 각 썸네일 우측 상단 × 버튼으로 개별 삭제 가능 |

**사용자 흐름**
- [취소] 탭 (내용 입력이 하나라도 있는 경우) → "작업을 취소하시겠습니까?" 확인 팝업 → 확인 시 이전 화면 복귀 (작성 내용 초기화) / 취소 시 팝업 닫기
- [취소] 탭 (아무것도 입력하지 않은 경우) → 확인 팝업 없이 바로 이전 화면 복귀
- [등록] 탭 (필수 항목 모두 입력 완료 시) → 등록 후 게시판 홈(M-002)으로 이동

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 게시판 미선택 / 제목 미입력 / 본문 미입력 | [등록] 비활성화 또는 해당 필드 오류 표시, 진행 불가 |
| 이미지 파일 형식 오류 | JPG·PNG만 지원. 기타 형식 선택 시 "JPG, PNG 파일만 첨부 가능합니다" 안내 후 선택 취소 |

| 이미지 10 MB 초과 | "파일은 최대 10 MB까지 첨부 가능합니다" 안내 후 해당 파일 첨부 취소 |

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 공통 입력 UI 규칙 참조)
> **데이터 출처**: 게시판·게시글·댓글은 **경리나라 게시판 앱**에서 관리되며, MY인사 모바일 앱은 API로 연동해 **현재 로그인한 사업장 기준**으로 표시한다. 게시판 생성·삭제 등 운영 관리는 게시판 앱에서 수행한다. 게시판 목록은 API 동적 로드한다.

### 1. M-002 게시판 홈 (`board`)

- **상단 타이틀/부제**: `게시판` + 부제 `우리 회사 소식과 자유로운 대화`. 필터 적용 시 부제(`#boardFilterLabel`)는 선택한 **게시판명**으로 대체된다.
- **필터 아이콘(≡)**: 탭 시 게시판 선택 바텀 시트. 시트 항목 = `전체` + 운영 게시판 목록(API). 필터 적용 중이면 아이콘 테두리가 accent 색으로 강조된다.
- **알림 아이콘(🔔)**: 탭 시 알림 목록(`notifications`, M-050) 이동. 읽지 않은 알림이 1건 이상이면 우상단 빨간 점 뱃지(`#notifBadge`) 표시, 전체 읽음 시 소멸.
- **게시글 피드 행**: 각 행은 `제목` + (읽지 않은 새 글이면 `N` 뱃지) + `{게시판명} · {작성자}` + `{M월 D일}` + 우측 화살표. 탭 시 해당 게시글 상세로 진입(홈에서는 상세로 직접 이동, 뒤로가기 시 홈 복귀).
- **빈 상태**: 선택 게시판에 글이 없으면 `등록된 게시글이 없습니다.` + `아직 작성된 게시글이 없어요. 첫 번째 글을 작성해 보세요!`.
- **글쓰기 FAB(+)**: 우하단 고정. 탭 시 글 작성(`compose`) 진입.

### 2. M-003 게시글 목록 (`post-list`)

- **상단**: [← 뒤로] + **게시판명**(`#postListTitle`).
- **게시글 행**: `제목`(공지 글은 앞에 `공지` 핀 뱃지, 항상 상단 고정) + 메타 `{작성자} · {M월 D일} · 댓글 {N}`. 일반 게시판은 진입 시 상위 N개 행에 `N`(NEW) 뱃지 표시.
- **인사 FAQ 채널 분기**: 인사 FAQ 게시판으로 진입한 경우 FAQ 글 목록을 표시하고 **글쓰기 FAB은 숨김**(인사담당자만 PC에서 등록). 일반 게시판은 FAB 노출.
- **글쓰기 FAB(+)**: 일반 게시판에서만 노출, 탭 시 `compose`.

### 3. M-004 게시글 상세 (`post-detail`)

- **상단**: [← 뒤로] + **게시판명**(`#postDetailBoardTitle`) + 더보기(···) 버튼(작성자 본인일 때만). 더보기 → 액션 시트 [수정하기]/[삭제하기].
- **게시글 본문**(`_POST_DETAIL[postId]` 기준): `제목`(공지 글은 앞에 `공지` 핀 뱃지) / 메타 `{작성자} · {등록일시 YYYY.MM.DD HH:MM}` / `본문`(HTML, 단락·강조 포함) / 첨부 이미지(있을 때만). **조회수는 표시하지 않는다.**
- **댓글 영역**: 헤더 `댓글 {N}`. 각 댓글 = **작성자 이름 + 부서(있을 때)** + 내용 + 작성 시간(`{M월 D일 HH:MM}`). **아바타는 표시하지 않는다.** 내 댓글(작성자 본인)에는 우측 더보기(···) → [수정하기]/[삭제하기].
- **댓글 입력창**(하단 고정): placeholder `댓글을 입력하세요` + [등록]. 최대 300자·다중 라인·카운터, 빈 입력 시 [등록] 비활성. *(프로토타입은 입력창·등록 버튼이 목업 — 실제 등록 동작 미구현)*
- **수정/삭제 흐름**: 게시글 [수정하기] → 글 수정 폼(`post-edit`, 기존 제목·게시판·본문 pre-fill, [저장]). [삭제하기] → `게시글을 삭제하시겠습니까?` 확인 후 목록 복귀. 댓글 [수정하기] → 인라인 편집, [삭제하기] → `댓글을 삭제하시겠습니까?` 확인.

### 4. M-005 글 작성 (`compose`)

- **상단 내비**: [취소](좌) / `글 작성`(중앙) / [등록](우). [등록]은 제목·본문이 모두 채워지면 accent로 활성, 아니면 비활성 색(`checkComposeReady()`).
- **게시판 선택**: 드롭다운(`#composeBoardSelect`, **필수**). 운영 게시판 동적 로드. **인사 FAQ는 선택지에 없음**(직원 작성 불가).
- **제목**: 단일 라인, **필수**, 최대 100자.
- **본문**: 다중 라인, **필수**, 최대 3,000자, 카운터 표시.
- **이미지 첨부**: [첨부]/[이미지] 버튼. 최대 5장 / JPG·PNG / 파일당 10 MB / 썸네일 가로 스크롤 미리보기·개별 × 삭제. *(프로토타입은 버튼 탭 시 안내 토스트만 — 실제 첨부·미리보기 미구현)*
- **등록/취소**: [등록] → 필수값 검증 후 토스트 `✅ 게시글이 등록되었습니다` + 게시판 홈(`board`) 복귀. [취소] → 입력 내용이 있으면 `작업을 취소하시겠습니까?` 확인 팝업, 없으면 바로 이전 화면 복귀.
