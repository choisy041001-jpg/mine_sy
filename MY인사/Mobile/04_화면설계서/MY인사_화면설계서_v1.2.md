## 1. 문서 개요

### 1.1 목적

본 문서는 MY인사의 모바일 앱 및 PC 웹앱 화면을 기획자 관점에서 정의한다. 각 화면의 목적, 구성 요소, 사용자 흐름, 노출 조건, 유효성 규칙을 기술하며, 디자이너·개발자·QA가 화면을 구현하고 검증하는 기준으로 활용된다.

### 1.2 서비스 구성

| 구분 | 플랫폼 | 대상 사용자 | 주요 기능 |
|------|--------|-----------|---------|
| 모바일 앱 | 모바일 웹앱 (375 px) | 직원 | 게시판 · 연차 신청/조회 · 영수증 등록/결의서 · 증명서 신청 · FAQ · 내 정보 · **알림 · 알림 설정** |
| PC 웹앱 | 웹 브라우저 (최소 가로 1280 px) | 관리자 (인사·경리담당자) | 연차 결재 · 증명서 발급 · FAQ 관리 · 대시보드 |
| PC 웹앱 | 웹 브라우저 | 직원 | 연차 조회 · 증명서 신청 · 프로필 관리 |

### 1.3 용어 및 공통 규칙

| 용어/규칙 | 설명 |
|----------|------|
| 필수 항목 | 입력하지 않으면 다음 단계로 진행되지 않는 항목. 항목 레이블 옆에 빨간 `*` 표시 |
| 선택 항목 | 입력하지 않아도 진행 가능. 항목 레이블에 `(선택)` 표시 |
| 상태 뱃지 | 처리 상태를 색상 태그로 표시. **진행중**(주황) · **승인/완료**(초록) · **반려**(빨강) |
| FAB | 화면 우하단에 고정된 주요 액션 버튼(+). 탭 시 세부 액션 목록 표시 또는 직접 이동 |
| 탭바 | 모바일 하단 고정 네비게이션. 5개 탭: 게시판 · 연차 · 영수증 · 증명서 · 더보기 |
| 결재선 | 경리나라 시스템에서 자동 지정. 사용자가 직접 변경 불가 |
| 사업장 | 1개 계정으로 여러 사업장 관리 가능. 로그인 시 사업장 선택, 앱 내 전환 가능 |
| 빈 상태 | 목록/검색 결과가 없을 때 표시하는 안내 화면 (일러스트 + 안내 문구) |
| 로그인 유지 | 자동 로그인 체크 시 재접속 시 로그인 생략 |

### 1.4 공통 입력 UI 규칙

> 이하 규칙은 별도 명시가 없는 한 모든 화면·플랫폼에 적용된다.

**글자 수 제한 및 카운터**

| 규칙 | 설명 |
|------|------|
| **글자 수 초과 차단** | 최대 글자 수 도달 시 추가 입력 불가. 붙여넣기 시에도 최대 글자 수까지만 반영, 초과분 자동 잘림. |
| **다중 라인 카운터** | 최대 글자 수 제한이 있는 `<textarea>`(여러 줄 입력 필드)에 한해 우측 하단에 "현재/최대" 형식 카운터 표시 (예: 85/200). 최대치의 90% 초과 시 카운터 텍스트 주황색, 최대치 도달 시 빨간색. |
| **단일 라인 카운터** | 단일 라인 `<input>` 필드는 카운터 미표시. |

**오버플로우 처리**

| 상황 | 처리 방식 |
|------|---------|
| 단일 라인 입력 필드 — 입력 중 | 커서 위치 기준 가로 스크롤. 별도 툴팁 없음. |
| 단일 라인 입력 필드 — 포커스 해제 후 | 텍스트 앞부분(처음)부터 표시. 시각적 말줄임(…) 적용. **모바일**: 길게 누르기(500 ms) 시 전체 내용 말풍선 툴팁. **PC**: 마우스 오버 시 `title` 속성 또는 커스텀 툴팁으로 전체 내용 표시. |
| 목록·카드·테이블 셀의 텍스트 | 1줄(게시글 제목·사용처 등) 또는 2줄(FAQ 질문 등) 말줄임(…) 처리 후 동일 툴팁 규칙 적용. |

**툴팁 규격**

| 플랫폼 | 트리거 | 스타일 |
|-------|-------|------|
| 모바일 | 길게 누르기 500 ms | 반투명 검정 배경 · 흰색 텍스트 · 최대 너비 280 px · 자동 줄바꿈 · 화면 밖으로 나가지 않도록 위치 자동 조정 |
| PC | 마우스 오버 | 최대 너비 320 px · 자동 줄바꿈 |

**숫자·날짜·특수 형식 입력**

| 필드 유형 | 입력 규칙 | 표시 형식 |
|---------|---------|---------|
| 금액 | 숫자만 허용 (콤마·소수점·음수 불가) | 입력 즉시 3자리마다 콤마 자동 삽입 (예: 1,500,000) |
| 전화번호 | 숫자만 허용 | 하이픈(-) 자동 삽입 (예: 010-1234-5678) |
| 이메일 | 포커스 이탈 시 형식 검증 | 오류 시 필드 하단 "올바른 이메일 형식을 입력하세요" |
| 날짜 (목록·카드) | — | "YYYY.MM.DD" 형식. 당일 → "오늘", 전일 → "어제". 시각이 필요한 경우 "YYYY.MM.DD HH:mm" |
| 날짜 (상세·이력) | — | "YYYY년 MM월 DD일 HH:mm" 형식 |

**캘린더 선택기 공통 규칙** (M-011·M-014·P-002 등 날짜 선택이 있는 모든 화면)

| 항목 | 규칙 |
|------|------|
| 선택 가능 범위 | 오늘 이후(미래) 날짜만 선택 가능. 과거 날짜는 비활성(회색) 처리하여 선택 불가. |
| 주말 표시 | 토요일 파란색, 일요일 빨간색 텍스트로 구분 표시. 선택은 가능. |
| 공휴일 | 해당 연도 법정 공휴일 빨간색 텍스트 표시. 선택 가능. |
| 이미 승인된 연차 | 점(●) 표시. 연차 신청 시 해당 날짜 선택 불가 (중복 신청 방지). |
| 선택 범위 제한 | 연속 선택 최대 30일. 초과 시 "최대 30일까지 선택 가능합니다" 안내 후 31일째 선택 차단. |

**검색 필드**

| 항목 | 규칙 |
|------|------|
| 최소 입력 글자 | 1자 이상 입력 시 결과 표시 |
| 디바운스 | 입력 후 300 ms 대기 후 자동 검색 실행 |
| 결과 없음 | "검색 결과가 없습니다" 안내 표시 |
| 검색어 초기화 | 필드 우측 × 아이콘 탭/클릭 시 검색어 초기화 및 전체 목록 복원 |

**이미지 첨부 공통 규칙**

| 항목 | 규칙 |
|------|------|
| 지원 형식 | JPG, PNG |
| 파일당 최대 크기 | 10 MB |
| 게시글(M-005) 이미지 | 최대 5장 (초과 시 "이미지는 최대 5장까지 첨부 가능합니다" 안내, 추가 선택 차단) |
| 영수증(M-021)·증명서(M-031) 첨부 | 1건 (교체 방식) |
| 용량 초과 시 처리 | "파일은 최대 10 MB까지 첨부 가능합니다" 안내 후 해당 파일 첨부 취소 |

**공통 다이얼로그·알림**

| 유형 | 규칙 |
|------|------|
| 토스트 메시지 | 화면 하단 표시, 3초 후 자동 사라짐. 여러 개 발생 시 위로 쌓임(최신이 위). |
| 확인 다이얼로그 | 버튼 구성: [취소](좌, 기본) · [동작명](우, 강조). 삭제 시 [삭제](빨강), 로그아웃 시 [로그아웃](빨강). |
| 로딩 중 버튼 | API 호출 중 해당 버튼 비활성화 + 스피너 표시 (중복 제출 방지). |
| 로딩 중 버튼 텍스트 | 버튼 텍스트 그대로 유지하되 스피너를 텍스트 앞에 표시. |

## 3. 모바일 화면 상세 설계

> **모바일 공통 화면설계서**: 모바일 전 화면이 공유하는 화면 셸(상단바·하단 탭바·FAB)·공통 컴포넌트(바텀 시트·토스트·다이얼로그·빈 상태·상태 뱃지)·사업장·알림·데이터 표시 요건은 `screens/M-000_모바일공통.md`에 별도 정리한다(디자인 토큰·CSS·구현 세부는 별도 퍼블리싱 규칙 문서 소관, 플랫폼 공통 입력 규칙은 1.4 참조). 본 절(3.x) 화면별 설계는 해당 공통 규칙을 전제로 하며, 특히 **부서 = 선택 항목** 요건(부서명이 없는 직원이 존재할 수 있으므로 댓글 작성자·프로필·연결형 메타에서 부서 표기를 공란 처리하거나 토큰·구분자(· )를 생략하고 빈칸·`undefined`·`null`을 노출하지 않음)을 모든 모바일 화면에 적용한다.

#### M-001 로그인

| 항목 | 내용 |
|------|------|
| **화면 목적** | 사용자가 아이디·비밀번호로 인증하고, 소속 사업장을 선택하여 앱에 진입한다. |
| **진입 조건** | ① 앱 최초 실행 시 ② 로그아웃 후 ③ 세션 만료(자동 로그인 미설정 시 앱 재실행) 시 → 하단 탭바 미표시 |
| **다음 화면** | 사업장 선택 팝업 M-001-P |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 로고 영역 | 앱 아이콘 + 앱 이름 | 파란 원형 아이콘 + "MY인사" |
| 입력 영역 | 아이디 입력 필드 | placeholder "이메일 또는 사번" / 최대 100자 / 영문·숫자·특수문자 |
| 입력 영역 | 비밀번호 입력 필드 | 텍스트 가림 처리 + 우측 표시/숨기기 토글 아이콘 / 최대 50자 |
| 옵션 영역 | 자동 로그인 체크박스 | 기본값: 미체크 |
| 액션 영역 | [로그인] 버튼 | 화면 전체 너비 |
| 액션 영역 | 비밀번호 찾기 링크 | 버튼 하단 중앙 정렬 |

**사용자 흐름**

1. 아이디 + 비밀번호 입력
2. [로그인] 버튼 탭
   - 인증 성공 → 사업장 선택 팝업(M-001-P) 표시
   - 인증 실패 → 에러 메시지 표시 (화면 유지)
3. 사업장 선택 → 게시판 홈(M-002) 진입

**조건부 규칙**
- 자동 로그인 체크 시: 다음 앱 실행 시 로그인 화면 건너뛰고 사업장 선택 팝업 또는 마지막 사업장으로 바로 진입
- 비밀번호 표시/숨기기: 토글 아이콘 탭마다 전환

**유효성 규칙**

| 오류 조건 | 오류 표시 위치 | 오류 메시지 | 처리 결과 |
|---------|-------------|-----------|---------|
| 아이디 또는 비밀번호 불일치 | 비밀번호 입력 필드 하단 | "아이디 또는 비밀번호가 올바르지 않습니다" | 비밀번호 필드 초기화, 화면 유지 |

- 목록이 일정 높이 초과 시: 내부 스크롤 처리

#### M-002 게시판 홈

| **진입 조건** | 로그인 → 사업장 선택 후 / 하단 탭바 [게시판] 탭 탭 |

- **알림(🔔) 아이콘 탭 → 알림 목록(M-050)으로 이동**
- FAB(+) 탭 → 글 작성(M-005)으로 이동

**조건부 규칙**
- 선택한 게시판에 게시글이 없을 때: "등록된 게시글이 없습니다. 첫 번째 글을 작성해 보세요!" 빈 상태 표시
- 게시판 필터 선택 후: 필터 아이콘 테두리 색 강조 표시 (필터 적용 중임을 알림)
- NEW 뱃지: 읽지 않은 새 게시글에 표시 (빨간 원형 N)
- **알림 뱃지(●): 읽지 않은 알림이 1건 이상일 때 알림 아이콘 우상단에 빨간 점 표시. 모든 알림을 읽으면 뱃지 자동 소멸**

### 3.3 인사 FAQ

> 인사 FAQ는 인사담당자가 관리자 PC **FAQ 관리(P-004)** 에서 등록한 '자주 묻는 질문과 답변'을 직원이 모바일에서 **읽기 전용**으로 열람하는 게시판형 채널이다. 전용 화면이 아니라 **게시판(M-게시판)의 게시글 목록(M-003)·게시글 상세(M-004) 화면을 `인사 FAQ` 채널로 재사용**하며, 인사 FAQ 채널에서는 직원 글 작성(글쓰기 FAB)이 노출되지 않는다. 세부 필드 정의는 `screens/M-FAQ.md` 참조.

## 4. PC 화면 상세 설계

PC 앱은 **좌측 사이드바(240 px) + 우측 메인 콘텐츠 영역**으로 구성된다. 로그인 계정 권한에 따라 표시 항목이 자동으로 결정된다.

**사이드바 구성**

| 구성 요소 | 설명 |
|---------|------|
| 브랜드 영역 | 앱 로고 + "MY인사" 명칭 |
| 관리자 메뉴 그룹 | 대시보드 / 연차 관리 / 업무신청관리 / FAQ 관리 — 관리자 계정 로그인 시에만 표시 |
| 직원 메뉴 그룹 | 대시보드 / 업무신청 / 프로필 관리 — 직원 계정 로그인 시에만 표시 |
| 공통 메뉴 | MY인사 소개 / 모바일 안내 |
| 사용자 카드 | 하단 로그인 사용자 이름·직책 표시 |

> **PC 공통 화면설계서**: PC 전 화면이 공유하는 레이아웃·툴바·테이블 그리드·데이터 표시 요건은 `screens/P-000_PC공통.md`에 별도 정리한다(디자인 토큰·CSS·그리드 구현 세부는 별도 퍼블리싱 규칙 문서 소관). 본 절(4.x) 화면별 설계는 해당 공통 규칙을 전제로 하며, 특히 아래 두 공통 요건을 모든 PC 화면에 적용한다.
> - **부서는 선택 항목**: 부서는 경리나라 직원관리 필수 등록 항목이 아니므로 부서명이 없는 직원이 존재할 수 있다. 부서 정보가 없으면 테이블 부서 컬럼·메타는 **공란 처리하거나 부서 토큰·구분자(· )를 생략**하며(빈칸·`undefined`·`null` 미노출), 부서 없음이 목록·집계 누락 사유가 되지 않는다. (부서 필터는 PC 전 화면 미제공)
> - **테이블 그리드 컬럼 리사이징**: 모든 데이터 테이블 그리드는 헤더 셀 우측 경계 드래그로 **컬럼(항목) 너비를 조절**할 수 있다(마지막 컬럼은 잔여 폭 차지, 최소 너비 이하로 축소 불가). 단, 컬럼 너비 조절 시 **모든 컬럼 폭의 합이 테이블 표시 영역을 넘지 않도록** 한다(특정 컬럼을 넓히면 인접 컬럼이 줄어 전체 폭 유지 — 컬럼이 테이블 영역을 벗어나 잘리지 않음). 상세는 `P-000_PC공통.md` 3.1.
> - **셀 내 버튼·태그 컬럼 폭**: 셀에 [수정] 등 **버튼이나 상태 태그(비텍스트 인라인 요소)** 가 들어가는 컬럼은 **`요소 폭 + 좌우 셀 패딩` 이상**으로 너비를 확보한다. 고정 폭(`table-layout:fixed`) 그리드에서 컬럼 폭이 부족하면 버튼/태그가 셀을 넘쳐 한 줄 말줄임 규칙에 걸려 **요소 우측에 말줄임표(`…`)가 표기**되므로 주의한다(예: P-004 FAQ 관리 액션 컬럼). 상세는 `P-000_PC공통.md` 3.
> - **최소 지원 해상도**: PC 웹앱 최소 지원 브라우저 가로 해상도는 **1280 px**다(세로 높이 요건 없음). 상세는 `P-000_PC공통.md` 1.4.

| 상단 툴바 | 연도 선택 | 현재 연도 기준 최근 2년 동적 표시 (*예시: 2025 / 2026*) |
| 상단 툴바 | 직원명 검색 | 최대 50자 / 1자 이상 입력 시 실시간 필터링 (300 ms 디바운스) / × 버튼으로 초기화 |
| 상단 툴바 | [+ 일반근로자 불러오기] 버튼 | 경리나라 직원관리의 일반근로자 목록 호출 후 일괄 등록 |
| 상단 툴바 | [엑셀 다운로드] 버튼 | |

| 연차 현황 테이블 | 총 연차 셀 | 더블클릭 시 직접 편집 가능 (**소수점 입력 허용**, 엔터/포커스 이탈 시 저장. 인사급여 연동 고객사는 편집 비활성화) |
| 연차 현황 테이블 | 월별 셀 | 마우스 오버 시 해당 월 **사용·취소 이력** 툴팁 표시. 셀 값은 `사용 − 취소` 순값 |
| 우측 상세 패널 | 직원 기본 정보 + 연차 통계 | 직원명 클릭 시 슬라이드 오픈 |
| 우측 상세 패널 | 결재 현황 목록 (조회 전용) | 선택 연도의 휴가신청서(연차 사용·취소) 결재 건 표시. **결재 처리 버튼 없음** — 실제 승인·반려는 결재함 앱에서 수행 |

**사용자 흐름**
- 직원명 클릭 → 우측 상세 패널 오픈 (조회 전용)
- [+ 일반근로자 불러오기] 클릭 → 경리나라 직원관리의 일반근로자 목록 조회 → 직원 일괄 등록 팝업 오픈

**연차 집계·계산 규칙**
- 연차 관리 진입 시 각 직원의 사용·취소 연차를 **월별로 집계**: 월별 셀 = `사용 − 취소`, 사용 = `총 사용 − 총 취소`, **잔여 = 총 연차 − (사용 − 취소)**.
- 경리나라 결재함에서 **최종결재(접수)가 완료된 건만** 집계한다. 결재 대기 / 접수 대기 / 반려 건은 사용·취소·잔여에 반영하지 않는다.
- 월별 셀 툴팁: 사용 이력 `mm월 dd일 {휴가종류}`, 취소 이력 `mm월 dd일 {휴가종류(연차/오전반차/오후반차)} 취소`. **일수 괄호(`(N일)`)는 표기하지 않으며, 여러 날 연속 연차기간은 시작일로 줄이지 않고 포함 날짜를 한 줄에 하나씩 모두 나열**한다(예: 04월 10일~04월 11일 → `04월 10일 연차` / `04월 11일 연차`).

**조건부 규칙**
- 결재 이력 없는 직원 패널: "결재 이력이 없습니다" 안내 표시
- **부서 필터 미제공**: 경리나라에 부서 공식 등록 메뉴가 없어 부서 필터 드롭다운은 제공하지 않는다. 테이블의 부서 컬럼은 표시용으로 유지한다.
- 일반근로자 불러오기 모달 요약 배너: `{yyyy}년 미등록된 경리나라 일반근로자 {N}명이 확인됩니다` — {yyyy}는 상단 연도 필터에서 현재 선택된 연도를 반영.

**데이터 동기화 규칙**
- 경리나라 **인사급여** 이용 고객사의 경우, 직원별 연차 데이터(총 연차·사용·잔여)는 인사급여의 연차와 자동 동기화된다.
- 인사급여 연동 고객사는 총 연차 셀 더블클릭 수동 편집이 비활성화된다 (인사급여 데이터가 원천).
- 인사급여 미사용 고객사는 총 연차 셀 더블클릭 수동 편집 방식을 그대로 유지한다.

## 5. 유효성 규칙 및 오류 처리

### 5.1 공통 유효성 표시 원칙

- 필수 항목은 레이블 옆에 빨간 `*` 표시
- 미입력 또는 오류 발생 시: 해당 입력 필드 테두리 빨간색으로 강조 + 필드 아래 오류 안내 문구 표시
- 진행 버튼 비활성화: 필수 조건 미충족 시 버튼 비활성 처리
- [확인] 다이얼로그: 삭제 등 되돌릴 수 없는 액션 전에 한 번 더 확인

### 5.2 화면별 유효성 및 오류 케이스

### 5.4 텍스트 오버플로우 처리 상세

#### 5.4.1 목록·카드·테이블 셀의 말줄임 기준

| 텍스트 유형 | 말줄임 최대 줄 수 | 툴팁 트리거 (모바일) | 툴팁 트리거 (PC) |
|----------|--------------|-----------------|--------------|
| 게시글 제목 (목록) | 1줄 | 길게 누르기 500 ms | 마우스 오버 |
| FAQ 질문 제목 (목록) | 2줄 | 길게 누르기 500 ms | 마우스 오버 |
| 사용처·용도 (목록) | 1줄 | 길게 누르기 500 ms | 마우스 오버 |
| 사업장명 (팝업·헤더) | 1줄 | 길게 누르기 500 ms | 마우스 오버 |
| 결의서 제목 (목록) | 1줄 | 길게 누르기 500 ms | 마우스 오버 |
| 테이블 셀 — 일반 (PC) | 1줄 | — | 마우스 오버 |
| 상태 뱃지 텍스트 | 말줄임 없음 (고정 텍스트) | — | — |
| 직원명·이름 | 말줄임 없음 (짧음) | — | — |

#### 5.4.2 툴팁 세부 규격

**모바일 툴팁**
- 트리거: 해당 요소 위에서 500 ms 이상 누르고 있을 때
- 해제: 손가락 떼거나 다른 영역 탭 시
- 스타일: 반투명 검정(#000000, opacity 80%) 배경 · 흰색 텍스트 · 모서리 반경 8 px · 패딩 8 px 12 px
- 최대 너비: 280 px · 자동 줄바꿈 · 최대 4줄 표시 (초과 시 말줄임)
- 위치: 요소 위 또는 아래 (화면 경계 감지 후 자동 선택)

**PC 툴팁**
- 트리거: 마우스 포인터가 해당 요소 위에 올라갔을 때 (200 ms 딜레이)
- 해제: 포인터가 요소 밖으로 이동 시
- 스타일: 시스템 기본 `title` 속성 또는 동일한 커스텀 툴팁 (디자인 시스템과 협의)
- 최대 너비: 320 px · 자동 줄바꿈

#### 5.4.3 단일 라인 입력 필드 포커스 해제 후 표시

입력값이 필드 너비보다 길 때 포커스 해제 시 텍스트를 시각적으로 말줄임(…) 처리한다. 이때:
- 실제 저장·전송되는 값은 말줄임되지 않은 전체 텍스트
- 사용자가 다시 포커스하면 전체 텍스트가 커서와 함께 표시됨
- 필드 우측 끝에서 말줄임 처리 (앞부분이 보이도록)
