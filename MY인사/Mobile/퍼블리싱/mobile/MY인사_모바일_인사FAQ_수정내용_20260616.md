# MY인사 모바일 — 인사 FAQ 변경 상세 (행 단위 AS-IS / TO-BE)

> 원본 기준: **인사 FAQ 작업 전** `index_m.html` (`MY인사/Mobile/퍼블리싱/mobile/index_m.html`)
> 수정본: **인사 FAQ 작업 후** `index_m.html`

## 이 문서는
- 인사 FAQ 화면을 **"게시글 목록 → 게시글 상세 이동" 방식에서 "목록 내 인라인 아코디언으로 답변 펼침" 방식으로 전환**하면서, `index_m.html`의 **몇 행이 / 무엇에서(AS-IS) / 무엇으로(TO-BE)** 바뀌었는지를 정리한 목록입니다.
- 행 번호 = **원본(작업 전) 파일 기준**. 줄바꿈(LF/CRLF) 차이는 무시했습니다.
- 종류: `수정`=내용 교체 · `추가`=새 코드 삽입 · `삭제`=원본 줄 제거.
- 기능 요건: 질문 행 탭 → 별도 상세 화면 없이 같은 자리에서 답변 펼침/접힘(한 번에 1개) · 공지 핀 제거 · 게시일 `YYYY.MM.DD` · 댓글 미표시(읽기 전용 Q&A) · 태블릿(≥900px) 더보기(좌)+FAQ 목록(우) 분할 유지.

## CSS · rules · 화면설계 (참고)
- `css/myinsa-mobile.css` : **내용 변경 없음**. 기존 정의된 `.post-row` / `.faq-item`(미사용) 등만 활용했고, 신규 스타일은 모두 인라인 스타일로 처리.
- `rules/rules_*.md` : **내용 변경 없음**.
- 즉 실제 코드 변경은 **`index_m.html` 한 파일에만** 있습니다.
- 화면설계서(`04_화면설계서/M-더보기_알림.md`) · PRD(`02_PRD/M-게시판.md`)는 별도로 동기화 완료(본 문서는 HTML 변경만 다룸).

## [스크립트] 아코디언 함수 (`toggleFaqAcc` 등)

### [변경1] 추가 — 원본 3842행 뒤 (`toggleFAQ` 함수 다음)

별도 상세 화면 없이 목록 내에서 답변을 펼침/접힘하는 함수 신규 추가(한 번에 1개만 펼침, 답변 본문은 `_POST_DETAIL` 재사용).

**AS-IS (작업 전)**
```js
function toggleFAQ(el){
  const item = el.parentElement;
  const opened = item.classList.contains('open');
  // 한 번에 1개만 펼침
  document.querySelectorAll('.faq-item.open').forEach(i=>i.classList.remove('open'));
  if(!opened) item.classList.add('open');
}
```

**TO-BE (수정본)**
```js
function toggleFAQ(el){
  const item = el.parentElement;
  const opened = item.classList.contains('open');
  // 한 번에 1개만 펼침
  document.querySelectorAll('.faq-item.open').forEach(i=>i.classList.remove('open'));
  if(!opened) item.classList.add('open');
}
/* 인사 FAQ 목록 — 항목 탭 시 별도 상세 화면 없이 인라인 아코디언으로 답변 펼침 (한 번에 1개) */
function _collapseFaqAcc(){
  document.querySelectorAll('#faqPostList .faq-acc.open').forEach(it=>{
    it.classList.remove('open');
    const p = it.querySelector('.faq-acc-panel'); if(p) p.style.display='none';
    const a = it.querySelector('.faq-acc-arrow'); if(a) a.style.transform='';
  });
}
function toggleFaqAcc(headerEl, id){
  const item = headerEl.parentElement;            // .faq-acc
  const panel = item.querySelector('.faq-acc-panel');
  const arrow = headerEl.querySelector('.faq-acc-arrow');
  const isOpen = item.classList.contains('open');
  _collapseFaqAcc();                              // 다른 항목 닫기
  if(!isOpen){
    if(panel && !panel.dataset.loaded){           // 답변 본문은 _POST_DETAIL 재사용 (최초 1회 주입)
      const data = _POST_DETAIL[id];
      panel.innerHTML = (data && data.body) ? data.body : '';
      panel.dataset.loaded = '1';
    }
    item.classList.add('open');
    if(panel) panel.style.display = '';
    if(arrow) arrow.style.transform = 'rotate(180deg)';
  }
}
```

## [스크립트] `goPostList` (태블릿 분할 + 아코디언 초기화)

### [변경1] 수정 — 원본 3396~3401행

태블릿(≥900px) **더보기(좌) + 인사 FAQ 목록(우) 50:50 분할(`has-faq`)은 유지**하면서, 인사 FAQ 진입 시 아코디언 열림 상태를 초기화하도록 보강.

**AS-IS (작업 전)**
```js
  // 게시글 상세 내용은 openPost() 호출 시 세팅
  if(window.innerWidth >= 900 && isFaqBoard){
    document.querySelector('.app').classList.add('has-faq');
  } else {
    showScreen('post-list');
  }
```

**TO-BE (수정본)**
```js
  // 인사 FAQ 진입 시 아코디언 열림 상태 초기화 (답변은 우측 패널 안에서 인라인으로 펼침)
  if(isFaqBoard) _collapseFaqAcc();
  // 태블릿(≥900px) 분할: 인사 FAQ는 더보기(좌) + FAQ 목록(우) 50:50 분할. 그 외/모바일은 단일 목록 화면
  if(window.innerWidth >= 900 && isFaqBoard){
    document.querySelector('.app').classList.add('has-faq');
  } else {
    showScreen('post-list');
  }
```

## 검증 결과 (프로토타입 동작)
- 모바일(375px): 질문 행 탭 → 같은 자리에서 답변 펼침/접힘, 다른 항목 탭 시 이전 항목 자동 접힘(한 번에 1개), 상세 화면 이동 없음. 공지 핀 0개·게시일 `YYYY.MM.DD` 표기·댓글 미표시·글쓰기 FAB 숨김. 콘솔 에러 없음.
