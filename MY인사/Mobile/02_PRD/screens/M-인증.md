## 데이터 표시 기준

**① 데이터 출처**
- 로그인 인증·계정 잠금·아이디/비밀번호 찾기는 **경리나라 로그인 정책**에 연동된다. 사업장 목록은 **경리나라**에서 해당 계정에 연결된 사업장이다. MY인사는 인증 결과만 받아 화면을 전환한다.
- 로그인 아이디는 **경리나라 계정 아이디**로 인증한다. 아이디는 원칙적으로 이메일 형식이나 이메일 구조가 아닌 아이디 사용자도 존재하므로 아이디·이메일을 모두 입력 가능하도록 한다. **사번은 사용하지 않는다.**

**② M-001 로그인 (`login`)**
- 로고: 아이콘 `MY` + `MY인사` + 서브 `중소기업용 통합 HR 솔루션`(정적).
- 아이디 필드: placeholder `아이디 또는 이메일`, 최대 100자(영문·숫자·특수문자).
- 비밀번호 필드: 가림 처리 + 표시/숨기기 토글, 최대 50자.
- 자동 로그인 체크박스: **기본 미체크**. 체크 시 다음 실행에서 로그인 화면 생략. **여러 사업장 로그인 가능 계정이면 사업장 선택 팝업 없이 가장 최근 로그인했던 사업장으로 자동 로그인.**
- 실패 처리: 인증 실패 시 `아이디 또는 비밀번호가 올바르지 않습니다`(화면 유지·비밀번호 필드 초기화·횟수 누적). **5회 연속 실패** 시 danger 박스 `로그인 시도 횟수를 초과하였습니다. 계정이 잠겼습니다. 관리자에게 문의하세요.` + 계정 잠금(경리나라 로그인 정책 연동).

**③ M-001-P 사업장 선택 팝업**
- 헤더: 소제목 `로그인` / 제목 `사업장을 선택하세요` / 부제 `{이름} 님이 소속된 사업장 목록입니다`.
- 목록 항목: **사업장명** + `{사업자번호} · {가입 제품명}`(예: `we경리나라`) + 우측 화살표. 경리나라에서 계정에 연결된 사업장.
- 표시 규칙: 연결 사업장 1개면 팝업 생략·바로 진입, 2개 이상이면 팝업. 항목 탭 → 해당 사업장으로 `board` 이동, [취소] → 로그인 화면 유지.

**④ M-009 아이디 찾기 / 비밀번호 변경 (`pw-find`)**
- 진입 모드 고정(아이디 찾기 / 비밀번호 변경), 타이틀 동적 표시. 인증 방식 탭: `휴대폰` / `이메일`.
- 입력: 성명(공통) / 아이디(비밀번호 변경 모드만) / 휴대폰번호(국가코드 `+82` 고정 + [인증]) / 이메일 / CAPTCHA(이미지 + 새로고침 + 텍스트 입력) / 인증번호(+[확인]).
- 토스트: 인증번호 발송 `인증번호가 발송되었습니다` / 인증 확인 `인증되었습니다` / CAPTCHA 새로고침 `새 CAPTCHA가 생성되었습니다`.
- 입력 누락 검증: [찾기]/[변경] 탭 시 필수 항목을 위에서부터 검사 → 첫 미입력 항목에 대해 팝업 `{항목명}이(가) 입력되지 않았습니다.`(받침에 따라 이/가) 표시 후 중단. 항목 순서 — 휴대폰: 성명·휴대폰번호·자동입력 방지문자·인증번호 / 이메일: 성명·이메일 / 비밀번호 변경은 아이디 선행.
- [찾기] → **별도 토스트 없이** 곧바로 **결과 팝업**(제목 `아이디 찾기 결과` / `회원님의 아이디는` / 아이디 값 / `가입일: {YYYY.MM.DD}` / [닫기]·[로그인]).
- [변경] → 입력한 아이디·성명·휴대폰/이메일의 계정 일치 검증. **일치** → 새 비밀번호 설정(`pw-reset`), **불일치** → 비밀번호 변경 결과(`pw-change-result`). 임시 비밀번호 발송 방식 아님.

**⑤ M-009-S 새로운 비밀번호 설정 (`pw-reset`)**
- 입력: `비밀번호`(가림, 최대 50자) / `비밀번호 확인`(가림, 일치 필요).
- [변경하기] → 미입력 시 `비밀번호를 입력해주세요.`, 불일치 시 `비밀번호가 일치하지 않습니다.` 팝업(확인 시 화면 유지). 정상 시 `변경이 완료되었습니다.` 팝업 → [확인] → `login`.
- [닫기] → `login` / [이전 페이지] → `pw-find`.

**⑥ M-009-R 비밀번호 변경 결과 (`pw-change-result`)**
- 안내: `입력하신 정보와는 일치하는 사용자가 없습니다.` + 오류 사유(대소문자 오입력, 아이디 뒤 공란).
- `[입력정보]`: 아이디 / 이름 / 휴대폰(또는 이메일) — 직전 화면에서 사용자가 입력한 값을 그대로 표시.
- [다시찾기] → `pw-find` / [취소]·닫기(×) → `login`.
