## 데이터 표시 기준

> 화면별 표시 데이터의 출처·형식·연동 결과를 필드 단위로 정의한다. 직원이 모바일에서 제출한 증명서 신청 건은 관리자용 MY인사 PC의 **업무신청관리(P-003)** 목록에 표시되며, **인사 담당자**가 발급/반려 처리를 수행한다. (별도 '인사팀' 부서로 신청하는 구조가 아니라 인사 담당자가 직접 처리한다.)

### 공통 — 데이터 출처
- **증명서 종류**: 재직·경력·기타 3종 고정. 기타증명서만 [증명서 명칭] 직접 입력.
- **신청자**: 로그인 직원 정보(이름).
- **처리 담당자**: 관리자 PC 업무신청관리(P-003)에서 발급/반려 처리한 **인사 담당자**(이름). 부서 단위가 아닌 인사 담당자 개인 기준이며 '담당부서' 개념은 사용하지 않는다.
- **발급 부수**: 수량(부수/매수) 입력·표시 항목 **없음**. 1건 신청 = 1건 발급 기준.
- **상태값**: 진행중 / 발급완료 / 반려. 발급완료·반려 시 앱 푸시 알림 발송 → 알림 행 탭으로 M-034·M-035 딥링크.
- **일시 형식**: 목록 `M월 D일 HH:mm` 또는 `신청/발급/반려 M월 D일`, 상세 `YYYY.MM.DD HH:mm`. 금액·수량 표기 없음.

### M-030 증명서 관리
- 종류 카드: 종류명 + 고정 설명 문구. 발급 이력(인라인): 종류 뱃지·용도·신청/발급/반려 일시·상태 뱃지(부수 없음). **별도 제목행·[전체 보기]·발급 이력 화면 없이** 종류 카드 하단 인라인 목록으로 전체 표시(배경 카드 없음).

### M-031 증명서 신청 (바텀 시트 모달)
- 종류 카드 탭 시 `openCertModal(type)` 오버레이. 명칭(기타 필수)/용도(필수) 입력. 안내 배너는 종류별 예상 처리 시간(재직·경력 당일~1영업일, 기타 검토 후 1~2영업일) + 앱 알림 안내. [발급하기] → M-033. 부수 입력 없음.

### M-033 진행중 / M-034 발급완료 / M-035 반려
- **배경/레이아웃**: 화면 배경은 회색, **정보 블록(상태 헤더·신청/발급 정보·처리 이력·메모)만 흰색 카드**(블록 간 회색 간격). 안내 배너·반려 사유 박스는 고유 색 유지. 신청/발급 정보 섹션에는 제목(`신청 정보`/`발급 정보`)을 표시.
- **M-033**: 진행중 뱃지 + 신청 정보(종류·용도·신청일시·담당자(인사 담당자)) + 처리 이력 타임라인(접수→검토) + 안내 배너. 읽기 전용.
- **M-034**: 발급 정보(종류·용도·발급일시·처리 담당자, 부수 없음 / **직인 적용 여부 표시 없음**) + 처리 이력 타임라인 + 인사담당자 메모. [저장(다운로드)]=PDF, [이메일로 전송]=바텀 시트(수신 이메일 직접 입력·첨부 파일명/용량 표시·발송하기).
- **M-035**: 반려 사유 박스(담당자 메시지·일시) + 신청 정보 + 인사담당자 메모 + 처리 이력 타임라인(접수→반려). 읽기 전용(하단 버튼 없음). 재신청은 M-030에서 재신청.
