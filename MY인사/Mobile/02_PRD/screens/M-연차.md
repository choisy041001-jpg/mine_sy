## 데이터 표시 기준

**① 데이터 출처**
- 잔여 연차·사용 일수·연차 이력은 **경리나라**에서 관리하는 현재 로그인 사업장·로그인 직원 기준 데이터를 API로 연동해 표시한다. 신청·취소의 실제 결재(승인·반려)는 경리나라 **결재함** 앱에서 처리되며, MY인사는 결재 결과를 받아 상태를 표시한다.
- 결재선(신청자→결재자)은 **경리나라를 기준으로 자동 지정**되며, 결재선이 2개 이상인 경우에만 [결재선 변경]으로 라디오 버튼 팝업에서 선택할 수 있다.

**①-1 경리나라 전자결재 상신 매핑**
- **연차 신청(M-011)** 상신 시 → 경리나라 전자결재에 **휴가신청서** 문서로 상신된다. 매핑: `신청 구분` = **신청**, `휴가 기간·일수` = 연차 신청 화면에서 지정한 기간·일수, `휴가 종류` = **항상 `연차` 타입**(반차 등 다른 타입으로 매핑하지 않음).
- **연차 취소 신청(M-014)** 상신 시 → 동일하게 **휴가신청서** 문서로 상신된다. 매핑: `신청 구분` = **취소**, `휴가 기간·일수` = 취소 신청 화면에서 지정한 기간·일수, `휴가 종류` = **항상 `연차` 타입**.
- 취소 신청 날짜는 **승인된 연차 날짜로 제한하지 않는다.** 사용자가 지정한 기간·일수를 그대로 취소 휴가신청서로 상신한다.

**② M-010 연차 관리 (`leave-approval`)**
- 타이틀 `연차 관리` + 부제 `연차 신청 및 사용 내역`.
- 잔여 연차 카드: `잔여 일수`(대형 숫자, 예 `12일`) / `총 연차 {N}일` · `사용 {N}일` / 진행 막대(사용/총 비율). 잔여 0일이면 막대 0%.
- 잔여 연차 카드: `잔여 연차` 라벨을 카드 좌측 상단(대형 숫자 위)에 배치.
- 연차 이력 행(헤더·섹션 구분 없는 구분선형 — **별도 섹션 헤더 없음**, 날짜 역순): `{문서명}`(예 `휴가신청서`) / **`{사용일자} {휴가 종류} · {사용 일수}`**(예 `6월 10일 연차 · 1일`) / `상신 {M월 D일} · 결재자 {결재자명}` / 상태 뱃지(`진행중`·`승인`·`반려`) + 우측 화살표. 이력 없으면 `신청한 연차 내역이 없습니다`.
- FAB(+): `연차 신청`(→ `leave-apply`) / `취소 신청`(→ `leave-cancel`).

**③ M-011 연차 신청 (`leave-apply`)**
- 타이틀 `휴가신청서`. 전체 화면 전환.
- 기간: 시작일~종료일 date 입력 + 캘린더(이전/다음 월, 오늘 표시, **드래그로 기간 선택**(또는 탭)·구간 하이라이트). **점(●) 표시 없음.**
- 휴가 종류 토글(필수): `연차`(1일) / `오전 반차`(0.5일) / `오후 반차`(0.5일). **반차는 하루만 선택**(범위 상태에서 반차 전환 시 시작일만 유지 + 알림 `반차는 하루만 선택해주세요.`).
- 사유(필수): 텍스트에어리어, 최대 200자·카운터.
- 비상 연락망(선택): 텍스트 입력, placeholder `예) 010-1234-5678`. *(프로토타입은 자릿수·하이픈 자동 미구현 — 목업)*
- 잔여 연차 미리보기: `현재 잔여 {N}일` → `신청 시 {차감}일 차감 → 잔여 {N}일`.
- 결재선: `홍길동(신청자) → 김팀장(1차 결재)` + `결재선은 경리나라를 기준으로 자동 지정됩니다` + [결재선 변경].
- [신청하기] → `✅ 결재 상신이 완료되었습니다` + `leave-approval` 복귀. 상신 시 휴가신청서(신청 구분 `신청`, 종류 `연차`)로 전자결재에 등록(①-1). 날짜 미선택·사유 미입력 시 진행 불가, 잔여 부족 시 인라인 경고·버튼 비활성화.

**④ M-014 연차 취소 신청 (`leave-cancel`)**
- 타이틀 `연차 취소 신청`. 안내 배너(warning): `취소할 연차 날짜를 선택하세요. 선택한 기간·일수만큼 취소 신청이 상신됩니다.`
- 취소할 날짜(필수): 시작일~종료일 date 입력 + 캘린더(탭/드래그 범위 선택). 하단에 `선택: {YYYY년 M월 D일 (요일)}` 표시. **승인된 연차 날짜로 제한하지 않으며, 별도 점(●) 표시·범례 없음.**
- 휴가 종류 토글(필수, **기본값 `연차`**): `연차`(1일) / `오전 반차`(0.5일) / `오후 반차`(0.5일) — 취소 대상 휴가 종류 선택. 반차는 하루만(M-011 동일). 복구 일수도 종류 따라 1일/0.5일.
- 취소 사유(필수): 토글 `일정 변경` / `업무 일정` / `개인 사정` / `기타` + 텍스트에어리어(추가 입력, 선택).
- 취소 후 연차 현황: `현재 잔여 연차 {N}일` / `취소 후 복구 +{N}일` / `취소 후 잔여 {N}일`.
- 결재선: M-011과 동일(자동 지정 + [결재선 변경]).
- [취소 신청하기] → `✅ 취소 신청이 상신되었습니다` + `leave-approval` 복귀. 상신 시 휴가신청서(신청 구분 `취소`, 종류 `연차`)로 전자결재에 등록(①-1).

**⑤ M-015 / M-016 / M-017 휴가신청서 상세 (`leave-detail-wait` / `leave-detail-done` / `leave-detail-reject`)**
- 타이틀 `휴가신청서` + 상단 상태 뱃지(진행중 주황 / 승인 초록 / 반려 빨강). 앱 푸시 알림 탭으로 직접 진입 가능(M-016·M-017).
- 상태 배너 안내: 진행중 `결재 진행중 / {결재자명} 확인 전입니다` / 승인 `{결재자명}이 승인하였습니다` / 반려 `{결재자명}이 반려하였습니다`.
- 배경: 상세 본문은 흰색, **연차 현황 영역만 배경 비움**(앱 기본 배경)으로 구분(M-000 §2.1). 3개 화면 공통.
- 결재선: 신청자→**결재/접수 다단계** 표시. 각 단계는 **결재(결재자)·접수(접수자)**, 역할·단계별 뱃지(`상신완료`·`진행중`·`승인`·`반려`·`접수완료`·`대기`), 단계 많으면 가로 스크롤(M-000 §6).
- 신청 내용: `신청 유형 {휴가신청서}` / `휴가 종류 {예: 종일 (1일) / 오전 반차 (0.5일)}` / `날짜` / `사용 일수` / `사유` / `메모`(사유와 별도).
- 연차 현황 패널: **진행중·승인 화면만** 표시(반려 화면 제외).
- 처리 이력 타임라인: 상신·결재 처리 단계와 일시(`YYYY.MM.DD HH:MM · {결재자명}`).
- M-017 반려: 추가로 **반려 사유 박스**(붉은 박스 안에 `반려 사유` 뱃지 + 반려 메시지 + `{처리 일시} · {반려 처리자}`) 표시, 연차 현황 패널 없음.
