## 데이터 표시 기준

> M-006/M-008(인사 FAQ)의 데이터 표시 기준은 **M-FAQ** 문서에서 정의한다. 본 섹션은 게시판(M-002~M-005)을 다룬다.

**① 데이터 출처**
- 게시판·게시글·댓글은 **경리나라 게시판 앱**에서 관리되며, MY인사 앱은 API로 연동해 **현재 로그인한 사업장 기준**으로 표시한다. 게시판 생성·삭제 등 운영 관리는 게시판 앱에서 수행한다. 게시판 목록은 API 동적 로드한다.

**② M-002 게시판 홈 (`board`)**
- 타이틀 `게시판` + 부제 `우리 회사 소식과 자유로운 대화`(필터 적용 시 게시판명으로 대체).
- 필터(≡): 게시판 선택 바텀 시트(`전체`+운영 게시판), 적용 중이면 아이콘 강조. 알림(🔔): 미읽음 1건 이상 시 빨간 점 뱃지, 전체 읽음 시 소멸.
- 피드 행: `제목`(+새 글 `N` 뱃지) / `{게시판명} · {작성자}` / `{M월 D일}`. 빈 상태 `등록된 게시글이 없습니다.`. FAB(+) → `compose`.

**③ M-003 게시글 목록 (`post-list`)**
- 상단 게시판명. 행: `제목`(공지 글 `공지` 핀·상단 고정) + `{작성자} · {M월 D일} · 댓글 {N}`. 인사 FAQ 채널은 글쓰기 FAB 숨김.

**④ M-004 게시글 상세 (`post-detail`)**
- 본문: `제목`(공지 핀) / `{작성자} · {등록일시 YYYY.MM.DD HH:MM}` / 본문 / 첨부 이미지. **조회수 미표시.**
- 댓글: 헤더 `댓글 {N}`, 각 댓글 = **이름 + 부서 + 내용 + 작성시간**(아바타 없음). 내 댓글은 더보기(···) → 수정/삭제. 입력창은 최대 300자·카운터(프로토타입은 목업).
- 본인 글 더보기 → 수정(`post-edit` pre-fill)/삭제 확인 팝업.

**⑤ M-005 글 작성 (`compose`)**
- 게시판 선택(필수, 인사 FAQ 제외) / 제목(필수·100자) / 본문(필수·3,000자·카운터) / 이미지 첨부(최대 5장·JPG·PNG·10 MB·썸네일 미리보기, 프로토타입은 버튼 토스트만).
- [등록] → 검증 후 `✅ 게시글이 등록되었습니다` + `board` 복귀. [취소] → 입력 있으면 `작업을 취소하시겠습니까?` 확인.
