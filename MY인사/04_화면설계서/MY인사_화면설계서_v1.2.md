## 1. 문서 개요

### 1.1 목적

본 문서는 MY인사의 모바일 앱 및 PC 웹앱 화면을 기획자 관점에서 정의한다. 각 화면의 목적, 구성 요소, 사용자 흐름, 노출 조건, 유효성 규칙을 기술하며, 디자이너·개발자·QA가 화면을 구현하고 검증하는 기준으로 활용된다.

### 1.2 서비스 구성

| 구분 | 플랫폼 | 대상 사용자 | 주요 기능 |
|------|--------|-----------|---------|
| 모바일 앱 | 모바일 웹앱 (375 px) | 직원 | 게시판 · 연차 신청/조회 · 영수증 등록/결의서 · 증명서 신청 · FAQ · 내 정보 |
| PC 웹앱 | 웹 브라우저 (1280 px 이상 권장) | 관리자 (인사·경리담당자) | 연차 결재 · 증명서 발급 · FAQ 관리 · 대시보드 |
| PC 웹앱 | 웹 브라우저 | 직원 | 연차 조회 · 증명서 신청 · 프로필 관리 |

### 1.3 용어 및 공통 규칙

| 용어/규칙 | 설명 |
|----------|------|
| 필수 항목 | 입력하지 않으면 다음 단계로 진행되지 않는 항목. 항목 레이블 옆에 빨간 `*` 표시 |
| 선택 항목 | 입력하지 않아도 진행 가능. 항목 레이블에 `(선택)` 표시 |
| 상태 뱃지 | 처리 상태를 색상 태그로 표시. **진행중**(주황) · **승인/완료**(초록) · **반려**(빨강) |
| FAB | 화면 우하단에 고정된 주요 액션 버튼(+). 탭 시 세부 액션 목록 표시 또는 직접 이동 |
| 탭바 | 모바일 하단 고정 네비게이션. 5개 탭: 게시판 · 연차 · 영수증 · 증명서 · 더보기 |
| 결재선 | 경리나라 시스템에서 자동 지정. 사용자가 직접 변경 불가 |
| 사업장 | 1개 계정으로 여러 사업장 관리 가능. 로그인 시 사업장 선택, 앱 내 전환 가능 |
| 빈 상태 | 목록/검색 결과가 없을 때 표시하는 안내 화면 (일러스트 + 안내 문구) |
| 로그인 유지 | 자동 로그인 체크 시 재접속 시 로그인 생략 |

## 3. 모바일 화면 상세 설계

### 3.1 인증

---

#### M-001 로그인

| 항목 | 내용 |
|------|------|
| **화면 목적** | 사용자가 아이디·비밀번호로 인증하고, 소속 사업장을 선택하여 앱에 진입한다. |
| **진입 조건** | ① 앱 최초 실행 시 ② 로그아웃 후 ③ 세션 만료(자동 로그인 미설정 시 앱 재실행) 시 → 하단 탭바 미표시 |
| **다음 화면** | 사업장 선택 팝업 M-001-P |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 로고 영역 | 앱 아이콘 + 앱 이름 | 파란 원형 아이콘 + "MY인사" |
| 입력 영역 | 아이디 입력 필드 | placeholder "이메일 또는 사번" |
| 입력 영역 | 비밀번호 입력 필드 | 텍스트 가림 처리 + 우측 표시/숨기기 토글 아이콘 |
| 옵션 영역 | 자동 로그인 체크박스 | 기본값: 미체크 |
| 액션 영역 | [로그인] 버튼 | 화면 전체 너비 |
| 액션 영역 | 비밀번호 찾기 링크 | 버튼 하단 중앙 정렬 |

**사용자 흐름**

1. 아이디 + 비밀번호 입력
2. [로그인] 버튼 탭
   - 인증 성공 → 사업장 선택 팝업(M-001-P) 표시
   - 인증 실패 → 에러 메시지 표시 (화면 유지)
3. 사업장 선택 → 게시판 홈(M-002) 진입

**조건부 규칙**
- 자동 로그인 체크 시: 다음 앱 실행 시 로그인 화면 건너뛰고 사업장 선택 팝업 또는 마지막 사업장으로 바로 진입
- 비밀번호 표시/숨기기: 토글 아이콘 탭마다 전환

**유효성 규칙**

| 오류 조건 | 오류 표시 위치 | 오류 메시지 | 처리 결과 |
|---------|-------------|-----------|---------|
| 아이디 또는 비밀번호 불일치 | 비밀번호 입력 필드 하단 | "아이디 또는 비밀번호가 올바르지 않습니다" | 비밀번호 필드 초기화, 화면 유지 |

---

#### M-001-P 사업장 선택 팝업 `[팝업 오버레이]`

| 항목 | 내용 |
|------|------|
| **화면 목적** | 로그인 성공 후, 해당 계정에 연결된 사업장 중 하나를 선택하여 해당 사업장 기준으로 앱을 사용한다. |
| **진입 조건** | 로그인 버튼 탭 → 인증 성공 시 현재 화면 위에 오버레이로 표시 |
| **다음 화면** | 게시판 홈 M-002 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 헤더 영역 | 소제목 | "로그인" |
| 헤더 영역 | 제목 | "사업장을 선택하세요" |
| 헤더 영역 | 부제 | "{이름} 님이 소속된 사업장 목록입니다" |
| 사업장 목록 | 항목 | 사업장명 / 사업자번호 · 가입 제품명(we경리나라) + 우측 화살표(→) |
| 하단 버튼 | [취소] 버튼 | 팝업 닫기, 로그인 화면 유지 |

**사용자 흐름**
- 사업장 항목 탭 → 해당 사업장으로 로그인 → 게시판 홈(M-002) 이동
- [취소] 탭 → 팝업 닫힘, 로그인 화면 유지 (재로그인 가능)

**조건부 규칙**
- 소속 사업장이 1개뿐인 경우: 팝업 없이 바로 해당 사업장으로 진입 (UX 간소화)
- 소속 사업장이 2개 이상인 경우: 팝업 표시
- 목록이 일정 높이 초과 시: 내부 스크롤 처리

### 3.2 게시판

---

#### M-002 게시판 홈

| 항목 | 내용 |
|------|------|
| **화면 목적** | 전 게시판의 최근 게시글을 피드 형태로 보여주는 앱의 기본 진입 화면. |
| **진입 조건** | 로그인 → 사업장 선택 후 / 하단 탭바 [게시판] 탭 탭 |
| **다음 화면** | 게시글 목록 M-003 / 글 작성 M-005 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 | "게시판" |
| 상단 영역 | 부제 | "우리 회사 소식과 자유로운 대화" |
| 상단 영역 | 필터 아이콘(≡) | 우측 상단 — 게시판 선택 시트 열기 |
| 게시글 목록 | 게시글 행 | 제목 / NEW 뱃지 / 게시판명·작성자 / 날짜 / 화살표 |
| 게시판 선택 시트 | 바텀 시트 | 전체 / 자유게시판 / 경영지원부서 공지 / 경조사게시판 / 일일업무현황 / 사내규정&세칙 |
| 하단 | 글쓰기 FAB(+) | 화면 우하단 고정 |

**사용자 흐름**
- 게시글 행 탭 → 해당 게시판의 게시글 목록(M-003)으로 이동
- 필터(≡) 아이콘 탭 → 게시판 선택 시트 표시 → 게시판 선택 → 해당 게시판 글만 목록 표시
- FAB(+) 탭 → 글 작성(M-005)으로 이동

**조건부 규칙**
- 선택한 게시판에 게시글이 없을 때: "등록된 게시글이 없습니다. 첫 번째 글을 작성해 보세요!" 빈 상태 표시
- 게시판 필터 선택 후: 필터 아이콘 테두리 색 강조 표시 (필터 적용 중임을 알림)
- NEW 뱃지: 읽지 않은 새 게시글에 표시 (빨간 원형 N)

- FAB → [결의서 작성] → 선택 모드 진입 → 선택 후 [결의서 작성 (N건)] 탭 → M-023

**조건부 규칙**

| 좌측 패널 | 프로필 사진 + 인사 기본 정보 | 사번 / 입사일 / 부서 / 직급 / 고용형태 — **읽기 전용** |
| 우측 패널 — 개인 정보 | 수정 가능 항목 | 이름 / 영문 이름 / 이메일 / 연락처 |
| 우측 패널 — 내 인사 정보 | 수정 가능 항목 | 자격증 / 학력 / 어학 |
| 하단 | [수정] 버튼 | 클릭 시 편집 모드 전환 |
| 편집 모드 | 수정 가능 항목 입력 필드 활성화 + 프로필 사진 변경 버튼 | [저장] · [취소] 버튼 |

**조건부 규칙**
- 인사 기본 정보: 읽기 전용, 수정 불가
- [취소] 클릭: 변경 내용 초기화, 저장 전 원본 데이터 복원
- 프로필 사진: 편집 모드에서만 변경 가능

## 5. 유효성 규칙 및 오류 처리

### 5.1 공통 유효성 표시 원칙

- 필수 항목은 레이블 옆에 빨간 `*` 표시
- 미입력 또는 오류 발생 시: 해당 입력 필드 테두리 빨간색으로 강조 + 필드 아래 오류 안내 문구 표시
- 진행 버튼 비활성화: 필수 조건 미충족 시 버튼 비활성 처리
- [확인] 다이얼로그: 삭제 등 되돌릴 수 없는 액션 전에 한 번 더 확인

### 5.2 화면별 유효성 및 오류 케이스

| 화면 ID | 오류 조건 | 오류 표시 방식 | 처리 결과 |
|--------|---------|-------------|---------|
| M-001 | 아이디 또는 비밀번호 불일치 | 비밀번호 필드 하단 오류 문구 | 비밀번호 필드 초기화, 화면 유지 |
| M-005 | 제목·본문 미입력 | 해당 필드 테두리 빨간색 + 오류 문구 | [등록] 진행 불가 |
| M-011 | 날짜 미선택 | "날짜를 선택하세요" 안내 문구 | [신청하기] 진행 불가 |
| M-011 | 잔여 연차 부족 | "잔여 연차가 부족합니다" 경고 문구 박스 | [신청하기] 비활성화 |
| M-013 | 날짜 미선택 | [제출하기] 비활성화 | 진행 불가 |
| M-021 | 사용일자·금액·사용처 미입력 | 해당 필드 테두리 빨간색 + 오류 문구 | [등록하기] 진행 불가 |
| M-021 | 파일 용량 초과 (10 MB) | "파일은 최대 10 MB까지 첨부 가능합니다" 안내 | 파일 첨부 취소 처리 |
| M-023 | 영수증 미선택 | 이전 화면(M-020)에서 진입 자체 차단 | 결의서 작성 화면 진입 불가 |
| M-031 | 용도 미입력 | 해당 필드 테두리 빨간색 + 오류 문구 | [신청하기] 진행 불가 |
| M-031 | 기타증명서 파일 미첨부 | 첨부 영역 강조 표시 | [신청하기] 비활성화 |
| P-003 | 증명서 발급처리 PDF 미첨부 | [발급 완료] 버튼 비활성화 유지 | 발급 완료 진행 불가 |

```
M-001 로그인

  소속 사업장 선택
```

```
  FAB → 연차 신청
    ↓

  날짜·종류·사유 입력
    ↓

```

```
  영수증 선택 후 결의서 작성
```
