## 화면 상세 설계

### M-010 연차 관리

| 항목 | 내용 |
|------|------|
| **화면 목적** | 나의 잔여 연차 현황을 확인하고, 연차 이력 전체를 인라인 스크롤로 조회하며, FAB를 통해 연차 신청·취소·계획서를 작성한다. |
| **진입 조건** | 하단 탭바 [연차] 탭 |
| **다음 화면** | M-011 / M-013 / M-014 / M-015 / M-016 / M-017 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 영역 | 타이틀 | "연차 관리" |
| 잔여 연차 카드 | 잔여 일수 | 크게 강조 표시 |
| 잔여 연차 카드 | 총 연차 · 사용 일수 · 진행 막대 | |
| 연차 이력 | 이력 행 (구분선형 · 전체 너비) | 문서명·종류·일수·상신일·결재자 + 상태 뱃지 + 화살표 / 날짜 역순 / 별도 섹션 헤더 없음 |
| 하단 | 연차 FAB(+) | 연차 신청 / 취소 신청 / 연차사용계획서 |

**사용자 흐름**
- 이력 행 탭: 진행중 → M-015 / 승인 → M-016 / 반려 → M-017
- FAB(+) 탭 → 액션 목록: [연차 신청] → M-011 / [취소 신청] → M-014 / [연차사용계획서] → M-013

**조건부 규칙**
- 잔여 연차 0일: 현황 카드의 진행 막대 0% 표시
- 연차 이력 없을 경우: "신청한 연차 내역이 없습니다" 빈 상태 표시

### M-011 연차 신청

| 항목 | 내용 |
|------|------|
| **화면 목적** | 캘린더에서 날짜를 선택하고 휴가 종류·사유를 입력하여 연차 결재를 상신한다. |
| **진입 조건** | 연차 관리(M-010) FAB → [연차 신청] (**전체 화면 전환** — 바텀 시트 오버레이 아님) |
| **다음 화면** | 연차 관리 M-010 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "휴가신청서" | |
| 기간 입력 | 시작일 ~ 종료일 | 날짜 선택 필드 |
| 캘린더 | 이전/다음 월 이동 버튼 | |
| 캘린더 | 날짜 탭 또는 드래그로 범위 선택 | 선택 구간 하이라이트 표시 |
| 캘린더 | 오늘 날짜 구분 표시 | |
| 캘린더 | 이미 승인된 연차 날짜 | 점(●) 표시 |
| 휴가 종류 | 단일 선택 토글 | 연차 / 오전 반차 / 오후 반차 — **필수** / 오전 반차 선택 시 0.5일, 오후 반차 선택 시 0.5일 차감 |
| 사유 입력 | 텍스트 입력 영역 | **필수** / 최대 200자 / 다중 라인 / 카운터 표시 |
| 비상 연락망 | 텍스트 입력 필드 | 선택 / 최대 20자 / 숫자만 허용 / 하이픈 자동 삽입 (예: 010-1234-5678) |
| 잔여 연차 미리보기 | 신청 시 차감 후 잔여 일수 표시 | |
| 결재선 | 신청자 → 결재자 순서 표시 | 경리나라 자동 지정 안내 문구. **결재선이 2개 이상인 경우 [결재선 변경] 선택 시** 라디오 버튼으로 결재선 선택 팝업 호출 |
| 하단 | [신청하기] 버튼 | 전체 너비 |

**조건부 규칙 (이탈 처리)**
- [← 뒤로] 탭 (날짜·종류·사유 등 하나라도 입력된 경우): "작성 중인 내용이 있습니다. 취소하시겠습니까?" 확인 팝업 → 확인 시 이전 화면(M-010) 복귀 / 취소 시 팝업 닫기

**유효성 규칙**

| 오류 조건 | 오류 메시지 | 처리 결과 |
|---------|-----------|---------|
| 날짜 미선택 | "날짜를 선택하세요" | 진행 불가 |
| 잔여 연차 부족 | "잔여 연차가 부족합니다" 경고 | [신청하기] 비활성화 |
| 사유 미입력 | 해당 필드 오류 표시 | 진행 불가 |

### M-013 연차사용계획서

| 항목 | 내용 |
|------|------|
| **화면 목적** | 월 단위로 사용 예정인 연차 날짜를 미리 계획하여 제출한다. |
| **진입 조건** | 연차 관리(M-010) FAB → [연차사용계획서] |
| **다음 화면** | 연차 관리 M-010 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "연차사용계획서" | |
| 기간 헤더 | 현재 월 표시 (예: 2026년 5월) | 이전/다음 월 이동 버튼 — 월별 보기만 지원 (분기별 탭 없음) |
| 캘린더 | 날짜 개별 탭 또는 드래그로 다중 날짜 선택 | 선택일 강조 |
| 선택 현황 | 선택 날짜 수 + 잔여 연차 수 | |
| 메모 | 계획 사유 텍스트 입력 | 선택 / 최대 200자 / 다중 라인 / 카운터 표시 |
| 결재선 | 신청자 → 결재자 표시 | **결재선이 2개 이상인 경우 [결재선 변경] 선택 시** 라디오 버튼으로 결재선 선택 팝업 호출 |
| 하단 | [제출하기] 버튼 | |

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 날짜 미선택 | [제출하기] 비활성화 |
| 선택 일수가 잔여 연차 초과 | "선택한 날짜가 잔여 연차를 초과합니다" 안내 표시 및 제출 불가 |
| 연속 선택 30일 초과 | "최대 30일까지 선택 가능합니다" 안내 후 31일째 선택 차단 |

### M-015 / M-016 / M-017 휴가신청서 상세 (진행중 · 승인 · 반려)

> 3개 화면은 동일한 레이아웃이며, 처리 상태에 따라 상태 뱃지·안내 문구·하단 영역만 다르게 표시된다.

| 항목 | 내용 |
|------|------|
| **화면 목적** | 결재 현황 카드를 탭했을 때, 해당 신청서의 상세 내용과 처리 상태를 확인한다. |
| **진입 조건** | M-015: 연차 관리(M-010) 이력 행 진행중 탭 / M-016: 이력 행 승인 탭 (또는 앱 푸시 알림) / M-017: 이력 행 반려 탭 (또는 앱 푸시 알림) |

**공통 화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 내비게이션 | [← 뒤로] + "휴가신청서" | |
| 처리 상태 뱃지 | 진행중(주황) · 승인(초록) · 반려(빨강) | |
| 상태별 안내 문구 | 아래 조건부 규칙 참조 | |
| 결재선 | 신청자 → 결재자 | 처리 완료 시 완료 표시 |
| 신청 내용 | 신청 날짜 · 종류(연차/반차) · 사유 | |

**조건부 규칙**

| 화면 | 상태 뱃지 | 안내 문구 | 추가 요소 |
|------|---------|---------|---------|
| M-015 진행중 | 주황 | "결재 진행중 / {결재자명} 확인 전입니다" | 없음 |
| M-016 승인 | 초록 | "{결재자명}이 승인하였습니다" | 결재선 승인 완료 체크 표시 |
| M-017 반려 | 빨강 | "{결재자명}이 반려하였습니다" | 반려 사유 박스(결재자 메시지·처리 일시) |

**사용자 흐름**
- M-015·M-016·M-017: 읽기 전용. [← 뒤로] 탭으로 복귀

## 화면 전환 흐름

```
M-010 연차 관리
  ├─ 이력 행 탭 (진행중) → M-015 휴가신청서 상세 — 진행중
  ├─ 이력 행 탭 (승인)   → M-016 휴가신청서 상세 — 승인
  ├─ 이력 행 탭 (반려)   → M-017 휴가신청서 상세 — 반려
  │
  ├─ FAB → 연차 신청
  │     ↓
  │   M-011 연차 신청
  │     날짜·종류·사유 입력
  │       ↓
  │     M-010 연차 관리 (상신 완료)
  │
  ├─ FAB → 취소 신청
  │     ↓
  │   M-014 연차 취소 신청
  │       ↓
  │     M-010 연차 관리 (상신 완료)
  │
  └─ FAB → 연차사용계획서
        ↓
      M-013 연차사용계획서
        ↓
      M-010 연차 관리 (제출 완료)
```
