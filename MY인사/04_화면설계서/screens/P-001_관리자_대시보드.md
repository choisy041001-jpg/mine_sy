## 화면 상세 설계

### P-001 관리자 대시보드

| 항목 | 내용 |
|------|------|
| **화면 목적** | 인사담당자가 주요 처리 현황을 한눈에 파악하고 업무를 신속히 처리한다. |
| **진입 조건** | 관리자 계정으로 로그인 시 기본 화면 / 사이드바 [대시보드] 클릭 |
| **다음 화면** | P-002 / P-003 / P-004 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| KPI 카드 (상단 4개) | 대기중 연차 결재 건수 | 클릭 시 연차 관리(P-002) 이동 |
| KPI 카드 | 미처리 증명서 건수 | 클릭 시 업무신청관리(P-003) 이동 |
| KPI 카드 | 노출중인 질문 | 클릭 시 FAQ 관리(P-004) 이동 |
| KPI 카드 | 총 직원수 | 클릭 불가 |
| 미처리 업무 패널 | 대기중 증명서 신청 행 | 직원명·종류·신청일·[발급] 버튼 — 클릭 시 P-003 업무신청관리의 발급처리 팝업 호출 |
| 미처리 업무 패널 | 대기중 연차 신청 행 | 직원명·날짜·종류·[승인] 버튼 — 클릭 시 경리나라 결재함을 새 탭으로 열어줌 |
| 직원 현황 패널 (우측) | 탭 | 전체 / 재직 / 휴직 / 퇴직 |
| 직원 현황 패널 | 테이블 | 이름·부서·직급·입사일·연락처·상태 |

**사용자 흐름**
- KPI 카드 클릭 → 해당 관리 페이지 이동
- 미처리 업무 증명서/연차 행 → 해당 처리 페이지 이동

**조건부 규칙**

**KPI 카드**
- 모든 KPI 수치(대기중 연차 결재 건수 / 미처리 증명서 건수 / 노출중인 질문 수 / 총 직원수)는 **페이지 로드 시 1회 갱신**. 실시간 폴링 없음.

**미처리 업무 패널**
- 증명서·연차 각각 최대 **10건**까지 표시. 10건 초과분은 표시하지 않으며, 전체 목록은 해당 관리 페이지에서 조회.
- 10건 미만일 경우에도 미처리 업무 패널 카드와 직원 현황 카드의 높이는 **최대 사이즈(10건 기준)로 고정**. 건수가 적다고 카드 하단이 줄어들지 않음.
- 미처리 업무가 0건일 때: 패널 내 **"미처리 업무가 없습니다."** 빈 상태 표시.
- [승인] 버튼 클릭 시 경리나라 결재함을 **새 창(새 탭)으로 열어줌**. MY인사에서 직접 연차를 승인하지 않음.
- [발급] 버튼 클릭 시 **P-003 업무신청관리의 발급처리 팝업을 호출**. 팝업에서 PDF 첨부 후 발급 완료 처리.

**직원 현황 패널**
- 기본 탭: **전체**. 페이지 로드 시 전체 탭이 선택된 상태로 진입.
- 표시 대상: 경리나라 직원관리에 등록된 **일반근로자** 목록.

## PC 공통 규칙

> 모든 PC 화면(P-001 ~ P-012)에 공통 적용됩니다.

| 규칙 | 내용 |
|------|------|
| **로그인 방식** | MY인사 PC는 별도 로그인 화면 없이 그룹웨어(경리나라) 로그인 세션 정보를 사용하여 자동 진입. 세션 만료 시 그룹웨어 로그인 화면으로 리디렉션. |
| **권한 위반 접근 차단** | 직원이 관리자 전용 URL(P-001~P-004)에 직접 접근할 경우 접근 불가 처리. 관리자가 직원 전용 URL(P-010~P-012)에 접근할 경우도 동일하게 접근 불가 처리. |
