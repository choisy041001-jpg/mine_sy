### P-001 관리자 대시보드

| 항목 | 내용 |

| **화면 목적** | 인사담당자가 주요 처리 현황을 한눈에 파악하고 업무를 신속히 처리한다. |
| **진입 조건** | 관리자 계정으로 로그인 시 기본 화면 / 사이드바 [대시보드] 클릭 |

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| KPI 카드 (상단 4개) | 승인할 연차 | 클릭 시 연차 관리(P-002) 이동 |

**사용자 흐름**
- KPI 카드 클릭 → 해당 관리 페이지 이동
- 미처리 업무 증명서/연차 행 → 해당 처리 페이지 이동

**조건부 규칙**

**KPI 카드 — 데이터 기준**

각 카드 수치는 아래 기준으로 집계한다. 모든 KPI 수치는 **페이지 로드 시 1회 갱신**하며 실시간 폴링은 없다.

| 카드 | 집계 데이터 기준 | 클릭 이동 |
|------|----------------|----------|
| **승인할 연차** | **로그인한 관리자 본인이 결재해야 하는** 연차 결재 대기 건수. 대상 결재 유형: ① 연차 사용 신청(휴가신청서) ② 연차 취소 신청. 경리나라 결재함에서 본인이 현재 결재선상 결재 차례인(결재 대기) 건만 집계한다. 본인이 결재자가 아니거나 이미 처리한 건은 제외. | P-002 연차 관리 |
| **미처리 증명서** | 업무신청관리(P-003)의 처리 상태가 **'대기'** 인 증명서 신청 건수. '발급완료'·'반려' 건은 제외. | P-003 업무신청관리 |
| **노출중인 질문** | FAQ 관리(P-004)의 노출 상태가 **'노출 중'** 인 FAQ 건수. '비노출' 건은 제외. | P-004 FAQ 관리 |
| **총 직원수** | 경리나라 직원관리에 등록된 일반근로자 전체 인원수(재직·휴직·퇴직 포함, 직원 현황 패널 '전체' 탭과 동일 기준). | 클릭 불가 |

**미처리 업무 패널**
- 증명서·연차 각각 최대 **10건**까지 표시. 10건 초과분은 표시하지 않으며, 전체 목록은 해당 관리 페이지에서 조회.
- 10건 미만일 경우에도 미처리 업무 패널 카드와 직원 현황 카드의 높이는 **최대 사이즈(10건 기준)로 고정**. 건수가 적다고 카드 하단이 줄어들지 않음.
- 미처리 업무가 0건일 때: 패널 내 **"미처리 업무가 없습니다."** 빈 상태 표시.
- [승인] 버튼 클릭 시 경리나라 결재함을 **새 창(새 탭)으로 열어줌**. MY인사에서 직접 연차를 승인하지 않음.
- [발급] 버튼 클릭 시 **P-003 업무신청관리의 발급처리 팝업을 호출**. 팝업에서 PDF 첨부 후 발급 완료 처리.

**직원 현황 패널**
- 기본 탭: **전체**. 페이지 로드 시 전체 탭이 선택된 상태로 진입.
- 표시 대상: 경리나라 직원관리에 등록된 **일반근로자** 목록.

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)

### 1. KPI 카드 (상단 4개)

각 카드는 `라벨`(상단 텍스트) + `수치`(하단 대형 숫자) 2개 필드로 구성된다. 집계 기준은 위 **KPI 카드 — 데이터 기준** 표를 따른다. 필드별 표시 규칙은 다음과 같다.

- **라벨**: 고정 문구. 좌→우 순서로 `승인할 연차` / `미처리 증명서` / `노출중인 질문` / `총 직원수`.
- **수치**: 정수만 표시(소수점·단위 없음). 집계 결과가 0이면 `0`을 그대로 표시한다(카드를 숨기지 않음). 데이터 조회 실패 시에도 카드 레이아웃은 유지하며 수치 자리에 `-`를 표시한다.
- **갱신 시점**: 페이지 최초 로드 시 1회 조회. 이후 실시간 폴링·자동 갱신 없음. 다른 화면에서 돌아오거나 새로고침 시 재조회.
- **클릭 동작**: `승인할 연차`→P-002, `미처리 증명서`→P-003, `노출중인 질문`→P-004로 이동. `총 직원수`는 클릭 불가(커서 변화 없음).

### 2. 미처리 업무 패널 (좌측)

패널 헤더에 `미처리 업무 (N)`을 표시하며, **N = 표시 중인 증명서 행 수 + 연차 행 수**이다. 패널 본문은 증명서 신청 행과 연차 신청 행이 시간순으로 혼합되어 나열된다. 각 행은 아래 필드로 구성된다.

- **유형 아이콘**: 행 종류(증명서/연차)에 따라 서로 다른 아이콘 표시.
- **제목(`alert-title`)**:
  - 증명서 행 — `{증명서 종류} 발급 요청` (예: `재직증명서 발급 요청`). 종류는 업무신청관리 신청 데이터의 증명서 종류를 따른다.
  - 연차 행 — `연차 신청 — {신청 날짜} ({일수})` (예: `연차 신청 — 5/4 (1일)`). 날짜·일수는 결재함 연차 신청 데이터를 따르며, 반차/반반차는 `0.5일`·`0.25일`로 표기.
- **메타(`alert-meta`)**: `{신청자명} · {부서} · {요청 일시}` 형식. 요청 일시는 신청 시각을 **`YYYY-MM-DD HH:MM`** 형식의 절대 일시로 표기한다(예: `2026-05-29 15:10`). 상대 시간(예: `2시간 전`)은 사용하지 않는다.
- **액션 버튼**:
  - 증명서 행 — `[발급]` 버튼. 클릭 시 P-003 업무신청관리의 **발급처리 팝업** 호출.
  - 연차 행 — `[승인]` 버튼. 클릭 시 **경리나라 결재함을 새 창(새 탭)으로** 오픈. MY인사에서 직접 승인하지 않음.
- **표시 데이터 기준**:
  - 증명서 행 — 업무신청관리 처리 상태 `대기` 건(미처리 증명서 KPI와 동일 모집단).
  - 연차 행 — 로그인 관리자 본인이 결재 차례인 연차 사용/취소 결재 대기 건(승인할 연차 KPI와 동일 모집단).
  - 정렬: 증명서·연차 통합하여 신청 시각 최신순.
  - 각 유형 최대 **10건**까지 표시. 초과분은 표시하지 않으며 전체 목록은 해당 관리 페이지에서 조회.
- **빈 상태**: 미처리 업무가 0건이면 본문에 **"미처리 업무가 없습니다."** 표시. 이때도 패널 높이는 최대 사이즈(10건 기준)로 고정.

### 3. 직원 현황 패널 (우측)

상단 **상태 탭** + 하단 **직원 테이블**로 구성된다.

- **상태 탭**: `전체` / `재직` / `휴직` / `퇴직` 4종. 각 탭 우측에 해당 상태 인원수를 표시(예: `전체 9`, `재직 8`). 기본 선택 탭은 **전체**이며 페이지 로드 시 전체가 선택된 상태로 진입. 탭 클릭 시 테이블이 해당 상태 직원으로 필터링된다.
- **직원 테이블 열(필드)**: `이름` / `부서` / `직급` / `입사일` / `연락처` / `상태`.
  - 모든 값은 경리나라 직원관리에 등록된 **일반근로자** 데이터에서 가져온다.
  - `입사일`은 `YYYY.MM.DD` 형식.
  - `상태`는 색상 뱃지로 표시: 재직(녹색) / 휴직(주황) / 퇴직(적색).
- **빈 상태**: 선택한 탭에 해당하는 직원이 없으면 테이블 영역에 빈 상태 안내를 표시한다.

## PC 공통 규칙

> 모든 PC 화면(P-001 ~ P-012)에 공통 적용됩니다.

| 규칙 | 내용 |
|------|------|
| **로그인 방식** | MY인사 PC는 별도 로그인 화면 없이 그룹웨어(경리나라) 로그인 세션 정보를 사용하여 자동 진입. 세션 만료 시 그룹웨어 로그인 화면으로 리디렉션. |
| **권한 위반 접근 차단** | 직원이 관리자 전용 URL(P-001~P-004)에 직접 접근할 경우 접근 불가 처리. 관리자가 직원 전용 URL(P-010~P-012)에 접근할 경우도 동일하게 접근 불가 처리. |
