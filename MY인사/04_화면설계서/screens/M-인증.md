## 화면 상세 설계

### M-001 로그인

| 항목 | 내용 |
|------|------|
| **화면 목적** | 사용자가 아이디·비밀번호로 인증하고, 소속 사업장을 선택하여 앱에 진입한다. |
| **진입 조건** | ① 앱 최초 실행 시 ② 로그아웃 후 ③ 세션 만료(자동 로그인 미설정 시 앱 재실행) 시 → 하단 탭바 미표시 |
| **다음 화면** | 사업장 선택 팝업 M-001-P |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 로고 영역 | 앱 아이콘 + 앱 이름 | 파란 원형 아이콘 + "MY인사" |
| 입력 영역 | 아이디 입력 필드 | placeholder "이메일 또는 사번" / 최대 100자 / 영문·숫자·특수문자 |
| 입력 영역 | 비밀번호 입력 필드 | 텍스트 가림 처리 + 우측 표시/숨기기 토글 아이콘 / 최대 50자 |
| 옵션 영역 | 자동 로그인 체크박스 | 기본값: 미체크 |
| 액션 영역 | [로그인] 버튼 | 화면 전체 너비 |
| 액션 영역 | 아이디 찾기 링크 | 버튼 하단. 탭 시 M-009(아이디 찾기 모드)로 이동 |
| 액션 영역 | 비밀번호 변경 링크 | 버튼 하단. 탭 시 M-009(비밀번호 변경 모드)로 이동 |

**사용자 흐름**

1. 아이디 + 비밀번호 입력
2. [로그인] 버튼 탭
   - 인증 성공 → 사업장 선택 팝업(M-001-P) 표시
   - 인증 실패 → 에러 메시지 표시 (화면 유지)
3. 사업장 선택 → 게시판 홈(M-002) 진입
- [아이디 찾기] 탭 → M-009 (아이디 찾기 모드)
- [비밀번호 변경] 탭 → M-009 (비밀번호 변경 모드)

**조건부 규칙**
- 자동 로그인 체크 시: 다음 앱 실행 시 로그인 화면 건너뛰고 사업장 선택 팝업 또는 마지막 사업장으로 바로 진입
- 비밀번호 표시/숨기기: 토글 아이콘 탭마다 전환

**유효성 규칙**

| 오류 조건 | 오류 표시 위치 | 오류 메시지 | 처리 결과 |
|---------|-------------|-----------|---------|
| 아이디 또는 비밀번호 불일치 | 비밀번호 입력 필드 하단 | "아이디 또는 비밀번호가 올바르지 않습니다" | 비밀번호 필드 초기화, 화면 유지. 실패 횟수 누적 |
| 로그인 5회 연속 실패 | 비밀번호 입력 필드 하단 | "로그인 시도 횟수(5회)를 초과하였습니다. 계정이 잠겼습니다. 관리자에게 문의하세요" | 계정 잠금 (경리나라 계정 잠금 정책 연동). 잠금 해제 전까지 로그인 불가 |

### M-001-P 사업장 선택 팝업 `[팝업 오버레이]`

| 항목 | 내용 |
|------|------|
| **화면 목적** | 로그인 성공 후, 해당 계정에 연결된 사업장 중 하나를 선택하여 해당 사업장 기준으로 앱을 사용한다. |
| **진입 조건** | 로그인 버튼 탭 → 인증 성공 시 현재 화면 위에 오버레이로 표시 |
| **다음 화면** | 게시판 홈 M-002 |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 헤더 영역 | 소제목 | "로그인" |
| 헤더 영역 | 제목 | "사업장을 선택하세요" |
| 헤더 영역 | 부제 | "{이름} 님이 소속된 사업장 목록입니다" |
| 사업장 목록 | 항목 | 사업장명 / 사업자번호 · 가입 제품명(we경리나라) + 우측 화살표(→) |
| 하단 버튼 | [취소] 버튼 | 팝업 닫기, 로그인 화면 유지 |

**사용자 흐름**
- 사업장 항목 탭 → 해당 사업장으로 로그인 → 게시판 홈(M-002) 이동
- [취소] 탭 → 팝업 닫힘, 로그인 화면 유지 (재로그인 가능)

**조건부 규칙**
- 소속 사업장이 1개뿐인 경우: 팝업 없이 바로 해당 사업장으로 진입 (UX 간소화)
- 소속 사업장이 2개 이상인 경우: 팝업 표시
- 목록이 일정 높이 초과 시: 내부 스크롤 처리

### M-009 아이디 찾기 / 비밀번호 변경

| 항목 | 내용 |
|------|------|
| **화면 목적** | 아이디를 잊은 경우 성명·휴대폰 또는 이메일 인증으로 아이디를 확인하거나, 비밀번호를 변경한다. |
| **진입 조건** | 로그인(M-001) → [아이디 찾기] 탭 (아이디 찾기 모드) / [비밀번호 변경] 탭 (비밀번호 변경 모드) |
| **다음 화면** | 로그인 M-001 |

> 진입 시 모드가 결정되며, 화면 내 별도 모드 전환 탭은 없다. 타이틀은 진입 모드에 따라 동적으로 표시된다.

**공통 화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 헤더 | ×(닫기) 버튼 | 탭 시 M-001 복귀 |
| 헤더 | 타이틀 | 진입 모드에 따라 "아이디 찾기" 또는 "비밀번호 변경" 동적 표시 |
| 인증 방식 탭 | 휴대폰 / 이메일 | 탭 전환 시 해당 인증 폼으로 전환 |

**아이디 찾기 — 휴대폰 인증**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 입력 폼 | 성명 | 직접 입력 |
| 입력 폼 | 휴대폰번호 | 국가코드(+82) + 번호 입력 / 우측 [인증] 버튼 탭 시 인증번호 발송 |
| CAPTCHA | 이미지 텍스트 + 새로고침 버튼 | |
| 입력 폼 | CAPTCHA 텍스트 | 이미지 텍스트 직접 입력 |
| 입력 폼 | 인증번호 | 발송된 번호 입력 / 우측 [확인] 버튼 |
| 하단 버튼 | [찾기] | 아이디 조회 실행 |
| 하단 버튼 | [취소] | M-001 복귀 |

**아이디 찾기 — 이메일 인증**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 입력 폼 | 성명 | 직접 입력 |
| 입력 폼 | 이메일 | 직접 입력 |
| 하단 버튼 | [찾기] | 이메일로 아이디 발송 실행 |
| 하단 버튼 | [취소] | M-001 복귀 |

**비밀번호 변경 — 휴대폰 인증**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 입력 폼 | 아이디 | 직접 입력 |
| 입력 폼 | 성명 | 직접 입력 |
| 입력 폼 | 휴대폰번호 | 국가코드(+82) + 번호 입력 / 우측 [인증] 버튼 탭 시 인증번호 발송 |
| CAPTCHA | 이미지 텍스트 + 새로고침 버튼 | |
| 입력 폼 | CAPTCHA 텍스트 | 이미지 텍스트 직접 입력 |
| 입력 폼 | 인증번호 | 발송된 번호 입력 / 우측 [확인] 버튼 |
| 하단 버튼 | [변경] | 비밀번호 변경 실행 |
| 하단 버튼 | [취소] | M-001 복귀 |

**비밀번호 변경 — 이메일 인증**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 입력 폼 | 아이디 | 직접 입력 |
| 입력 폼 | 성명 | 직접 입력 |
| 입력 폼 | 이메일 | 직접 입력 |
| 하단 버튼 | [변경] | 이메일로 비밀번호 변경 링크 발송 실행 |
| 하단 버튼 | [취소] | M-001 복귀 |

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 인증번호 불일치 | "인증번호가 올바르지 않습니다" 안내 메시지 |
| 일치하는 정보 없음 | "입력하신 정보와 일치하는 계정이 없습니다" 안내 메시지 |

## 화면 전환 흐름

```
M-001 로그인
  ├─ 아이디·비밀번호 입력 → [로그인]
  │     ↓
  │   M-001-P 사업장 선택
  │     소속 사업장 선택
  │       ↓
  │     M-002 게시판 홈
  │
  ├─ [아이디 찾기] 탭
  │     ↓
  │   M-009 아이디 찾기 모드
  │     인증 완료 → 아이디 표시
  │       ↓
  │     M-001 로그인 복귀
  │
  └─ [비밀번호 변경] 탭
        ↓
      M-009 비밀번호 변경 모드
        인증 완료 → 비밀번호 변경 처리
          ↓
        M-001 로그인 복귀
```
