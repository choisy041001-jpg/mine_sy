| **진입 조건** | 사이드바 [업무신청] |

**빈 상태**

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)
> **데이터 출처**: 이력 테이블은 **직원 본인의 증명서 신청 내역**이다. 신청 시 입력값(종류·제출처/용도·요청 메모·기타증명서 명칭)은 관리자 업무신청관리(P-003)로 전달되어 발급 처리된다. 본 화면에서 직원은 신청·취소·다운로드·반려 사유 확인만 수행한다.

### 1. 상단 필터·액션

- **증명서 종류 필터**: `전체 종류` / `재직증명서` / `경력증명서` / `기타증명서`.
- **상태 필터**: `전체 상태` / `대기` / `발급완료` / `반려`.
- **[증명서 신청] 버튼**(accent): 클릭 시 2단계 신청 팝업(Step 1 종류 선택) 오픈.

### 2. 이력 테이블

각 행은 본인의 증명서 신청 1건이다. **신청일 기준 최신순** 정렬, 전체 이력 표시. 열(필드)은 좌→우로 다음과 같다.

- **신청일**: 신청 일시. `YYYY.MM.DD HH:MM` 형식(예: `2026.04.29 09:14`).
- **증명서 종류**: 종류 뱃지 — `재직증명서` / `경력증명서` / `기타증명서`(종류별 색상 뱃지).
- **제출처 / 용도**: 신청 시 입력한 제출처·용도 텍스트(예: `은행 대출 신청용`).
- **처리 상태**: 뱃지 — `대기`(주황, `badge w`) / `발급완료`(녹색, `badge s`) / `반려`(빨강, `badge d`).
- **액션 버튼**(상태별):
  - 대기 → **[신청 취소]**: 확인 다이얼로그(`신청을 취소하시겠습니까?`) → 확인 시 취소 처리 + 토스트(`신청이 취소되었습니다.`).
  - 발급완료 → **[다운로드]**(accent): 클릭 시 발급 PDF 다운로드 + 토스트(`📄 {종류}를 다운로드합니다.`).
  - 반려 → **[사유보기]**: 반려 사유 팝업 오픈.
- **빈 상태**: 표시할 이력이 없으면 **"내역이 없습니다."**

### 3. 증명서 신청 팝업 (2단계)

- **Step 1 — 종류 선택**(제목 `증명서 신청 — 종류 선택`): 항목 버튼 3종. 각 버튼은 아이콘 + 종류명 + 설명을 포함한다.
  - 재직증명서 — `현재 재직 사실을 증명하는 문서`.
  - 경력증명서 — `입사일·직위·담당업무 등 경력 증명`.
  - 기타증명서 — `근속확인서, 급여확인서 등 기타 문서`.
  - 항목 클릭 시 Step 2로 전환.
- **Step 2 — 신청 정보 입력**(제목 `{종류} 신청`):
  - **명칭**(기타증명서 선택 시에만 노출, **필수**): 단일 라인, 최대 50자. placeholder `예) 근속확인서, 급여확인서 등`.
  - **제출처 / 용도**(**필수**): 단일 라인, 최대 100자. placeholder `예) 은행 대출 신청용, 비자 신청용 등`.
  - **요청 메모**(선택): 멀티 라인. placeholder `특별 요청사항을 입력해주세요(예: 담당자 서명 추가 요청 등)`.
  - **버튼**: [취소] / [신청하기]. 신청하기 시 필수값 검증(기타증명서 명칭·제출처/용도) 후 팝업 닫고 토스트 `✅ {종류} 신청이 완료되었습니다. 인사담당자 확인 후 발급됩니다.`.

### 4. 반려 사유 팝업

- **제목**: `반려 사유`.
- **본문**: 관리자(P-003)가 입력한 반려 사유를 danger 배경 박스에 읽기 전용으로 표시.
- **버튼**: [확인].
