## 화면 상세 설계

### P-003 업무신청관리 — 증명서

| 항목 | 내용 |
|------|------|
| **화면 목적** | 직원이 신청한 증명서 목록을 관리하고, 발급 처리 또는 반려 처리를 진행한다. |
| **진입 조건** | 사이드바 [업무신청관리] / 대시보드 KPI 카드 클릭 |
| **다음 화면** | 없음 (현재 페이지 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 필터 | 증명서 종류 필터 / 처리 상태 필터 / 기간 필터 (날짜 범위) | **기본값: 최근 1개월**. 직접 날짜 범위 변경 가능 |
| 신청 목록 테이블 | 열 구성 | 신청일 / 신청자 / 증명서 종류 / 용도 / 처리 상태 / 액션 버튼 |
| 신청 목록 테이블 | 대기 상태 | [발급처리] 버튼 |
| 신청 목록 테이블 | 발급완료 상태 | [이력보기] 버튼 |
| 신청 목록 테이블 | 반려 상태 | [사유보기] 버튼 |
| 발급처리 팝업 | 안내 문구 + PDF 파일 첨부(**필수**) + 메모(선택) | PDF: 최대 50 MB / 단일 파일만 지원 / 재첨부 시 기존 파일 교체 / 메모: 최대 300자 / 다중 라인 |
| 발급처리 팝업 | [반려] 버튼 + [발급 완료] 버튼 | |
| 이력보기 팝업 | 발급 이력 상세 | 발급 완료된 증명서의 신청 사유 / 첨부한 PDF 파일 / 메모 내용 확인. 인사담당자가 발급 내역을 나중에 조회하기 위한 기능 |
| 반려 사유 팝업 | 반려 사유 표시 | 인사담당자가 반려 처리 당시 작성한 반려 사유 텍스트 표시 |

**사용자 흐름**
- [발급처리] 클릭 → 발급처리 팝업 오픈
- 팝업 [발급 완료] 클릭 → 발급 처리 + 직원 MY인사로 알림 발송 + 상태 업데이트
- 팝업 [반려] 클릭 → 사유 입력 후 반려 처리 + 직원 MY인사로 알림 발송
- [이력보기] 클릭 → 이력보기 팝업 오픈 (신청 사유 / 첨부 PDF / 메모 확인)
- [사유보기] 클릭 → 반려 사유 팝업 오픈 (반려 처리 당시 작성한 반려 사유 표시)

**조건부 규칙**
- **목록 기본 정렬**: 신청일 기준 최신순
- **앱 알림 전송 실패 처리**: 발급 완료 처리 후 직원 앱 알림 전송이 실패하더라도 발급 완료 상태 처리에는 영향 없음. 알림 전송 성공 여부와 발급 처리는 독립적.

**빈 상태**
- 조건에 해당하는 신청 내역이 없을 경우: **"내역이 없습니다."** 표시

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 발급처리 팝업 — PDF 미첨부 | [발급 완료] 버튼 비활성화 |
| [반려] 클릭 시 사유 미입력 | 반려 사유 입력 필수 |
