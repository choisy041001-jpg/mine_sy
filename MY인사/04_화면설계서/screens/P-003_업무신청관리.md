## 화면 상세 설계

### P-003 업무신청관리 — 증명서

| 항목 | 내용 |
|------|------|
| **화면 목적** | 직원이 신청한 증명서 목록을 관리하고, 발급 처리 또는 반려 처리를 진행한다. |
| **진입 조건** | 사이드바 [업무신청관리] / 대시보드 KPI 카드 클릭 |
| **다음 화면** | 없음 (현재 페이지 내 처리) |

**화면 구성**

| 영역 | 컴포넌트 | 상세 |
|------|---------|------|
| 상단 필터 | 증명서 종류 필터 / 처리 상태 필터 / 기간 필터 (날짜 범위) | **기본값: 최근 1개월**. 직접 날짜 범위 변경 가능 |
| 상단 필터 | 신청자 검색 | 직원명으로 검색 (이 화면에서는 신청자 이름만 확인 가능하므로 이름으로만 검색) |
| 신청 목록 테이블 | 열 구성 | 신청일 / 신청자 / 증명서 종류 / 제출처·용도 / 처리 상태 / 액션 버튼 |
| 신청 목록 테이블 | 대기 상태 | [발급처리] 버튼 |
| 신청 목록 테이블 | 발급완료 상태 | [이력보기] 버튼 |
| 신청 목록 테이블 | 반려 상태 | [사유보기] 버튼 |
| 발급처리 팝업 | 안내 문구 + PDF 파일 첨부(**필수**) + 메모(선택) | PDF: 최대 50 MB / 단일 파일만 지원 / 재첨부 시 기존 파일 교체 / 메모: 최대 300자 / 다중 라인 |
| 발급처리 팝업 | [반려] 버튼 + [발급 완료] 버튼 | [반려] 클릭 시 발급처리 팝업을 닫고 별도 **반려 입력 팝업** 오픈 |
| 반려 입력 팝업 | 신청자·종류 요약 + 반려 사유 입력(**필수**) | 사유: 최대 300자 / 다중 라인 / 카운터 표시. [반려 확정] 시 직원 앱 알림 발송 |
| 이력보기 팝업 | 발급 이력 상세 | 발급 완료된 증명서의 신청 사유 / 첨부한 PDF 파일 / 메모 내용 확인. 인사담당자가 발급 내역을 나중에 조회하기 위한 기능 |
| 반려 사유 팝업 | 반려 사유 표시 | 인사담당자가 반려 처리 당시 작성한 반려 사유 텍스트 표시 |

**사용자 흐름**
- [발급처리] 클릭 → 발급처리 팝업 오픈
- 팝업 [발급 완료] 클릭 → 발급 처리 + 직원 MY인사로 알림 발송 + 상태 업데이트
- 팝업 [반려] 클릭 → 사유 입력 후 반려 처리 + 직원 MY인사로 알림 발송
- [이력보기] 클릭 → 이력보기 팝업 오픈 (신청 사유 / 첨부 PDF / 메모 확인)
- [사유보기] 클릭 → 반려 사유 팝업 오픈 (반려 처리 당시 작성한 반려 사유 표시)

**조건부 규칙**
- **목록 기본 정렬**: 신청일 기준 최신순
- **앱 알림 전송 실패 처리**: 발급 완료 처리 후 직원 앱 알림 전송이 실패하더라도 발급 완료 상태 처리에는 영향 없음. 알림 전송 성공 여부와 발급 처리는 독립적.
- **좌측 메뉴 숫자 뱃지**: 좌측 사이드바 [업무신청관리] 메뉴에 표시되는 숫자 뱃지는 처리 상태가 **'대기'** 인 증명서 신청 건수와 동일하다(대시보드 '미처리 증명서' KPI와 같은 기준). **'대기' 건수가 0건이면 뱃지를 숨기지 않고 `0`으로 표시한다.** 좌측 메뉴 숫자 뱃지는 업무신청관리 메뉴에만 표시하며, 연차 관리·FAQ 관리 메뉴에는 표시하지 않는다.

**빈 상태**
- 조건에 해당하는 신청 내역이 없을 경우: **"내역이 없습니다."** 표시

**유효성 규칙**

| 오류 조건 | 처리 결과 |
|---------|---------|
| 발급처리 팝업 — PDF 미첨부 | [발급 완료] 버튼 비활성화 |
| [반려] 클릭 시 사유 미입력 | 반려 사유 입력 필수 |

## 세부 설계 요건 — 데이터 표시 기준

> 화면의 각 영역이 **어떤 데이터를 어디서 가져와 어떤 형식으로 표시하는지**를 필드 단위로 정의한다. (UI 배치·스타일은 위 화면 구성 및 PC 컴포넌트 규칙 참조)
> **데이터 출처**: 직원이 모바일 증명서 신청(M-031)에서 제출한 신청 건이 관리자 업무신청관리 목록으로 수집된다. 신청자·신청일·종류·용도는 직원의 신청 데이터, 발급 PDF·메모·반려 사유는 관리자(인사담당자) 처리 데이터다.

### 1. 상단 필터

직원 신청 목록을 좁히기 위한 조회 조건. 변경 시 목록이 즉시 갱신된다.

- **증명서 종류 필터**: 단일 선택 드롭다운. 옵션 = `전체 종류` / `재직증명서` / `경력증명서` / `기타증명서`. 기본값 `전체 종류`.
- **처리 상태 필터**: 단일 선택 드롭다운. 옵션 = `전체 상태` / `대기` / `발급완료` / `반려`. 기본값 `전체 상태`.
- **기간 필터**: 시작일·종료일 두 개의 날짜 입력(`YYYY-MM-DD`). **신청일 기준** 범위. **기본값: 최근 1개월**(오늘 기준 직전 1개월 ~ 오늘). 직접 변경 가능.
- **신청자 검색**: 텍스트 입력. **직원명**으로 부분 일치 검색. 이 화면에서는 신청자 이름만 확인 가능하므로 사번은 검색 대상에 포함하지 않는다. 1자 이상 입력 시 실시간 필터링(300ms 디바운스).

### 2. 신청 목록 테이블

각 행은 직원이 신청한 증명서 1건이다. 열(필드)은 좌→우로 다음과 같다. **신청일 기준 최신순** 정렬(정렬 변경 UI 없음).

- **신청일**: 직원이 증명서를 신청한 일시. 형식 `YYYY.MM.DD HH:MM`(24시간제). (예: `2026.04.29 09:14`)
- **신청자**: 신청 직원명(굵게). 경리나라 직원관리 등록 직원명. (사번은 표시하지 않으며 검색 대상도 아님)
- **증명서 종류**: 종류별 파스텔 색상 뱃지(`cert-badge-*`). `재직증명서` / `경력증명서` / `기타증명서`.
- **제출처 / 용도**: 직원이 신청 시 입력한 자유 텍스트(제출처·사용 목적). (예: `은행 대출 신청용`, `해외 프리랜서 계약`, `육아휴직 신청용 근속확인서`)
- **상태**: 처리 상태 뱃지. `대기`(주황) / `발급완료`(녹색) / `반려`(빨강).
- **액션**(상태별 단일 버튼):
  - 대기 → **[발급처리]**(primary) → 발급 처리 모달(3) 오픈.
  - 발급완료 → **[이력보기]** → 발급 이력보기 모달(5) 오픈.
  - 반려 → **[사유보기]** → 반려 사유 보기 모달(6) 오픈.
- **빈 상태**: 필터 결과가 없으면 **"내역이 없습니다."** 표시.

### 3. 발급 처리 모달 (대기 건 [발급처리])

대기 상태 신청 건을 발급 완료 또는 반려 처리하는 모달.

- **제목**: `{증명서 종류} 발급 처리` (예: `재직증명서 발급 처리`).
- **서브 메타**: `{신청자} · {제출처/용도} · {신청일시}` (예: `홍길동 · 은행 대출 신청용 · 2026.04.29 09:14`).
- **안내 문구**(고정): "직접 작성한 PDF 파일을 첨부하여 발급합니다. 발급 완료 시 신청자의 MY인사 앱으로 즉시 전달됩니다."
- **증명서 파일 첨부**(필수, ＊): 허용 형식 **PDF 단일 파일**, **최대 50 MB**. 재첨부 시 기존 파일 교체(1건 유지). 첨부 후 파일명·용량이 목록으로 표시.
- **인사담당자 메모**(선택): 다중 라인 입력. **최대 300자**, 카운터 표시. 입력 내용은 **직원에게 전달**되며 발급 이력 모달·직원 앱에서 확인된다.
- **버튼 구성**:
  - **[반려]**(좌측, danger) → 현재 모달을 닫고 **반려 입력 모달**(4)을 연다.
  - **[취소]** → 모달 닫기(처리 없음).
  - **[발급 완료]**(primary) → **PDF 미첨부 시 처리 차단**(첨부 없이 클릭 시 "파일을 첨부해주세요." 안내). 첨부 완료 시 발급 처리 → 상태 `발급완료`로 변경 + 직원 앱 푸시(발급완료) + "발급이 완료되었습니다. 신청자에게 발송되었습니다." 토스트.

### 4. 반려 입력 모달

발급 처리 모달의 [반려]에서 진입. 반려 사유를 입력해 신청을 반려한다.

- **제목**: `신청 반려`.
- **요약 정보**: `신청자`, `증명서 종류`(발급 처리 대상 건에서 전달).
- **반려 사유**(필수, ＊): 다중 라인 입력. **최대 300자**, 카운터 표시. 미입력 시 반려 확정 불가(필드 강조).
- **안내 문구**(고정): "＊ 작성한 사유는 신청자에게 전달됩니다."
- **버튼**: **[취소]** / **[반려 확정]**(danger) → 상태 `반려`로 변경 + 반려 사유 저장 + 직원 앱 푸시(반려) + "반려 처리가 완료되었습니다." 토스트.

### 5. 발급 이력보기 모달 (발급완료 건 [이력보기])

발급 완료된 증명서의 발급 내역을 조회한다(읽기 전용).

- **제목**: `발급 이력 — {신청자}`. 서브 메타: `{신청자} · {증명서 종류} · {발급일시} 발급`.
- **발급 상세** 섹션(라벨 → 값):
  - `증명서 종류` → 종류명.
  - `신청자` → 직원명.
  - `신청 사유` → 직원이 신청 시 입력한 제출처/용도.
  - `발급일시` → 발급 완료 처리 일시.
  - `발급 담당자` → 처리한 인사담당자명.
  - `수령 방법` → `MY인사 앱·PC(직원용) 다운로드`(직원이 **모바일 MY인사 앱** 또는 **PC(직원용)** 에서 발급 완료된 PDF를 다운로드).
- **첨부파일** 섹션: 발급 시 첨부한 PDF 목록. 각 항목 = 파일명 + 용량(예: `재직증명서_이대리_20260427.pdf · 142 KB`). 클릭 시 다운로드. 첨부가 없으면 **"첨부파일이 없습니다."**
- **인사담당자 메모** 섹션: 발급 시 입력한 메모 텍스트(없으면 빈 영역).

### 6. 반려 사유 보기 모달 (반려 건 [사유보기])

- **제목**: `반려 사유`.
- **본문**: 반려 처리 당시 인사담당자가 작성한 **반려 사유 텍스트**(danger 톤 박스). 신청자에게 전달된 것과 동일한 내용.
- **버튼**: **[확인]** → 닫기.
