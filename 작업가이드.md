## 1. 기본 개념 이해하기

### Git이란?
문서의 **변경 이력을 저장하는 도구**예요.
마치 문서를 저장할 때마다 "스냅샷"을 찍어두는 것과 같아요.
언제든지 과거 버전으로 돌아갈 수 있어요.

### GitHub란?
내 컴퓨터의 문서를 **인터넷에 올려두는 저장소**예요.
팀원 모두가 같은 문서를 보고 수정할 수 있어요.

### 작업 흐름 한눈에 보기

```
내 컴퓨터에서 문서 작성
        ↓
  git add (변경사항 선택)
        ↓
  git commit (저장 확정)
        ↓
  git push (GitHub에 업로드)
        ↓
  GitHub에서 팀원과 공유
```

## 2. 처음 시작할 때 (최초 1회)

### VS Code 열기
1. VS Code 실행
2. 상단 메뉴 → **터미널** → **새 터미널**
3. 터미널에서 작업 폴더로 이동:
```bash
cd "C:\Users\최서연\Desktop\서비스관리"
```

### 잘 연결됐는지 확인
```bash

```
아래처럼 뜨면 정상이에요:
```
On branch main
Your branch is up to date with 'origin/main'.
nothing to commit, working tree clean
```

## 3. 매일 문서 작업하는 방법

### Step 1 — 작업 시작 전: 최신 파일 받아오기
팀원이 수정한 내용이 있을 수 있으니, 작업 전에 항상 먼저 받아오세요.
```bash
git pull
```

### Step 2 — 문서 작성/수정
VS Code에서 원하는 파일을 열어서 작성하세요.
(Markdown 미리보기: `Ctrl + Shift + V`)

### Step 3 — 변경사항 확인
내가 뭘 수정했는지 확인하기:
```bash
git status
```

빨간색으로 표시된 파일 = 수정된 파일

### Step 4 — 변경사항 선택 (add)
```bash
git add .
```
> `.` 은 변경된 파일 전체를 선택한다는 뜻이에요.
> 특정 파일만 선택하려면: `git add 파일명.md`

### Step 5 — 저장 확정 (commit)
```bash
git commit -m "update: 로그인 화면 스토리보드 수정"
```
> `""` 안에 어떤 내용을 수정했는지 간단히 적어주세요.

### Step 6 — GitHub에 올리기 (push)
```bash
git push
```

## 4. 새 문서 만들기

### 방법 1 — 템플릿 복사해서 시작하기 (추천)

1. `02_PRD/templates/PRD-template.md` 파일을 복사
2. 해당 폴더에 붙여넣고 파일명 변경
   - 예: `PRD-쇼핑앱-v1.0.md`
3. 파일 열어서 내용 작성
4. 작업 완료 후 GitHub에 올리기:

```bash
git add .
git commit -m "feat: 쇼핑앱 PRD 초안 추가"
git push
```

### 파일명 규칙

| 문서 유형 | 파일명 예시 |
|-----------|------------|
| 요구사항정의서 | `requirements-쇼핑앱-v1.0.md` |
| PRD | `PRD-쇼핑앱-v1.0.md` |
| 스토리보드 | `storyboard-로그인-v1.0.md` |
| 화면설계서 | `screen-메인화면-v1.0.md` |
| IA | `IA-쇼핑앱-v1.0.md` |
| 플로우차트 | `flow-결제-v1.0.md` |

## 5. 기존 문서 수정하기

### 버전 올리기
문서를 크게 수정할 때는 파일명의 버전을 올려주세요.
- `PRD-쇼핑앱-v1.0.md` → `PRD-쇼핑앱-v1.1.md` (내용 추가/수정)
- `PRD-쇼핑앱-v1.0.md` → `PRD-쇼핑앱-v2.0.md` (전면 개편)

### 문서 상태 표기 업데이트
문서 상단의 상태를 항상 최신으로 유지하세요:
```
상태: 작성중 → 검토중 → 확정
```

### 수정 후 GitHub 올리기
```bash
git add .
git commit -m "update: 쇼핑앱 PRD 결제 기능 요구사항 추가"
git push
```

## 6. 팀원과 함께 작업하기

### 팀원이 내 저장소를 사용하려면
1. GitHub 저장소 페이지 → **Settings** → **Collaborators**
2. 팀원의 GitHub 아이디 입력 → 초대
3. 팀원이 초대 수락 후 아래 명령어로 받아오기:
```bash
git clone https://github.com/choisy041001-jpg/-.git
```

### 리뷰 요청하기 (Pull Request)
큰 문서를 작성했을 때 팀원 검토를 받는 방법이에요.

**1. 새 브랜치 만들기**
```bash
git checkout -b feature/로그인-PRD
```

**2. 문서 작성 후 push**
```bash
git add .
git commit -m "feat: 로그인 PRD 작성"
git push origin feature/로그인-PRD
```

**3. GitHub에서 Pull Request 생성**
- GitHub 저장소 → **Pull requests** → **New pull request**
- 팀원을 리뷰어로 지정
- 검토 완료 후 **main**에 병합

## 9. 문서 작성 규칙

### 커밋 메시지 규칙

| 유형 | 사용 상황 | 예시 |
|------|-----------|------|
| `feat:` | 새 문서 추가 | `feat: 결제 화면 스토리보드 추가` |
| `update:` | 기존 문서 수정 | `update: 로그인 PRD 예외처리 추가` |
| `fix:` | 오류/오타 수정 | `fix: 화면 ID 오타 수정` |
| `review:` | 검토 의견 반영 | `review: 팀장 피드백 반영` |
| `delete:` | 문서 삭제 | `delete: 구버전 PRD 삭제` |

### 문서 상태 표기
모든 문서 상단 표에 반드시 상태를 표기하세요:

- `작성중` — 아직 작성 중인 초안
- `검토중` — 팀 내부 리뷰 진행 중
- `확정` — 최종 승인 완료된 문서

### Markdown 기본 문법

```markdown
# 제목 1단계
## 제목 2단계
### 제목 3단계

**굵게**
*기울임*

- 목록 항목
- 목록 항목

| 열1 | 열2 |
|-----|-----|
| 내용 | 내용 |

> 인용문 (강조할 내용)

[링크 텍스트](URL)
```

## 10. 자주 묻는 질문 (FAQ)

**Q. 실수로 잘못 커밋했어요. 어떻게 하나요?**
```bash
git revert HEAD
```
가장 최근 커밋을 취소하는 새 커밋을 만들어요. (안전한 방법)

---

**Q. 문서를 이전 버전으로 되돌리고 싶어요.**
```bash
git log --oneline          # 커밋 이력 확인
git checkout [커밋ID] 파일명.md   # 해당 시점으로 파일 복원
```

---

**Q. push가 안 돼요.**
```bash
git pull        # 먼저 최신 내용 받아오기
git push        # 다시 push
```

---

**Q. 같은 파일을 두 사람이 동시에 수정했어요.**
`git pull` 할 때 충돌(conflict)이 발생해요.
파일을 열면 아래처럼 표시돼요:
```
<<<<<<< HEAD
내가 수정한 내용
=======
팀원이 수정한 내용
>>>>>>> origin/main
```
두 내용을 직접 합쳐서 수정 후 다시 커밋하면 돼요.
